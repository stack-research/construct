### Freeze a candidate plan before individual GETs

From those exact index bytes only:

1. Parse `modules.json`.
2. For every module entry, collect vulnerability rows whose id starts
   `GO-2026-` and whose `fixed` field is a nonempty string.
3. Build `id → sorted unique module paths + fixed values`.
4. Join to `vulns.json`; retain only ids actually present there.
5. Exclude any id with no module path, any module equal to `stdlib` or
   `toolchain`, and any module/entity colliding exactly with K1, Family C,
   DEP0033, `rw-*`, or another prior scored-corpus entity.
6. Sort retained ids lexicographically ascending.
7. Freeze the first **24** ids as the maximum candidate sequence. Fewer is
   allowed and becomes an explicit ceiling shortfall.
8. Before any individual-report GET, write create-once
   `candidate_plan.json` with:
   - hashes of all three raw indexes;
   - complete derivation rule;
   - every retained candidate id in order;
   - indexed module paths/fixed values;
   - exact individual URL
     `https://vuln.go.dev/ID/<id>.json`;
   - ceiling 24;
   - candidate-plan canonical hash.

No Kagi-proposed GO id receives priority. The prior invented/partial D nominations are rejected trace, not input.

### Individual candidate GETs

Walk `candidate_plan.json` in order. Make at most one GET per candidate, zero retries. Stop making requests as soon as six candidates qualify. Never fetch candidates after the sixth pass.

A candidate qualifies only if its individual canonical JSON:

1. HTTP succeeds and parses as OSV JSON.
2. Returned id exactly equals planned id and publication starts `2026`.
3. It is not withdrawn.
4. `database_specific.review_status` is absent or equals `REVIEWED`; reject
   `UNREVIEWED` and unknown nonempty statuses.
5. It contains at least one `affected[]` object with:
   - `package.ecosystem == "Go"`;
   - a nonempty module name matching one of that id’s index-pinned module
     paths;
   - at least one range with a finite `fixed` or `last_affected` event;
   - `ecosystem_specific.imports[]` containing at least one nonempty
     `symbols[]` string.
6. All entity keys for the report are recorded. The candidate is rejected if
   any module path collides with a previously selected report’s module set or
   any frozen exclusion entity.
7. Qualification uses only these fields. Do not read summary/details/severity
   to select or rank.

Map the first six qualifying reports, in candidate order, to
`capD-01..capD-06`. Preserve candidate raw bytes; create each capture from
the exact candidate bytes without refetch/reserialization and preserve retrieval
time/hash. If fewer than six qualify after the candidate plan is exhausted,
emit `family_d_shortfall`; do not extend the ceiling or search.

The variable live-call budget is therefore exact after indexes:
