### B1 repair

`parse_ghsa_range` now fail-closes on every form Grok and Sol reproduced:

- `>= 7.23.0 < 7.28.0` → rejected (`invalid version token: '7.23.0 < 7.28.0'`)
- `>= 1.2.3 || < 2` → rejected (`unsupported range operator`)
- `>= 1.2.3 garbage`, `>=`, `< < 2.0.0`, doubled comma, `^1.2.3`, `~1.2.3` → rejected

Each comma-separated fragment is one anchored comparator (`>=`, `<=`, `<`, `>`, `=`) plus exactly one `numeric_dotted` version token (`DIGIT+(.DIGIT+)*`).
