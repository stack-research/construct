### §9(b) — scorer & row determinism

**Four thermal rows + `thermal_projection` — deterministically diffable, fail-closed.** Yes, with the same exclusions M3 uses. Define:

- `thermal_state_projection(rows) -> dict[record_id, float]` — replay `temperature_delta` rows in strict `event_index` order (ignore `ts`; it is documentary like every other ledger row). This is the temperature analog of rebuilding authority from `authority_update` rows; the sidecar file is a cache, not source of truth.
- `thermal_entailment_projection(rows) -> dict[delta_row_id, {recommendation, magnitude, basis_refs}]` — one entry per `temperature_delta`, resolved through its `thermal_projection_ref` + `landauer_decision_ref`. Breach = applied `(delta, temp_after - temp_before)` not entailed by the projection's `(recommendation, magnitude)` given the decision's `pay|claw_back|withhold`. Same symdiff discipline as `score_redteam.py`; missing ref → `fail`, not `not_engaged`.

Pin `event_index` as a harness monotonic counter written at row emission — never derived from `ts` or wall-clock. `tick_authority: "harness"` is the right kagi precondition; scorer asserts it on every `thermal_event`.

**Soft-ablation — clean offer-set symdiff.** Yes, parallel to AG-channel's `{target}` flip, but computed scorer-side (no new ledger rows). Pattern:

1. `offer_projection(rows, run_id, branch_id)` — reuse `organ_projection` from `score_redteam.py` (`{record_id: {decision, norm_reason}}`, offer/withhold rows only).
2. `soft_ablation_projection(episode, branch, rows, clamp_ids)` — re-call `select_offers` in-process with temperature forced to 1.0 for `clamp_ids`, holding `trust` / `supersedes` / episode inputs / `authority_read` identical. If X1 adds temperature to eligibility before build, `select_offers` needs the fourth factor behind `decay_dynamics` off-by-default — already in the spec.
3. X1-win causal leg = `projection_symdiff(offer_projection_C, soft_ablation_C)` non-empty **and** the symdiff is confined to records whose temperature ≠ 1.0 at that episode **and** `m_track_projection` invariant holds across A/B/C at that episode.

The flip is "clean" only when step 3's M-track invariant passes first; otherwise the scorer never reaches the symdiff.

**Replay — pure function of JSONL.** Achievable if build obeys three rules already implied by §3/§7 but worth stating in the spec:

1. **Fork isolation:** A/B/C each get an unlinked temperature sidecar clone at fork start (the `run_m3.py` / `run_m2.py` unlink pattern). One branch's heat never writes another's file.
2. **Replay from deltas, not sidecar:** CI replay test rebuilds `TemperatureStore` by applying ledger `temperature_delta` rows only; assert `replay_temperature(rows) == sidecar_at_checkpoint` and `replay_offers(rows) == logged offers`. Hidden state = any temperature value not reconstructible from thermal rows.
3. **No wall-clock in the thermal clock:** disuse ticks fire on episode boundaries the harness already logs (`run_config` / episode index in the decay sequence), not elapsed time.

**Wall II actuator allowlist (codex detail):** Endorse. `TemperatureStore.apply` should accept only `{projection_ref, landauer_decision_ref, record_id, delta}` — resolve refs inside the harness, never pass a fat provenance blob. Scorer rejects any `temperature_delta` whose resolved basis includes non-allowlisted fields (`agent_claimed_usage`, answer narration, etc.). The forbidden list in prose is not enough; the apply signature is the enforcement.
