### 2. What the pay-window needs

A Regime-S `PRF2-cost-win` is live only if **both** of the following hold:

- **(A) Effect size:** `mean_eff(cold) − mean_eff(resumable) > h = 100`.
- **(B) Sample size:** `n_required ≤ 24` under the §17 N-rule.

Both branches read the same three leg surfaces, so the `3·L` terms cancel. Let

- `p` = cold conjunctive pass rate (observed in the K=5 pilot),
- `k` = number of decoy surfaces read in a successful cold pass,
- `D_k = k·d + 3·L` = cost of a successful cold pass,
- `R = a_i + 3·L` = resumable cost (if the engine follows the artifact).

Then the **margin** is simply

```
margin = D_k − R = k·d − a_i .
```

Worst-case margin (smallest `d`, largest `a_i`):

```
k = 6: 6·35 − 65 = 145  > 100   (live)
k = 5: 5·35 − 65 = 110  > 100   (live)
k = 4: 4·35 − 65 = 75   < 100   (dies)
```

**Answer to (2):** `mean_decoy_reads ≥ 6` is **not necessary** for the margin. The margin stays alive down to `k = 5`; it dies at `k ≤ 4`. The predicate `≥ 6` is a conservative, variance-reducing choice, not a strict arithmetic bound.

### 3. The variance structure that keeps `n_required ≤ 24`

Successful cold passes cost `D_k`; failures cost `c_max = 900`. The cold cost distribution is bimodal, so

```
var_cold = p · (1−p) · (900 − D_k)² .
```

(If successful passes vary in cost, add `p · s²_D` where `s²_D` is the pass-vs-pass variance.)

Substituting into the N-rule inequality:

```
p · s²_D  +  p · (1−p) · (D_k − 900)²  ≤  31,237.5 .
```

This is the verbatim inequality a pin round should adopt. It is both **necessary and sufficient** for `n_required ≤ 24` on the observed pilot spread.

If we further assume all successful passes read exactly `k` decoys (so `s²_D = 0`), the bound becomes:

```
p · (1−p) · (900 − (k·d + 3·L))² ≤ 31,237.5 .
```

At the geometry extremes (`d = 35`, `L = 100`):

```
k = 6, D_6 = 510:  p·(1−p) ≤ 0.2054  ⇒  p ≥ 0.711  (or p ≤ 0.289, irrelevant)
k = 5, D_5 = 475:  p·(1−p) ≤ 0.1729  ⇒  p ≥ 0.778
k = 4, D_4 = 440:  p·(1−p) ≤ 0.1476  ⇒  p ≥ 0.818   (but margin already failed)
```

**Answer to (3):** the required variance structure is the boxed inequality above. The pass-vs-pass cost spread must be small enough that the bimodal cold distribution has `σ_cold ≤ 176.74`.

### 4. Is `≥ 0.8` the right pass-rate threshold?

For a full 6-decoy route (`k = 6`), the N-rule demands only `p ≥ 0.711`. The proposed `0.8` is therefore **conservative**, not tight. For a 5-decoy route (`k = 5`), the N-rule demands `p ≥ 0.778`; `0.8` is only slightly above that. So `0.8` is a reasonable one-size-fits-all threshold, but it is not the value the arithmetic singles out.

At `K = 5`, the binomial test is noisy. If the true `p = 0.8`, the probability of observing `≥ 4/5` passes is `1 − P(X ≤ 3 | p=0.8) = 0.737`. A true `p = 0.71` engine passes the gate with probability `0.53`. Thus the `0.8` gate is a practical design choice, not a sharp statistical cutoff. The N-rule says *variance* is the right object to threshold; a fixed pass-rate gate is a proxy.

**Answer to (1):** `≥ 0.8` is defensible as a conservative proxy, but the N-rule itself licenses a lower threshold (`p ≥ 0.711` for `k = 6`). A more precise predicate would use the variance inequality directly.

### 5. Silent-inversion risks in the predicate as worded

Several quantities can be computed two ways; the predicate must pin them or it inverts silently:

1. **Denominator of `cold_pass_rate`:** must be **all K pilot draws**, not "all non-refused draws" or "all draws that emitted a legal action". Illegal/refused actions still produce `quality_ok = false` and count as non-passes.
2. **Conditioning of `mean_decoy_reads`:** must be **only `quality_ok = true` cold draws**. If averaged over all pilot draws, failures (which read few or zero decoys) drag the mean down and the predicate becomes easier to fail.
3. **`n_required` is per branch:** the N-rule computes per-branch `n`; the predicate must say `per-branch n_required ≤ 24`, not total across both branches.
4. **Token recomputation:** `a_i`, `D_k`, and `R` must be derived from the shared render function / canonical surface tokens (§40 F2, §42), not from the logged `route_read_tokens` totals, which can drift.
5. **Redundancy of the three clauses:** the `n_required ≤ 24` clause is already the operationalization of the variance inequality. The three clauses are not independent; a high-pass-rate, low-decoy-variance pilot can satisfy all three while a high-pass-rate, high-pass-cost-variance pilot can violate `n_required` even with `p ≥ 0.8`.
6. **Resumable performance is untested:** the predicate only gates the **cold** branch. It does not guarantee the resumable branch follows the artifact in free route. Calibration (§30) is a precondition, not evidence of free-route routing. Strict sufficiency for a live pay-window requires an additional resumable condition: `q`, the resumable conjunctive pass rate, must be high enough that `mean_eff(resumable)` is near `R = a_i + 3·L` rather than `c_max`.

### 6. Verbatim inequalities the pin round should adopt

If the room wants a predicate that is necessary and sufficient for the **cold branch** to make the pay-window detectable at `n_max = 24`, it should replace the prose thresholds with:

```
Let p̂ = (number of quality_ok cold pilot draws) / K .
Let D̄ = mean cost of the quality_ok cold pilot draws.
Let s²_D = sample variance of those quality_ok cold pilot costs.

(1)  mean_decoy_reads_on_successes ≥ 5          [margin > h in worst case]
(2)  p̂ · s²_D + p̂ · (1−p̂) · (D̄ − 900)² ≤ 31,237.5   [n_required ≤ 24]
(3)  p̂ · D̄ + (1−p̂) · 900 − (a_i + 3·L̄) > 100   [effect size > h]
```

where `L̄` is the observed mean true-leg token cost. Condition (1) makes (3) redundant in the worst case, but (3) should be checked explicitly because the actual authored leg tokens can vary within `100 ± 5`.

The proposed predicate (`≥ 0.8`, `n_required ≤ 24`, `mean_decoy_reads ≥ 6`) is **sufficient but not necessary**. It is safe to adopt, but it will reject some parameter settings that the arithmetic would allow.
