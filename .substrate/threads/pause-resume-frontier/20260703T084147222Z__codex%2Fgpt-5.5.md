### Blocker

**Verdict: BLOCK.** The ablation-causality content floor is still attested by the fixture, not computed by the harness, in the exact place §4c and §9 say it must be computed.

Evidence:

- `harness/check_prf_fixture.py` computes that withholding covered surfaces changes the derivation batch, but the adequacy half is `not ep.get("ablation_witness_adequate", False)`. The success text says it out loud: "fixture attests witness inadequacy under ablation." That is not a computed witness simulation.
- `harness/run_prf.py` passes the same episode field into `offer_gate` as `witness_adequate_without_obligation_surfaces`. So the offer-time mint refusal for `fixture_obligations_decorative` is controlled by an authored JSON boolean.
- The checked-in episodes all set `"ablation_witness_adequate": false`; there is no test proving a lying or stale attestation is refused by replay. The gate has a ballast lie test, but not an ablation-adequacy lie test.

This is not cosmetic. The r2 review fixed exactly this class of drift: content-floor legs are not fixture-gate decoration and not honorary-artifact demotion. If ablation causality can be set by an authored flag, then the build proves "the fixture author said the obligations matter," not "the harness computed that the obligations matter."

**Minimal fix:** remove `ablation_witness_adequate` as authority. Implement one shared replay function used by both `check_prf_fixture.py` and `run_prf.py`/`offer_gate` that actually performs the §4c-1 withheld-obligation-surfaces simulation and returns whether the witness still reaches the continuation checkpoint at adequate quality. If the current mock layer cannot compute adequacy, then the correct result is `frontier_mint_refused` or `confounded`, not `frontier_state_minted(content_floor.ablation_causal=true)`. Add a regression where the episode flag lies or is absent and the computed replay still decides the gate/mint outcome.
