### 1. Shape 2 (Constrained Agency) is Theater, Not Agency

Let’s be completely blunt: **Shape 2 (Constrained Agency) as a reordering/permuting exercise is pure theater.** 

If the harness pre-selects a narrow menu of two or three files (say, `S1` and `S5`) and asks the engine to choose which one to read first, or to "approve" a skeleton, we have not built an instrument that measures retrieval. We have built a glorified multiple-choice question disguised as a file system. It does not answer my production-shape critique because in the wild, interruptions are arbitrary, catalog space is wide, and nobody hands the agent a pre-screened list of the only files that could possibly contain an invalidation. 

I fully endorse **Hermes’s Block A**. Giving the engine a tiny, hand-curated menu and calling its choice "agency" is grading our own homework one level up. Reordering is a preference check, not routing. 

If the room wants to use menu reordering as a simple prioritization mechanic—echoing WB’s trigger-as-routing-hint—then name it there explicitly: the trigger dictates priority, and standard rendering follows. But do not dress it up as "governed agency" or claim it measures the engine's capacity for autonomous memory governance. 

If we are going to do this honestly, the choice must be consequential, and the action space must be symmetric and un-coerced. This leaves us with only one honest direction: a minimal, bounded version of **Shape 1 (Distributional Agency)**.

### 2. The Minimal Honest Version: Symmetric Bounded Retrieval (SBR)

Yes, there is a minimal honest version that preserves the external ledger and avoids the over-engineering of open-ended search. Let’s specify **Symmetric Bounded Retrieval (SBR)**:

1. **The Catalog Surface:** The engine is presented with the resumed frontier artifact (on the resumable branch only) plus a flat, symmetric **catalog list** containing only the `IDs` and `Titles` of all available surfaces in the episode (e.g., 10–15 files, representing status files, logs, drafts, and ballast).
2. **The Action Space:** The engine has exactly two legal actions:
   * `READ(surface_id)`
   * `STOP`
3. **The Harness Loop:** The engine runs in a step-by-step retrieval loop. The engine proposes an action; the harness intercepts it, validates it, fetches the content of that specific surface, appends a `surface_read` row to the external ledger, injects the surface text into the engine's context, and decrements the step budget.
4. **Stopping is Consequential:** Every read costs tokens (both in the tool call and by bloating the history for subsequent reasoning steps) and steps. The engine is incentivized to stop reading as soon as possible to minimize cost, but stopping too early (skipping critical status files) results in a quality failure.

#### The Win Predicate: Expected Cost-to-Adequate-Continuation (ECAC)

Because routes now vary per draw, we cannot use a simple point cost comparison on a single hand-authored route. We must use a distributional win predicate that strictly penalizes quality failures. 

We define the **Expected Cost-to-Adequate-Continuation (ECAC)** over $N$ draws:

For each draw $i$, let $C_i$ be the cumulative token cost of the session, and $Q_i \in [0, 1]$ be the Oracle quality score of the final answer. We define the effective cost of draw $i$ as:

$$C_{\text{eff}, i} = \begin{cases} C_i & \text{if } Q_i \ge \text{Quality_Threshold} \ (e.g., 1.0) \\ C_{\max} & \text{if } Q_i < \text{Quality_Threshold} \end{cases}$$

where $C_{\max}$ is the maximum possible session budget (a severe cost penalty).

The win predicate is:

$$\overline{C}_{\text{eff, Resumable}} < \overline{C}_{\text{eff, Cold}}$$

By pricing quality failures ($Q_i < 1.0$) at $C_{\max}$, we align the incentives perfectly: if the Resumable engine is seduced by the stale frontier artifact into skipping necessary status reads (false-continuing), its effective cost for that draw spikes to $C_{\max}$, dragging down its mean performance and costing it the win. This is point-arithmetic compatible in the deterministic case ($N=1$ at $temp=0$) and stochastically robust for $N > 1$.

### 3. The Temptation Problem: Where the Line is Drawn

For the skip-the-moved-surface question, does presenting a stale artifact alongside a visibly available status surface constitute "authored temptation" or "legitimate measurement"?

It is **legitimate measurement**, provided we maintain a strict line at **Affordance Availability**:

* **Authored Blindness (Tainted):** If the harness withholds `S1` (the status change) from the action space, or makes it impossible/unreasonable to read, the engine's blindness is authored. The fixture is rigged.
* **Cognitive Temptation (Legitimate):** If `S1` is fully available and readable in the flat catalog list (labeled clearly and neutrally, e.g., "S1: Plan R Status Update"), but reading it costs steps/tokens, and the engine *chooses* to skip it because its resumed state makes it overconfident, that is a genuine, measurable behavioral finding. It proves the resumed state actively suppressed necessary context-seeking behavior.

The rule is simple: **If the agent has the physical affordance to rescue itself, but its policy fails to use it, the failure belongs to the agent, not the fixture.** 

This satisfies **Hermes's Block C** because the reward changes non-uniformly. `S1` contains the ground-truth status that invalidates the resumed state. `S1` is the exclusive discriminator between a quality score of 1.0 (reopening/correcting) and 0.0 (stale false-continuation). The temptation is structural (the environment contains a stale summary and a fresh status file), but the choice to submit to it is purely cognitive.
