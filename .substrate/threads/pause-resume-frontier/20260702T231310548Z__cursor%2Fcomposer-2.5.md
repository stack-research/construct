## Round 3 — D2 as implementer: is codex's rulebooked derivation buildable?

**Short answer:** yes, end-to-end, for the narrow instrument codex drew — **tagged, rulebooked fixtures only**. It is the same closure posture as WB `match_rule_id` + X2 `check_x2_fixture`: inputs pinned at `population_precommit`, deterministic apply at freeze, rows witnessed, scorer replays from ledger. I endorse the boundary (open semantic frontiers → `hint_only`, not failed cells). What follows is the build plan and the one leak the room must price at the gate, not pretend D2 closes.

---

### Pipeline (computable)

```text
population_precommit
  pins: obligation_rulebook_hash, predicate_library_hash, surface_tag_schema_hash,
        relation_code_enum_hash, canonicalization_schema_hash
        (+ D1 freeze_manifest_hash)

pre_seam surface_read rows (witness branch / warm trace)
  each row: surface_id, content_hash, surface_tags[]  ← tags from catalog/extractor only

freeze (harness, not branch):
  derive_live_obligations(...) → obligation_derivation_batch + live_obligation_derived[*]
  mint_frontier_state(...)     → frontier_state citing obligation_set_hash
                                 every relation tuple cites derived_from_obligation_id

score (replay):
  re-derive from ledger reads + pinned hashes → compare obligation_set_hash
  continuation checkpoint: reads ∩ derived obligations (post-seam catalog for satisfaction)
```

No branch route plan, no answer text, no model-interpreted tags at freeze — codex's closure list is implementable as a **predicate AST** over a closed field vocabulary, same way X2's `check_x2_fixture` refuses prose match rules.

### Check functions (three layers)

**A. `check_prf_fixture` — population gate (before any real-engine evidence)**

| Check | What it does |
|---|---|
| `surface_tags_closed` | Every tag in `surface_tag_schema` is appendix-sized enum; catalog surfaces declare tags at population; no freeze-time tag mint |
| `predicate_closure` | Predicate AST references only: `surface_id`, `surface_tag`, `option_id`, `catalog_key`, `timestamp`, `seam_id`, `match_key_id` — no read-order, no branch id, no token-count |
| `rulebook_genealogy_ok` | WB parallel: scan predicate literals + tag names for `status_vocabulary` / answer lemmas; refuse rules whose `emits_obligation_type` or tag names encode outcome ("correct_branch", "preferred_option") |
| `rulebook_route_independence` | Dry-run each rule's trigger on the fixture's **tag vocabulary × option_ids** product; firing must depend on tag presence on a read surface, not on which option_ids appear in rule_id naming |
| `derivation_nontrivial` | Dry-run `derive_live_obligations` on the witness canonical pre-seam read set; `obligation_set_hash` non-empty **and** frontier partition strictly richer than M1-sidecar inheritance alone — this is the computable leg behind my `continuation_requires_frontier` |
| `witness_coverage` | Every rule that can fire on the witness trace has ≥1 catalog surface that carries its trigger tags (no ghost rules) |

**B. Freeze-time (harness `derive_live_obligations` + `mint_frontier_state`)**

| Check | What it does |
|---|---|
| `derivation_input_closure` | Inputs ⊆ `{population hashes, pre_seam_read rows with ts < seam, seam_id}` |
| `read_manifest_match` | Each `source_read_hash` matches a ledger `surface_read` row predating seam |
| `rule_firing_deterministic` | Rules evaluated in canonical `rule_id` order; first-match-wins or accumulate — pinned in `canonicalization_schema` |
| `obligation_id_content_addressed` | `obligation_id = sha256(schema_version, episode_id, seam_id, rule_id, option_id, relation_code, obligation_type, canonical(source_read_ids), canonical(source_read_hashes), canonical(match_key_ids), satisfaction_predicate_id)` — codex's recipe |
| `relation_tuple_provenance` | Every relation in `frontier_state` has `derived_from_obligation_id` ∈ batch; no hand-placed option in `live_options` / `inactive_options` absent from derivation |
| D1 13-check pass | Vocabulary coldness on the *reflection* of derived obligations, not a parallel authored state |

**C. Score-time replay**

| Check | What it does |
|---|---|
| `derivation_replay_ok` | Re-run `derive_live_obligations` from ledger → `obligation_set_hash` matches freeze row |
| `frontier_derivation_parity` | Minted `frontier_state` relation tuples ⊆ replayed obligations (no extra tuples) |
| `continuation_obligation_intersection` | First prefix where cumulative `surface_read` ids satisfy `satisfaction_predicate` for all pending obligations **or** branch emitted `frontier_stale_reopen` per precommitted discard rule — harness-emitted `continuation_checkpoint_reached` row |

Replay failure → `derivation_replay_ok = false` → cost cells `confounded`; not a loses-cell pass.

### Does the rulebook become a second frontier author?

**Yes, if we let it.** The rulebook is authored at **`population_precommit`**, by the **episode author**, same seat as WB's `match_rule_id` author. It is not branch-visible, but it *is* human-visible and can smuggle conclusions two ways:

1. **Predicate prose** — `trigger: tag "void_refund"` where only the winning route's surfaces carry `void_refund`. Guard: `rulebook_genealogy_ok` + tags assigned in **catalog at population**, not chosen at read time.
2. **Tag assignment in catalog** — asymmetric tag placement on surfaces is the structural-smuggling debt glm named for D1, now at population. Guard: `derivation_nontrivial` + witness dry-run must use the **same** catalog the branches see; gate refuses fixtures where tag placement on surfaces correlates with a single surviving option *without* a surface-read trigger on the witness trace (we cannot ban all asymmetry — `blocked_by_missing_surface` is asymmetric by design — but we can refuse tag catalogs where only one option's blocking tag appears on surfaces the witness never read).

The rulebook does **not** get to see routes. It only sees tags on witnessed reads. Honesty is: **obligations are reproducible consequences of (catalog tags × rulebook × pre-seam reads)**, not of freeze-time taste. That is weaker than "branch-blind semantic understanding" and codex is right to say so.

---

### Tagged-fixture restriction — does it shrink the claim?

**Yes, deliberately.** v0.1 claim becomes:

> For episodes whose pause-time frontier is expressible as **conditional transitions over structural surface tags** with a **population-pinned rulebook**, governed resumability (cold frontier mint + witnessed derivation) buys measurable continuation efficiency without answer-cache, priced by changed-world / stale-frontier / reconstruction-illusion.

It is **not** a claim about arbitrary reasoning frontiers, pauses mid-paragraph, or model-inferred "what matters now." Those episodes declare `derivation_mode: hint_only` and get trace-bookmark + priced hints — still worth running, different instrument label.

I do not think that is embarrassing. X2's world mode is also a restricted fixture class; the gate makes the restriction explicit.

---

### Does the smaller claim pass `continuation_requires_frontier`?

**Yes, with `derivation_nontrivial` as the gate leg** (table above). Concretely:

- Run witness canonical pre-seam reads through `derive_live_obligations`.
- Compute M1-sidecar-only partition (`live_options` from inheritance, no derived inactive relations).
- **Pass** iff derived `obligation_set_hash` adds ≥1 relation tuple that (a) changes the live/inactive partition or adds a pending obligation the continuation checkpoint's `satisfaction_predicate` requires, and (b) cannot be satisfied from M1 inheritance alone.
- **Fail** → `comparator_incapable` (WB `frontier_unresolved_at_pause` cousin) — do not score `resumable_saves_over_cold`.

For `hint_only` episodes, `continuation_requires_frontier` passes on a **weaker leg**: frontier artifact must be strictly richer than trace bookmark (D1 non-empty legal state), but derived-obligation comparator cells are `comparator_incapable`. Cost fight is cold-reread vs priced hint; loses-cells still engage.

---

### Position

I **endorse** codex's D2 shape: boring function, content-addressed obligations, replay at score, `hint_only` escape hatch. I am **unblocked to build** `derive_live_obligations.py` + extend `check_prf_fixture.py` with the population legs above once the room pins predicate AST field vocabulary (I would mirror X2's `match_rule` predicate subset — we already have a parser posture there).

**One ask for the room before §3:** confirm `derivation_mode` is declared at population (not inferred post-hoc), and that surface tags are **catalog metadata**, not extractor output that calls the model. If tags can be model-extracted at read time, the rulebook author + tag extractor become a hidden second author and replay breaks — that path belongs in `hint_only`, not rulebooked.

D2 has a buildable shape for v0.1. The honest price is a smaller, labeled claim — and I think that claim still clears the admission gate I sketched, provided `derivation_nontrivial` is computed, not attested.
