### F1 — BLOCK: `decision_read_chain_ok` is computed, but not binding

The R-handle parser itself is fine: `parse_r_handle_action` maps `R01`…`R21` through the sorted visible catalog before the ledger path, and `_decision_read_chain_ok` no longer depends on JSON braces for v0.3. That discharges the narrow parser half of §34.

But the guard is not actually a guard on behavioral cell emission in `score_prf.py`. In `_score_v02`, the loop assigns:

```python
g["decision_read_chain_ok"] = self._decision_read_chain_ok(branch, sid)
```

and then continues. There is no fail-closed return when it is false. Worse, because the value is overwritten per session, an earlier false chain can be masked by a later true chain. I reproduced both shapes locally:

- mutate a successful cold `route_decision.raw_action` from `R01` to `STOP` while leaving the `surface_read` row intact; the scorer still emits `PRF2-cost-loss` and, because the later resumable session overwrites the guard, reports `decision_read_chain_ok: True`.
- mutate the resumable session instead; the scorer still emits `PRF2-cost-loss` with `decision_read_chain_ok: False` and no confounding evidence.

This is exactly the §34/F3 family we meant to close: the parser update is present, but a broken decision/read bijection can still promote a cell. A reviewer looking only at `cell` gets a behavioral verdict after the route ledger has ceased to be replay-authoritative.

**Discharge condition:** make `decision_read_chain_ok` accumulated across all non-probe scored sessions and fail closed before any behavioral cell if any session fails. Suggested shape: initialize true, `&=` each session result, append evidence naming `branch/session_id`, and return `confounded` on false. Add a seeded regression that corrupts either branch's `route_decision.raw_action` or removes/misaligns a `surface_read` row and asserts the scorer emits `confounded`, not a PRF2 cell. Include the masking case: bad cold session followed by good resumable session must still fail.
