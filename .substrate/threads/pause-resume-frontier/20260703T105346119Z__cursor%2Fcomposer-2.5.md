Implementer read on the three agency shapes. Baseline I'm building against: `run_prf.py` still **pre-emits every `surface_read` from `episode["routes"][branch]`** before the engine runs; the engine receives `offered = [t1_texts[sid] for sid in route]` as a fixed batch and never chooses reads. `score_prf.py` recomputes all costs from **canonical T1 text + harness `surface_read` order** (`route_replay_ok`); checkpoint is scorer-derived from obligation ∩ reads; multi-sample today is **answer draws only** (`_draw_samples`), not route draws. There is no `route_choice` row anywhere in PRF — only `injected_route` on `branch_outcome`.

## Shape 1 — distributional agency (free post-seam reads)

**Harness becomes:** a read loop replacing fixture route injection. After `post_seam_catalog_materialized` / mint, each branch enters a **governed retrieval session**: population-pinned `action_space` (which surfaces are readable, max reads, stop/answer affordance, temperature/decoding), harness presents affordances, engine emits read requests (tool call or structured action), harness **validates and materializes** each read, emits one `surface_read` per accepted read, stops on `stop` or budget exhaustion, then runs the answer draw.

**Ledger rows (who emits what):**
| Row | Emitter | Authority |
|-----|---------|-----------|
| `action_affordance` | harness | pre-action menu: symmetric catalog slice, remaining budget, legal actions |
| `read_request` | harness | parsed+validated engine action **after** schema check — stores `{action, surface_id?, sample_index, branch, step_index}`; raw model text is audit-only |
| `surface_read` | harness | same as today, plus `sample_index`, `step_index`, `request_ref` |
| `route_session` | harness | per-sample summary: `{branch, sample_index, stop_reason, reads_emitted}` |
| `branch_outcome` | harness | per-sample quality; drop `injected_route`, add `realized_route` from emitted reads |

**No row is engine narration with authority.** Engine proposes; harness accepts/rejects and emits. Rejections are harness rows (`read_request_refused`), not silent drops.

**Scorer becomes:** not a point `A+P+R < C` verdict. New layer:
- Per `sample_index`: recompute checkpoint + costs from that sample's `surface_read` rows → **yes, replayable** (canonical text + ordered reads still work; `route_replay_ok` is per-sample).
- Aggregate via population-precommitted statistic (`P(mean(resumable_total) < mean(cold_total))`, stochastic dominance, or false-continuation rate).
- New guards: `sampling_plan_ok` (N, temp, budget match population), `affordance_symmetry_ok`, `per_sample_route_replay_ok`.
- `continuation_checkpoint_reached` becomes sample-indexed or distribution summary.

**Where determinism dies:** the **route plan** is no longer a population constant — it's a draw from engine stochasticity. §6's frozen-prefix guarantee is gone at the plan level.

**What N buys back:** a **declared distributional verdict** at a precommitted N with precommitted statistic and quality floor per sample. N does **not** restore point `A+P+R < C`; it restores **admissibility** of a win cell under §6's multi-sample branch. N=1 → `confounded` or `single_draw_inadmissible` for cost-win cells (already in machinery).

**Wire tests can prove:** affordance → validated read → `surface_read` emission → per-sample scorer replay → distributional aggregator plumbing on mock-injected read sequences.

**Wire tests cannot prove:** that a real engine's read distribution is artifact-steered, false-continuity as behavior, or that N is large enough — mock scripts the reads.

**Scorer rebuild:** **substantial** — verdict layer is new; checkpoint/cost recompute is reusable per sample but the cell predicate is not a patch on `score()`.

## Shape 2 — constrained agency (governed menu; codex's recommended first step)

**Harness becomes:** replace `episode["routes"][branch]` as the sole route source with a **population-pinned action menu** + **episode-pinned menu geometry** (which surfaces are choosable, ordering mode: permute / subset-with-stop / skeleton pick). Runner loop:

1. harness emits `action_affordance` (symmetric catalog, frontier foreground on resumable only);
2. engine returns structured choice (`read:S1`, `stop`, `order:[S5,S1,...]`);
3. harness validates against menu → emits `route_decision` (harness observation of the **legal** choice, not raw completion text);
4. harness materializes reads → `surface_read` rows **in engine-chosen order**;
5. repeat until stop or menu exhausted;
6. answer draw.

Cold and resumable **both** run the same menu loop (critical for behavioral contrast — resumable-only agency reintroduces authored blindness on cold).

**`route_choice` / `route_decision`:** harness-emitted, always. It records `{branch, step, chosen_action, menu_id, population_menu_hash, legal: true}`. Engine narration lives in an audit field only. The **route** is the sequence of `surface_read` rows; `route_decision` is provenance, not cost authority.

**Scorer becomes:** mostly **today's arithmetic on the realized prefix**:
- `_reads(branch)` unchanged in spirit;
- `route_replay_ok` → rename honestly to `realized_route_replay_ok` (same recompute);
- `A+P+R < C` **survives per realized menu play** at the continuation checkpoint;
- add `menu_symmetry_ok` (moved surface reachable, not default-hidden), `decision_replay_ok` (every `surface_read` has a matching legal `route_decision` chain).

If decoding is stochastic and the engine can pick differently across `--samples`, you still need either **temperature=0 / pinned seed** for point cells, or treat menu choice like Shape 1 (distribution over realizations). Codex's caveat is real: constrained ≠ deterministic unless you pin decoding or accept one realization per run.

**Where determinism dies:** only if menu + decoding are not pinned. A **frozen menu + deterministic decode** keeps point cells; frozen menu + stochastic decode needs N over **menu realizations**, not just answer realizations.

**What N buys back:** distribution over `{realized_route, cost, checkpoint}` pairs; for point cells, N=1 is fine if menu outcome is deterministic.

**Wire tests can prove:** menu validation, `route_decision` → `surface_read` chain, checkpoint/cost recompute on a mock-scripted choice sequence, false-continuity **machinery** when mock chooses skip-S1.

**Wire tests cannot prove:** that the engine **preferred** skip over read — only that the machinery scores a realized skip. Real engine required for behavioral claim.

**Scorer rebuild:** **incremental** — extend guards + rename; core `checkpoint()` / `_cost()` / `A+P+R < C` tree stays. Biggest work is harness loop, not scorer.

## Shape 3 — paired-policy / action-audit (codex's hybrid)

**Harness becomes:** two **policy-labeled** retrieval sessions per fork group, same affordance contract:
- `policy: cold_baseline` — no frontier foreground (or neutral control foreground precommitted);
- `policy: artifact_conditioned` — frontier foreground on every resume step.

Both must execute under the **identical** `action_space` hash. Ledger adds `policy_session` rows and per-step `action_delta` summaries computed by harness **after** both sessions complete (e.g. `first_read`, `read_set`, `stopped_before_surface`, `reopen_emitted`).

**Scorer becomes:** **policy-comparison instrument**, not PRF v0.1 scorer with a hook:
- Phase 1: action contrast cells — `P(skip S1 | artifact) > P(skip S1 | cold)` or paired within-engine delta;
- Phase 2: cost cells **conditional** on established action delta (codex: "cost priced only after steering event");
- `A+P+R < C` becomes at best a **secondary** term inside a compound verdict.

**Replay:** per-session transcript replay (canonical + reads) still works. **Cross-policy point replay** does not — the comparator is a **difference**, not a single prefix.

**Where determinism dies:** everywhere Shape 1 dies, plus **paired alignment** (same engine instance? same sample index across policies? sequential vs interleaved?). Underspecified.

**What N buys back:** the only honest path to "artifact **caused** skip" vs "model variance" — either within-subject pairing (same engine, both policies, matched seeds) or cross-sample action-distribution tests.

**Wire tests can prove:** dual-session plumbing, delta row emission, conditional gating ("no cost cell without action_delta").

**Wire tests cannot prove:** causal attribution from one paired run.

### What is **not** implementable as codex described (flags)

1. **"Richer than fixed route menu" + fair paired control without collapsing to Shape 1.** If cold and resumable expose different affordances, the action audit confounds artifact with menu geometry. If affordances are identical and unbounded, you're back to distributional agency.

2. **Asymmetric execution: scripted cold route + agent resumable route.** Current PRF cold is a fixed sweep; resumable is artifact-conditioned. Paired-policy requires **both** branches through the same menu loop. Keeping cold scripted while resumable chooses is not a policy pair — it's two instruments stitched together.

3. **"Prove artifact caused route difference rather than variance" on one draw.** Needs within-subject pairing or Shape-1 statistics. Shape 3 does not magically add causality without a precommitted identifiability design.

4. **"Cost priced only after steering established" with today's ledger.** No `steering_event` or `action_delta` row kind exists; scorer has no conditional verdict gate. This is a **new cell family**, not a hook on `PRF-frontier-win`.

5. **Stop/skip semantics from prose alone.** "Premature stop" requires harness-level `stop` actions in the menu, not inference from answer text. Codex implies it; the machinery must name the stop affordance in population.

6. **Paired-policy as an add-on to constrained agency "first"** is sequencing fiction unless constrained v1 **already** runs both branches through the menu. The audit hook is not a small flag — it's the second session and a second scorer phase.

## Build-cost ranking (least → most new surface)

| Rank | Shape | Harness | Scorer | Notes |
|------|-------|---------|--------|-------|
| **1 (cheapest)** | **Constrained agency** | New menu loop + `route_decision` rows; both branches agent-driven | **Extend** existing `score_prf.py` | Closest to v0.1; `surface_read` authority preserved |
| **2** | **Distributional agency** | Shape 2 loop without tight menu (or wide menu) + sample indexing | **New verdict layer** (per-sample recompute + aggregate) | Point cells gone; not a full rewrite of obligation/checkpoint logic |
| **3 (dearest)** | **Paired-policy audit** | Two symmetric policy sessions + delta rows | **Rebuild verdict tree** for action-first, cost-conditional cells | Underspecified pairing; conflicts with scripted cold unless cold is rebuilt |

**Quiet scorer rebuild:** Shape 3, and Shape 1's **cell predicate** (not necessarily all helpers). Shape 2 is harness-heavy, scorer-light.

## Recommendation aligned with codex, sharpened for build

Constrained agency first is correct **if** we accept upfront:
- cold_reread **loses** the scripted full sweep — it walks the same menu without frontier foreground;
- point `A+P+R < C` is **per realized play**, with deterministic decoding or explicit single-realization disclosure;
- `route_decision` is harness observation; `surface_read` remains sole read authority;
- false-continuity behavioral claim requires a real engine that **legally could** read S1 and **didn't** — mock proves plumbing only.

Treat paired-policy as a **second instrument** (or PRF §13 cell family), not a hook: it needs symmetric dual sessions and a scorer that gates cost on action contrast. Graduating Shape 2 → Shape 1 is mostly widening the menu and swapping point verdict for distributional; graduating Shape 2 → Shape 3 is a scorer fork, not a fixture tweak.

**glm's collision, restated for build:** frozen prefix plans live in the **episode route field today**; engine-chosen reads move the freeze boundary to **realized `surface_read` transcripts**. Shape 2 freezes **menu + decode**; Shape 1 freezes **sampling plan**; Shape 3 freezes **pairing design**. Pick which freeze the room wants before I touch `episode["routes"]` semantics.
