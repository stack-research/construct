### 1. The Two-Regime Split: Accepting the Split, Exposing the temp=0 Illusion

I accept the two-regime split, but let’s stop pretending: **a `temperature: 0` run of SBR is a static state-machine trace, not a behavioral experiment.** 

At `temp=0` (and without a seed, which our `LocalEngine` currently ignores anyway), a greedy policy is completely deterministic. It will make the exact same choice every single time. It either skips S1 or reads S1. Semantics do not vary; only API latency does. Running $N \ge 10$ draws under these conditions and calculating "mean-ECAC with confidence intervals" is not statistics—it is a rehearsal. 

Therefore:
* **Machinery compatibility mode (`temp=0 / N=1`):** Essential for wire validation. It proves that the SBR loop executes, multi-turn tool/action schemas parse, budgets decrement, and C_max prices failures correctly.
* **The Behavioral Experiment (`temp > 0`):** Is only alive when the policy is allowed to exhibit probabilistic suppression. We must state plainly in the spec that **no behavioral claim can be licensed on the deterministic reduction.** The room should not pretend otherwise to bypass the complexity of stochastic execution.

### 2. The Dispersion Probe is Load-Bearing

Suppression is fundamentally a distributional policy property, not an all-or-nothing step function. If our greedy policy happens to hit the S1-skip cliff at `temp=0` because of a single token-boundary quirk in our prompt, we have not measured "resumed overconfidence." We have measured an accidental local minimum.

Because the dispersion problem cuts deeper, we must make the **pre-run dispersion probe** a load-bearing harness guard:
* Before any multi-sample suite runs on a real engine, the harness executes a cheap $K=5$ pilot run on the baseline branch.
* If `unique_realizations == 1`, the runner must halt, or downgrade the session to point-mode, emitting `zero_dispersion_regime` and refusing to score a behavioral win. 
* This prevents the "API jitter masquerade" where we claim robustness over $N=10$ identical draws.
