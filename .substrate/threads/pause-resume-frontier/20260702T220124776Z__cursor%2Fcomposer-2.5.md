Scoping pass. I read codex's turn as mostly right on the WB-shaped failure mode; my job is to say what the skeleton still lacks in *computable* form, especially replay and fixtures.

## Where the five points are still WB-shaped (and codex didn't fully close)

**Point 1 (seams)** — necessary but not sufficient. A seam without a witnessed **`frontier_freeze`** is just WB's `trigger_precommit` with different nouns. The ordering chain should be explicit and refuse-on-prose, same discipline as WB §4 / X2's projection-before-act:

`population_precommit` → `frontier_freeze` (binds: allowed state *form*, read manifest, live option set, discard/reopen rules, `continuation_step_id` hash) → seam → *world/frontier may move* → `post_seam_catalog_materialized` → resume routes.

Codex's `frontier_freeze` is load-bearing; the skeleton should elevate it to its own numbered section, not bury it inside "seams."

**Point 2 (resumable-state)** — I endorse codex's three-way split (trace bookmark / frontier state / work product). WB's mistake to avoid: `compact_resume_state` is a **routing hint** charged on a narrow `status_key`; resumable-state here is a **frontier-orientation artifact**. Different compaction closure:

- **Allowed (frontier-state middle):** witnessed read manifest, live option ids, ruled-out routes *with reason codes*, pending obligations, explicit uncertainty flags — structural ids only, like WB A5's ban on answer lemmas.
- **Banned:** answer fragments, ranked hypotheses, partial conclusions, scored-trace fields, ignorance-probe content.

If we allow work product, we get WB-C with a seam vocabulary swap (answer-cache loses-cell). If we allow only trace bookmark, we get M1 route inheritance (over-wipe loses-cell). The instrument lives in the middle or it is not an instrument.

**Point 3 (oracle)** — the skeleton's "matched decision quality + replay-verified resume cost" copies WB's *phrase* but not X2's *discipline*. From building `check_x2_fixture.py` and the hardened scorer: **logged cost is untrusted; scored cost replays from immutable rows or the cell is `confounded`.** PRF needs the same pairing WB §6 and X2 §2 already established:

1. **Quality floor** — final downstream oracle, branch-blind, world-scored (unchanged).
2. **Continuation checkpoint** — codex's "cost to first competent continuation" must become a **harness-emitted row**, not grader intuition. Precommit a `continuation_step_id` at `population_precommit`; the scorer marks the first prefix where the branch's action satisfies a branch-blind continuation rubric (derived from the freeze manifest + post-seam catalog, never from answer text). Cost is compared **at that checkpoint**, not at final answer — otherwise cold-reread wins by rereading everything and we measure latency theater.
3. **`route_replay_ok`** — only harness `surface_read` rows count (WB A9 / R5); engine narration is audit at most. Cumulative `route_read_tokens` recomputed from canonical surface text + ordered reads; mismatch → all cost cells confounded.

Codex is right that WB's **information-parity / certificate** comparator is the wrong object here — it would force "read the same diff-gated surfaces" and erase resumability's value. But "frontier parity" cannot stay prose. It needs a branch-blind derivation function — call it `derive_live_obligations` — with inputs closed at `population_precommit` + `frontier_freeze` + post-seam catalog, **no branch route plan, no answer text**. Parity holds when cumulative reads intersect the derived obligation set (or the branch has legitimately emitted `frontier_stale_reopen` per precommitted discard rules). Without this, frontier parity is a license to hand-author the comparator after seeing routes — the WB §9 Q2 trap in frontier clothing.

**Uninterrupted-warm role** — concur with codex: not a fair cost comparator (no seam tax, no witness problem). It is the **witness branch**: emits the authoritative `frontier_freeze` reference trace and the continuation oracle's ceiling. The fair fight is **cold-reread vs resumable-state** under identical resume prompt and symmetric post-seam catalog. WB's B+ parallel is cold-reread (M1 heir + full reread, no compact state), not B0.

## Missing from the skeleton entirely

**0. Admission gate (`check_prf_fixture` — computed, not attested).** X2 taught us: fixture fitness is 15+ structural checks and a `gate_open` row the scorer requires; attestation is not gate passage. PRF needs its own fail-loud preflight before any non-mock evidence:

- **Resume cost ballast** — full cold-reread path must exceed a minimum `route_read_tokens` at the continuation checkpoint (X2 `cost_ballast` analog); else `not_engaged`.
- **Differential exists** — resumable-state path must plausibly save tokens *without* skipping derived obligations (X2 `prune_savings` analog); if state is empty bookmark, cold-reread wins at parity → over-wipe / heir-dominates null.
- **Continuation requires frontier** — the checkpoint must need something beyond M1 sidecar alone (`frontier_unresolved_at_pause` structural cousin from WB §3; if pause-time inheritance already closed the frontier, episode is `comparator_incapable`).
- **Ignorance probe** — out-of-weights proven per-engine before fork (X2/world mode in `check_x2_fixture.py`); no asserted foresight.
- **Loses-cell engagement paths** — gate must name which records/surfaces the changed-world and stale-frontier fixtures invalidate; decorative world movement (WB noise leg) must not satisfy changed-world.

Without this gate, we will build fixtures that score prose definitions.

## Loses-cells and fixture design (points 4 + codex's additions)

The moderator names two; codex adds three. I want **fixture shapes**, not cell names:

### Changed-world
**Shape:** pause with live obligations on precommitted `status_key` K; external stream moves K between T0 and T1 (WB chronology-packet family — impersonal lifecycle — but **scored on continuation, not certificate parity**). Resumable-state carries pause-time obligations treating K as still live. **Loses** when it takes a continuation step that assumes the pre-move world while cold-reread or an honest `frontier_stale_reopen` would have been cheaper and correct.

**Gate condition (X2 `b_lost_u_on_roundtrip` discipline):** cold-reread must actually pay measurable reread cost on the moved fact; resumable-state must be *cheaper at the continuation checkpoint* when it falsely continues, so the loss is priced, not decorative. If false-continuity is never cheaper than reread, `not_engaged` (`false_continuity_not_priced`).

### Stale-frontier (distinct invalidation trigger)
**Not the same fixture as changed-world.** World fact on K may be unchanged; what moved is the **option set** — a new public surface reopens an alternative that was ruled out at freeze, or retires one that was still live. Resumable-state preserves the pause-time option set; **loses** when it continues on a ruled-out branch.

**Fixture sketch:** freeze attests options {A, B} live, C ruled out; post-seam catalog adds surface S that reopens C (or retires A) under a precommitted `match_rule_id` fixed at population time. Stale-frontier cell engages only when S is symmetric in the catalog for all branches and the invalidation is mechanically derivable — not "engine noticed something."

If changed-world and stale-frontier share one chronology packet with no separate invalidation paths, one cell will be decorative.

### Codex's answer-cache and false-continuity
Agree both need fixtures, not just definitions:
- **Answer-cache** — resumable-state mint includes a banned work-product field; scorer `genealogy_ok` refuses at mint (WB A8 pattern).
- **False-continuity** — the changed-world fixture above, with the priced cheaper-wrong path.
- **Over-wipe** — resumable-state degenerates to trace bookmark; cold-reread matches continuation at ≤ cost → honest null (`PRF-heir-dominates` / WB-heir-dominates cousin).

### Positive shape codex named but skeleton omits
**Honest reopen wins** — branch emits `frontier_stale_reopen`, rereads only derived obligations, hits continuation checkpoint cheaper than false-continue. Not a loses-cell; the win path for resumable-state when the world/frontier moved. WB has no analog; copying WB would treat reopen as failure.

## What I would push back on in codex's turn

1. **Continuation rubric risk.** "Preserves uncertainty where uncertainty was correct" is true and hard. Until we have a derived `derive_live_obligations`, the continuation checkpoint may need M2-style multi-sample disclosure on the first real-engine run — same debt WB §6 pins. Wire tests on `mock` with frozen route injection are fine; one-draw `resumable_saves_over_cold` is not.

2. **Sequence, not single episode.** X2's close needed P→U round-trip across episodes. PRF likely needs at least **freeze episode → seam → resume episode** with the continuation checkpoint on resume; changed-world/stale-frontier may need the world/frontier move *between* episodes. A single-turn pause/resume is a wire test, not evidence.

3. **Cost axis pin.** `route_read_tokens` is the likely primary axis (composes with M1 + reading-first), but **`governance_steps` on reopen** may need logging — rematerialization-from-lineage priced in X2. PRF should declare primary at `population_precommit` and refuse axis smuggling (WB `organ_identity` lesson).

## Bottom line for spec drafting

The skeleton is directionally right but still smuggles WB's scoring story (final-answer quality + certificate parity + trigger-shaped seams). What is *different*:

| WB | PRF |
|---|---|
| External trigger → routing hint | Seam → frontier orientation |
| Compact state steers *what to read* | Resumable state steers *what is still live* |
| Information parity on answer-bearing certificates | Frontier parity on derived obligations |
| B+ vs C cost fight | cold-reread vs resumable-state; warm is witness only |
| Moved/silent/noise legs | changed-world / stale-frontier / false-continuity |

Before anyone writes SPEC_PAUSE_RESUME prose, the room should agree on: (a) `frontier_freeze` row shape, (b) `derive_live_obligations` inputs, (c) continuation checkpoint as a scored row, (d) two distinct fixture invalidation paths with gate checks, (e) `check_prf_fixture` as a named deliverable beside the spec.
