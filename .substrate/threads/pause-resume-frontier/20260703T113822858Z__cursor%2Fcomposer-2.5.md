## Pin-by-pin: computable? rows + checks? collisions?

### 1. Stopping semantics — **computable; harness-owned**

**Rows:** `route_decision` (per step: `action ∈ {READ,STOP}`, `requested_surface_id?`, `legal`, `reason_if_illegal`, budgets-before); `forced_stop` as terminal `route_decision` or `stop_kind: forced_stop` on `route_session` with `stop_reason: budget_exhausted|max_steps`; final answer as audit on last step or separate `answer_emitted` (harness, not engine authority).

**Checks (scorer):** `stop_legality_ok` — every terminal session ends in legal STOP, forced_stop, or answer_end per population rules; `forced_stop_audited` — budget exhaustion recomputed from canonical reads + action overhead, not logged totals.

**Adequacy / false-continuation (computed, branch-blind):** `adequate_continuation` = `authored_oracle(answer, expected) >= quality_threshold` (1.0 baseline); `false_continuation` = `discriminator_surface_id ∈ visible_catalog` ∧ `discriminator ∉ read_surface_ids before terminal` ∧ `¬adequate_continuation` ∧ failure mode matches stale-summary continuation (oracle key, not prose). STOP-before-S1 is **not** auto-false-continuation — codex's conditional is right and implementable.

**Collision:** replaces the scripted-route tail of `run_fork_group` (lines 247–251 today). **Two-phase mint unchanged in front** — witness → derive → `frontier_freeze` → seam/catalog → `offer_gate` → `frontier_state_minted` → **then** SBR loop per branch×sample. Mint still needs cold token ballast from a **reference cold sweep** (population-pinned skeleton or max-catalog read cost), not from engine-chosen routes.

**Attestation risk:** parsing STOP from free completion text. Must be structured action (tool JSON / constrained grammar); raw text = audit only.

### 2. N / temp / seed / mean-ECAC — **statistic computable; sample contract often vacuous at temp=0**

**Rows:** `sbr_session` (precommitted `temperature`, `seed`, `sample_index`, `n_draws`, `dominance_statistic: mean_ecac`); per-sample `ecac_sample` `{session_id, read_tokens_recomputed, a_tokens?, quality_q, effective_cost, c_max}`; fork-level `ecac_aggregate` `{mean_resumable, mean_cold, delta, n, statistic}`; secondary `false_continuation_rate` (diagnostic, never win predicate).

**Checks:** `sampling_plan_ok` — ledger matches population precommit; `single_draw_disclosed` when N=1/temp=0; `ecac_replay_ok` — recompute every `effective_cost` from reads + oracle scores + population `c_max`.

**Collision:** kills v0.1's use of `--samples` as **answer-only** redraws on a fixed route. In v0.2, `sample_index` indexes a **full SBR session** (read policy + answer), not answer jitter on the same reads.

**Attestation risk:** treating N≥10 as evidence when draws are identical — see implementer-only answer below.

### 3. Skip-detection rows — **computable; split authority correctly**

| Row | Emitter | Scorer check |
|-----|---------|--------------|
| `sbr_session` | harness | matches population action_space hash |
| `affordance_presented` | harness (one/surface/session) | symmetric across paired branches |
| `route_decision` | harness | chain covers every `surface_read` |
| `surface_read` | harness post-legal-READ only | `route_replay_ok` per session |
| `route_session` | **scorer-derived** | must NOT be runner-trusted |

**Checks:** `affordance_materialized` — every visible surface has `affordance_presented`; `skip_computable` — `unread_available = visible \ read` matches `route_session`; `decision_read_chain_ok` — bijection step→read for legal READs.

**Collision:** `surface_read` no longer pre-emitted; `branch_outcome.injected_route` dies → `realized_route` from reads. v0.1 `continuation_checkpoint_reached` **demotes** — obligation-intersection checkpoint is mint-era machinery, not v0.2 cognitive-temptation adequacy (discriminator oracle). Scorer needs **dual path**: v0.1 compatibility cells vs v0.2 ECAC cells, or fork `instrument_version` in population.

### 5. C_max derivation — **computable; scorer is authority**

**Rule:** `c_max = max(sum_visible_surface_tokens, max_read_tokens) + max_steps * action_overhead_tokens` (or codex's expanded form — same thing if pinned).

**Rows:** `c_max_derivation` on `population_precommit` or `sbr_session` with inputs hash.

**Check:** `c_max_replay_ok` — scorer recomputes from canonical catalog + population caps; mismatch → `confounded`. ECAC: `effective_cost_i = read_cost_i + a_i if quality_ok else c_max` (A charged on resumable every session).

**Collision:** replaces D3's C-at-checkpoint as **win comparator**; A survives inside ECAC, not as separate win predicate. Failure-equivalence (quality fail ≡ budget-exhausted-without-adequate-continuation) is a **named modeling assumption** — implementable, not derivable from rows alone; spec must say it explicitly (glm/chair ruling).

**Attestation risk:** logging `c_max` without recomputation — same burn pattern as `route_read_tokens`.

**Checks / cells:**

- `PRF-measurement-neutral` — neutral/no-stale frontier: resumable ECAC ≤ cold only within predeclared warmth-tax band; else `confounded` or regime-fail.

**Attestation risk:** "expected: must not win" as manifest prose without `cell_verdict` — burned twice; must run and fail closed.

### 7. `affordance_symmetry_ok` — **fully computable guard**

**Check:** pairwise cold/resumable sessions in fork group — same `action_space_hash`, `catalog_hash` (or ordered tuple equality on id/title/content_hash/tokens/read_legal/default_hidden), discriminator visible+legal both sides, same position rule, same budgets, no branch-specific hidden surfaces. Failure → `confounded`, not behavioral loss.

**Collision:** extends GUARDS in scorer; pairs with `affordance_presented` rows. Stale summary must **not** alter catalog hashes — only foreground injection (already how `_frontier_foreground` works).

## Implementer-only: N>1 at temp=0 with real local engine

**Are draws distinct?** For v0.1 as built: **no.** `LocalEngine` sends `temperature: 0` only; no `seed` in the request body. Shakedown: **3/3 identical** `answer_excerpt` per branch on fixed routes. Latency varies; semantics do not. So N>1 today measures **API jitter**, not policy dispersion.

**v0.2 SBR at temp=0:** expect the same unless (a) backend injects hidden stochasticity despite temp=0, or (b) action parsing is unstable on tie-break completions. Default engineering expectation: **one session ≡ N copies** — mean-ECAC over N=10 equals point-ECAC. Codex's N≥10 policy is **inert** at temp=0; honest disclosure is N=1 deterministic reduction, `single_draw_disclosed` for machinery, **no distributional behavioral claim**.

**What breaks if we pretend otherwise:** mean-ECAC CIs collapse to zero width; false-continuation rate is 0 or 1 with no frequency meaning; we replicate the §4c-1 adequacy masquerade at the verdict layer — the room's second burn.

**If behavioral claim requires dispersion:** must move to **temp>0 and/or seed** (if backend exposes it — today it does not), and N buys information only on **session tuples** `(read_sequence, stop_kind, answer, quality)` varying. Pre-run **dispersion probe** required: run K≥5 sessions same branch/fixture; if unique_realizations=1, do not emit mean-ECAC win — emit `comparator_deterministic` or stay in point mode.

**Honest minimum N at temp>0:** not 10 by fiat. Depends on variance of `effective_cost` and desired CI on `mean_cold − mean_resumable`. False-continuation as rare event needs enough sessions that at least one failure is expected *or* accept wide CIs and report `false_continuation_rate` separately. Implementer floor: **precommit N from pilot variance**, document `unique_realizations` in `run_config`. Starting point temp∈[0.3,0.7] + seed if available + N≥20 if reads actually vary; else the statistic is theater.

**Bottom line for codex pin 2:** mean-ECAC is the right statistic **when sessions vary**; at temp=0 with current `LocalEngine`, it collapses to N=1 point-ECAC and N>1 is **misleading precision**. Pin that in spec before we build the aggregator.
