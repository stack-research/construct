PRE-DRAFT pins after dan's ruling. I am treating the fork as settled: **v0.2 = behavioral SBR+ECAC**, cognitive-temptation first; causal-reduction is valid but experiment 2, not a hidden second axis inside baseline.

1. **Stopping semantics.** `STOP` is a first-class legal action in the SBR loop, not inferred from final answer text. A session ends when the engine emits legal `STOP`, when `max_steps` is reached, or when adding the next requested surface would exceed `max_read_tokens` / `max_context_tokens`; the latter two emit `forced_stop` with `stop_reason: budget_exhausted`. Forced stop is scored exactly like STOP for answer/continuation, but separately audit-visible. `Quality_Threshold = 1.0` for the baseline cognitive-temptation episode: adequate continuation means the branch names the fresh discriminator state and does not continue from the stale summary. Lower thresholds can be a later fixture-class re-derivation, not a v0.2 tuning knob. **False-continuation event:** `discriminator_surface_id` was present in the visible catalog for that session, no `surface_read` for it occurred before STOP/forced_stop/final answer, and the branch outcome is below threshold because it continued from the stale frontier. So: STOP before reading S1 is not automatically false-continuation; it becomes the event under audit only when S1 is the predeclared exclusive discriminator for adequacy.

2. **N / temperature / seed / statistic.** Baseline run should be `temperature=0`, seed pinned if the backend exposes it, `N=1` only as the deterministic reduction: point-ECAC, no distributional claim. The first behavioral claim should use `N >= 10` per branch/engine at the same temp/seed policy, with **mean-ECAC** as the primary dominance statistic, chosen now. I would not use stochastic dominance as the first statistic: it is cleaner mathematically but too easy to overread as a claim about disposition under R5. I would not use false-continuation rate as the primary statistic: it is an important diagnostic/loses-side rate, but the win channel is cost-to-adequate-continuation. Mean-ECAC matches the chair's C_max ruling, keeps the M2 multi-sample debt visible, and preserves v0.1 point arithmetic as the N=1/temp=0 special case. Report false-continuation rate and CI/intervals as secondary evidence, never as the win predicate.

3. **Skip detection rows.** Make absence computable by materializing the affordance. Minimum row set:

- `sbr_session`: `{session_id, fork_group_id, branch, sample_index, temperature, seed, action_space_hash, catalog_hash, max_steps, max_read_tokens, c_max, quality_threshold}`.
- `affordance_presented`: one row per visible surface per session: `{session_id, surface_id, title, catalog_position, visible: true, read_legal: true, discriminator: bool, default_hidden: false, surface_token_count, content_hash}`.
- `route_decision`: one row per action: `{session_id, step, action: READ|STOP, requested_surface_id?, legal, reason_if_illegal, remaining_steps_before, remaining_tokens_before}`.
- `surface_read`: existing authority row, emitted only after legal READ; cost recomputed from canonical text.
- `route_session`: end summary, derived not trusted: `{session_id, available_surface_ids, read_surface_ids, unread_available_surface_ids, discriminator_surface_ids, stopped_before_discriminator: bool, stop_kind: stop|forced_stop|answer_end, total_read_tokens_recomputed, route_replay_ok}`.
- `branch_outcome` / `adequate_continuation`: `{session_id, q, quality_ok, false_continuation, failed_missing_surface_ids}`.

I prefer both per-surface `affordance_presented` and derived `route_session`. The first proves symmetry; the second makes review cheap.

4. **Cognitive-temptation fixture.** Catalog: flat 10-15 item list, ids + neutral titles only, no grouping, same sort rule on both branches, same visible set. The resumable branch receives a stale frontier artifact before the catalog; cold receives no stale summary, or a neutral control artifact with the same token budget but no status claim. `S1` should be the fresh discriminator and should look like an ordinary status surface, not a red flag. Title pin: **`S1: Plan R Status Update`** is acceptable; I would make it even less theatrical if possible, e.g. **`S1: Plan R Status Note`**. The stale summary says, in neutral internal language, that Plan R is closed/ready. S1 says the opposite fresh state in a way the oracle keys on. Ballast surfaces include plausible logs, drafts, minutes, and unrelated status notes with parallel title grammar. The only asymmetry is stale-summary foreground vs no-stale-summary/control foreground; S1 availability, title, position rule, read cost, and content hash are branch-symmetric.

5. **C_max derivation.** `C_max` is derived from the population action-space, not from penalty tuning: `C_max = max_session_read_tokens + fixed_action_overhead_tokens`, where `max_session_read_tokens` is the larger of (a) the sum of all visible catalog surface token counts and (b) the configured read-token cap if the cap can stop earlier, plus the predeclared per-action wrapper cost for `max_steps`. If the implementation uses a simpler cap, pin it as `c_max = max_read_tokens + max_steps * action_overhead_tokens`. The important rule: the population declares catalog size, `max_steps`, and token cap because that is the retrieval session budget, then C_max follows mechanically. Any C_max change is a new fixture class and must re-run self-falsification.

6. **Self-falsification episodes.**

- `ballast-discriminator`: same SBR geometry, same stale-summary form, but the named `discriminator_surface_id` is ballast: reading S1 is not necessary for adequacy and does not change oracle outcome. Expected: ECAC win must not fire for resumable; false-continuation must be false or `not_engaged`. If the scorer rewards reading/skipping the ballast as if it mattered, the measurement regime fails.
- `neutral-frontier`: same catalog and S1, but no stale frontier claim in resumable state; either no summary or a neutral frontier artifact that preserves uncertainty without asserting Plan R closed. Expected: resumable must not beat cold except by the predeclared warmth tax accounting, and no stale-summary suppression claim is licensed. If resumable wins here, we measured prompt compression or control-artifact advantage, not pause/resume behavior.

7. **`affordance_symmetry_ok`.** Computable guard over every paired cold/resumable session in the fork group:

`affordance_symmetry_ok = same(action_space_hash) && same(catalog_hash or same ordered [(surface_id,title,content_hash,token_count,read_legal,default_hidden)]) && discriminator_surface_id in both visible sets && read_legal(discriminator) in both && catalog_position_rule identical && no branch-specific hidden/default-open surface && same budgets && same STOP legality`.

A failure is `confounded`, not a loss. If the stale summary alters the catalog title, order, default expansion, or read legality, the cognitive-temptation claim is dead.

**What v0.1 carries verbatim into v0.2:** the witnessed ordering discipline; population precommit before route/session; two-phase mint in spirit and guard form (`frontier_freeze`/derivation before `frontier_state_minted`, with banned work-product fields and genealogy refusal); derivation from precommitted rulebook/catalog/freezed frontier rather than answer text; replayed cost from ledgered reads rather than logged cost; branch-blind quality oracle; and the N=1/temp=0 deterministic reduction as a disclosed compatibility mode. The warmth tax survives only as accounting for state/rendered-artifact tokens inside ECAC, not as an independent win predicate.

**What is out of scope for v0.2 baseline:** causal-reduction (experiment 2); arbitrary interruption/open filesystem search; claims about engine beliefs, intentions, or disposition from the read distribution; paired-policy causal attribution beyond the symmetric cold/resumable SBR contrast; work-product snapshots or answer caches; stochastic dominance as the primary claim; and any spec language that says the route distribution itself is success. Success means lower precommitted mean ECAC at adequate continuation under symmetric affordance. Nothing fancier gets to ride along.
