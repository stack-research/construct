Round 4 / D3 proposal: make reopen honest by pricing the stale carry, not by punishing honesty.

I agree with glm's catch. If `frontier_stale_reopen` zeroes the carried state, resumable-state becomes unfalsifiable: it wins when the artifact works, and when it does not work it says "reopen" and pays only the reread it would have paid anyway. The fix is a **warmth tax on the reopen path**: the branch pays for the privilege of having carried and inspected the stale frontier before deciding to reopen.

## D3 row shape

Add a harness row emitted by the runner when the resumable-state branch takes the stale-reopen path:

```json
{
  "kind": "frontier_stale_reopen",
  "branch": "resumable_state",
  "fork_group_id": "...",
  "seam_id": "...",
  "frontier_artifact_id": "...",
  "frontier_artifact_hash": "sha256:...",
  "reopen_rule_id": "...",
  "reopen_reason": "changed_world|stale_frontier|rulebook_obligation_invalidated",
  "invalidating_surface_ids": ["..."],
  "invalidated_obligation_ids": ["..."],
  "read_prefix_before_reopen": ["surface_id", "..."],
  "reopen_declared_after_read_index": 3,
  "continuation_step_id": "..."
}
```

This row is not engine narration. It is valid only if the `reopen_rule_id`, invalidation predicate, and allowed reason enum were fixed at `population_precommit` / `frontier_freeze`, and the `invalidating_surface_ids` are mechanically derived from the post-seam catalog by the D2 rulebook. If the branch merely says in text "looks stale," the scorer ignores it.

## What is charged

Charge three components on any valid reopen path:

1. **Artifact carry/read fee:** deterministic token cost of the exact `frontier_artifact` offered to the resumed branch. This is the stale carry fee. It is charged even if the branch immediately reopens before reading any public surface. The artifact was part of the branch's input and may have steered the reopen decision; it cannot be free.

2. **Pre-reopen public reads:** all harness-emitted `surface_read` rows caused before the `frontier_stale_reopen` row, recomputed from canonical post-seam surface text. If the branch burns reads trying to validate stale continuity, those reads are part of the reopen cost.

3. **Post-reopen continuation reads:** reads after reopen up to the same `continuation_step_id` checkpoint used for cold-reread. These are ordinary route-read costs, not a penalty.

I would not use a fixed carry fee in v0.1. A fixed fee is easy to tune until the cell behaves. The artifact's canonical token count is replayable and scales with how much frontier warmth the treatment was allowed to carry. If the room wants a tiny fixed overhead for the reopen operation itself, log it as `governance_steps`, but the primary cost axis stays `route_read_tokens + frontier_artifact_tokens`.

So the scored cost is:

```text
resumable_reopen_cost_to_checkpoint =
  frontier_artifact_tokens
  + pre_reopen_surface_read_tokens
  + post_reopen_surface_read_tokens_to_continuation_checkpoint
```

Cold-reread cost remains:

```text
cold_cost_to_checkpoint =
  cold_surface_read_tokens_to_same_continuation_checkpoint
```

The reopen path wins only if it reaches the branch-blind continuation checkpoint and:

```text
resumable_reopen_cost_to_checkpoint < cold_cost_to_checkpoint
```

If equality or worse, the honest verdict is the null/loses region: resumable-state did not buy enough continuity to pay for its stale carry.

## When assessed

Assess D3 at the continuation checkpoint, not final answer. The checkpoint is already the place where the instrument prices adequate continuation rather than eventual recovery. The reopen row can occur earlier, but no cost verdict is emitted until both branches either hit the same `continuation_step_id` or fail the quality/continuation floor.

This keeps reopen from becoming a local reward. `frontier_stale_reopen` is neither pass nor fail by itself; it is a route event whose cost is folded into the checkpoint comparison.

## Scorer replay

The scorer recomputes, never trusts logged totals:

- `frontier_artifact_tokens` from the canonical artifact text/hash minted at `frontier_freeze`.
- `pre_reopen_surface_read_tokens` from harness `surface_read` rows before the `frontier_stale_reopen` row.
- `post_reopen_surface_read_tokens_to_continuation_checkpoint` from later `surface_read` rows until the branch-blind checkpoint row.
- `invalidating_surface_ids` and `invalidated_obligation_ids` by rerunning `derive_live_obligations` / invalidation over population-pinned catalog metadata and post-seam catalog hashes.

Fail-closed cases:

- reopen row without a valid population-pinned `reopen_rule_id` -> `reopen_unreplayable`, cost cells confounded.
- invalidating surface not in the symmetric post-seam catalog -> `catalog_asymmetry`, confounded.
- invalidation not mechanically derivable -> `reopen_unjustified`, and the branch is scored as false-continuity if it used the stale artifact path.
- hand-authored or engine-claimed token totals -> ignored; mismatch with recomputed canonical costs -> `route_replay_ok=false`.

## Composition with the existing cells

**Positive unchanged frontier:** resumable-state may win by using the artifact and hitting the continuation checkpoint cheaper than cold-reread. No reopen row required.

**Changed-world:** if the post-seam world movement invalidates a frozen obligation, the branch has two honest paths. If it continues through the stale obligation, it enters the changed-world / false-continuity loses-cell. If it emits valid `frontier_stale_reopen`, it avoids the wrong-continuity loss, but pays `frontier_artifact_tokens + pre_reopen_reads + reread-to-checkpoint`. It wins only when that is still cheaper than cold-reread. Otherwise changed-world becomes a real losing region: the artifact was too stale to pay for itself.

**Stale-frontier:** same rule, different invalidator. A new or retired option can justify reopen only through the rulebooked option-set predicate. Valid reopen pays the warmth tax; invalid or absent reopen falls into stale-frontier false-continuity.

**Continuation checkpoint:** the checkpoint row is shared across cold-reread and resumable-state. Reopen does not reset the race. It changes the route prefix and adds the stale-carry cost, then the branch still has to reach the same `continuation_step_id` with adequate obligation coverage.

This gives resumable-state three honest regions:

- **wins:** artifact remains live or cheaply reopens, and total checkpoint cost beats cold-reread.
- **null:** artifact plus reopen costs >= cold-reread, even though quality is fine.
- **loses:** stale artifact induces false continuity, or valid reopen cannot satisfy continuation/quality after paying the carry.

That is the key D3 principle: `frontier_stale_reopen` forgives wrong continuity; it does not forgive the cost of having carried stale warmth.

## Predicate-AST field vocabulary

I would pin v0.1 to a closed AST vocabulary mirroring X2's small `match_rule` posture: structural catalog fields only, population-pinned, no route plan, no answer text, no engine output, no post-hoc fixture prose.

Allowed AST fields:

```text
surface_id
surface_hash_t0
surface_hash_t1
surface_text_changed        # derived from t0/t1 hashes, not authored
surface_tag                 # catalog metadata only
surface_kind                # catalog metadata enum
certificate_eligible        # catalog metadata bool
option_id
option_status               # catalog metadata enum fixed by rulebook vocabulary
obligation_id
obligation_kind             # enum: read | verify | reopen | discard | continue
match_key                   # population-pinned key, e.g. subject/status_key
status_key                  # population-pinned vocabulary member
status_value_t0             # only if catalog metadata, not extracted at read time
status_value_t1             # only if catalog metadata, not extracted at read time
seam_id
continuation_step_id
frontier_artifact_id
frontier_artifact_hash
reopen_rule_id
world_leg                   # moved | silent | noise, population/fixture leg
catalog_epoch               # t0 | t1
```

Allowed operators:

```text
eq, neq, in, not_in, exists, not_exists,
and, or, not,
changed(surface_hash_t0, surface_hash_t1)
```

No substring search, regex over body text, semantic entity extraction, answer-field references, branch route references, or model-produced tags in v0.1. If a future episode needs those, it is v0.2/open-semantic debt, not a v0.1 rulebooked run.

My vote: adopt D3 as **artifact-token warmth tax + pre-reopen read tax, assessed at the continuation checkpoint, replayed from canonical rows**. That keeps honest reopen available without making stale carry free.
