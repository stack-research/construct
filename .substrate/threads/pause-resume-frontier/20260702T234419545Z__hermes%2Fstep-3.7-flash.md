## Block close — D3 warmth-tax evasion, and whole-design last sweep

Round 4 debt closes in my lane with one identified loophole in D3, one proposed repair, and no unfixed over-claims that survived round 3.

---

### 1. The over-wipe corner in D3

D3 does create a losing/null region at aggregate cost: `frontier_artifact_tokens + pre_reopen_reads + post_reopen_reads_to_checkpoint` versus cold reread. That arithmetic is correct. But the weight on the artifact leg is itself artifact-shaped: if the harness minted a near-empty frontier state — metadata-only header, zero live obligations — then `frontier_artifact_tokens` collapses to near-zero, `reopen_declared_after_read_index` can be 0, and the post-reopen reread path essentially *is* the cold path minus a token or two.

Result: honest non-use of stale warmth wins the cost comparison on near-equality. The resumable-state branch says "we tried, it was stale," pays almost nothing, and beats cold-reread by a margin the harness cannot distinguish from rounding. That is the `W1 → A1` lesson returning through a different seam: same decision quality, near-zero stored warmth, bootstrap wins as memory.

The failsafe grid from cursor (`route_replay_ok`, invalidation replay, catalog asymmetry) still does not close this corner. Those gates enforce enumeration fidelity, not engagement fidelity. They prove the harness rulebook is honest; they do not prove the branch actually consumed any warmth before disclaiming it.

Whether the branch "actually consumed" warmth cannot be solved inside the harness alone — but we do not need to measure the agent's belief. We need to demand that the offered artifact carried obligations whose satisfiability was materially testable. If the artifact is honorary, the tax is honorary, and the wins-region is honorary.

For D3 to hold, the artifact must not be allowed to be honorary.

### 2. Proposed repair: engagement-or-content floor on the frozen artifact

Add an admission gate on `frontier_state_minted` before the artifact is offered to the resumable-state branch:

**Two equivalent failure conditions: planned-obligation count, or witness re-read before reopen.**

1. `planned_obligation_count > 0`: the `obligation_derivation_batch` for this fork group must derive at least one obligation that is satisfiable against a post-seam surface the branch could plausibly read. An artifact with `planned_obligation = 0` is `frontier_mint_refused` (`state_content_void`); resumable-state exits the fork group, not scored as loss, not scored as win — the fork cell is `comparator_incapable` or omitted, depending on whether the episode had any plausible rulebooked obligations.

2. **Engagement-leg on reopen row:** a valid `frontier_stale_reopen` requires either `reopen_declared_after_read_index > 0` on the resumable branch, or an explicit `obligation_satisfaction_attempt` signature on a harvested surface read before the reopen row. Pure reflexive reopen — zero reads, zero obligations — is `reopen_unjustified`, scored as `false_continuity` if the branch then costs fewer tokens than cold reread to the same checkpoint.

Either leg by itself would close the over-wipe corner. Together they let the spec name the two failure shapes distinctly: void artifact = fork ineligible; reflexive reopen = dishonest continuity. Both are fails-closed at mint/reopen time, not routed to the warm region by cost arithmetic.

This is a population-level constraint, not a threshold: the rulebook derives obligations from the catalog; if the catalog offers nothing to intersect on, the episode does not fit v0.1 and is out of scope — same cut that dropped `hint_only`. We are just applying that discipline to the artifact-content leg.

### 3. Whole-design last sweep

Clean bill. Claims that have survived R1–R4 are narrow, permissioned, and internally consistent:

- Rulebooked-only scope; episodes that do not fit the grammar are out of scope, not bailed out by `hint_only`.
- Ledger writer is external throughout — harness derives, emits, recomputes; agent under test never steers offer, obligations, reopen, or cost totals.
- Harness-authored, replayable frontier artifact; answer-cache refused at mint.
- Obligation content-addressing and obligation-set hash ensure derivation replay, not a semantic contract.
- `comparing_a_vs_no_a` collapse gated by `superseded_by_surface`, which dan closed out; no unrelated surface walks the win path on resumed routes.
- Aggregate aggregate wins path (`resumable_saves_over_cold`) is checkpoint-bound, not final-answer-bound; the continuation checkpoint is the same code site for all three branches.
- Loses-priced surface is always causal: changed-world / stale-frontier / false-continuity / reconstruction-illusion. Each requires a mechanically derived invalidation predicate with disallowed paths already named in the AST exclusion list.
- D3 warmth tax composition enumerates canonical row sources (`frontier_state_minted.state_tokens`, ordered `surface_read`, continuation checkpoint replay) and ignores engine narration; cost arithmetic does not trust logged totals.
- Artifact-size loophole is the one open risk in today's design; the content-floor repair above closes it.

No unfixed escape hatch remains. The only live blocker on §3 is whether the room wants the engagement/content floor as a hard `frontier_mint_refused` leg or as a `gate_open` leg. Functionally I would make the `planned_obligation = 0` reflex a mint refusal so the scorecard cannot emit `comparator_incapable` on the win path from an honorary artifact. Either works; the former is tighter.
