### Ruling on the kickoff seam

**(a) Is a fixture-authored, unminted stale summary legal as the resumable foreground? — NO.**

`run_sbr.py:116–119` resolves `foreground = render_foreground_block(episode.get("resumable_foreground"))` and emits it straight into the session. There is no `frontier_freeze`, no `frontier_state_minted`/`frontier_mint_refused`, no `obligation_derivation_batch`, and **no `population_precommit` row anywhere in the v0.2 path**. §15 says no artifact is offered except through the two-phase mint; §22 carries "population precommit before any session," "the two-phase mint and every D1 guard," and "D2 derivation + replay-as-authority" **verbatim** into v0.2, and names `mint_frontier_state`, `derive_live_obligations`, `prf_ablation` as unchanged modules. The build runs none of them. The `foreground_disclosure` is honest that this was done, but honest disclosure is not exemption. The current build's approach (unminted foreground) would require a spec re-derivation to be legal — codex/hermes are correct on that narrow point.

**(b) Is pricing that string as `a_i` faithful to §16? — NO. This is the inversion my §20 sentence was written to prevent, and I confirm it from the code.**

`score_prf._artifact_tokens` (`score_prf.py:223–231`) returns `_tokens(render_foreground_block(episode.get("resumable_foreground")))`, and `0` when the foreground is absent/null. `ep-neutral-frontier.json:106` sets `resumable_foreground: null`. So on the one fixture where §20 pins *"resumable pays the artifact carry with no stale summary to compensate — that is the falsifier's anchor; comparing read-cost-only would erase the carry,"* the build sets `a_i_resumable = 0` (and `ecac.a_i_resumable = 0` in the ledger). The comparison on neutral is read-cost-only — exactly the erasure the sentence forbids. The build arrived at it through the seam: it conflated *foreground = stale summary* with *foreground = canonical frontier artifact*, so deleting the stale claim deleted the carry. §16 itself distinguishes them — "`a_i` is the rendered frontier-artifact tokens: `a_i(resumable)` = **the canonical artifact tokens**, `a_i(cold) = 0`." The stale summary is the temptation (§19), not the artifact (§16). The build priced the temptation string as the artifact.

**Smoking gun the three reviews did not name:** `ep-neutral-frontier.json:88–90` carries `band_rule: "effective cost including a_i; a_i(cold)=0"` and `band_tokens: 0`. §20's "band pinned at 0" is the **win tolerance** — zero tolerance for any strict ECAC win → self-refutation. The fixture's `band_tokens: 0` field reads as "a_i band = 0 tokens" and the build implemented exactly that misreading: `a_i = 0` on neutral. The win-band and the artifact-carry are different quantities; the fixture metadata conflates them, and the code follows the conflation. That is the inversion encoded at the fixture layer, not just the scorer.

**On the `score_prf.py:424–427` short-circuit.** Gemini calls it "a literal bypass of the band=0 predicate." I refine: the short-circuit *is* a faithful implementation of the band=0 **verdict rule** (§20: "ANY strict ECAC win → `PRF2-neutral-null`" → `if sf == "neutral_frontier" and cost_win: return PRF2-neutral-null`). The disease is not the short-circuit; the disease is that with `a_i = 0` the comparison is read-cost-only, so the short-circuit fires on the inverted accounting and produces the right cell name from the wrong numerator. The test `test_neutral_null_fires_on_resumable_win` (`test_prf_ecac.py:120–123`) asserts the cell name only and never asserts `a_i_resumable > 0` on neutral — so the suite is green while the falsifier anchor is gone. That is the test-honesty gap, and it is the reason wire-green did not catch this.

### What the three reviews missed (additions)

1. **No `population_precommit` row in the v0.2 path at all.** §22 carries it verbatim; the build omits the entire precommit spine, not just the mint rows. This is a harder conformance failure than "no mint" — it is "no population precommit before any session," which §22 lists first among the carries.

2. **Mock/real prompt divergence (renderer drift).** The two `system` strings differ in preamble (see finding 3 above). Wire tests validate the with-preamble observation only; the real engine additionally sees the no-preamble init message. `run_config` does not record a renderer hash for the v0.2 path (`renderer_version()` exists in `engine.py` but is not called from `run_sbr`). So a future change to either `system` string drifts silently — the exact failure mode AGENTS.md names.

3. **`band_tokens: 0` in the neutral fixture encodes the misreading.** The field name conflates the win-band (§20's zero tolerance) with the artifact carry (`a_i`). It is not read by the scorer, but it documents the inversion at the fixture layer and would mislead any future builder. Delete it or rename to `win_band: 0` and let `a_i` be derived from the minted artifact.

4. **`stale_only_asymmetry` gate leg is a hardcoded `True`.** `check_prf_fixture.py:132–135` appends `(stale_only_asymmetry[{eid}], True, …)` with no computation. Coverage is partially rescued by `affordance_symmetry_across_variants` and `variant_declared_overrides_only`, so not blocking on its own — but a §19 gate leg that always passes is a stub masquerading as a check. Compute it or remove the name.

5. **`decision_read_chain_ok` is a weak inequality, not a bijection.** `score_prf.py:383` checks `len(reads) <= len(steps)`. §15 names it a bijection step→read. Necessary but not sufficient — a session with duplicate/unordered reads against parsed steps would pass. Tighten to a step↔read index bijection over parsed READ decisions.

6. **`_false_continuation` is prose substring matching, not the oracle key.** `score_prf.py:286–290` matches `stale_markers` (`"pending survey"`, `"commissioning window pending"`, …) as substrings of the normalized answer. §18 says the failure mode is "content-matched against the stale state under the oracle key, never prose, never the title." This is prose matching. It is mock-only today (the answer is synthesized by `synthesize_mock_answer`), so it is a disclosed wire-test artifact, not a current blocker — but it must be replaced by an oracle-key check before any real-engine adequacy claim, and the debt should be named in §23, not left implicit.

7. **`run_config` does not pin the canonical artifact.** §16 says `a_i` is recomputed from the canonical state at `state_digest`; the v0.1 path does this (`recompute_state_tokens(freeze["canonical_state"])`). The v0.2 path has no artifact hash in `run_config` and recomputes `a_i` from a fixture string. Part of the mint seam, but worth naming separately: once the mint runs, `a_i` must be recomputed from the minted `canonical_state`, and the artifact hash pinned at `population_precommit` — not from the rendered foreground.
