### (2) The conflicts, precisely — and rulings from the round's own arguments

**C2a — Regime question: is deterministic SBR worth running at all?** Codex: yes, as the N=1 point-ECAC compatibility bridge. Composer: at temp=0 with current `LocalEngine` (no seed in the request body), 3/3 shakedown draws are identical — N>1 measures API jitter. Gemini: temp=0 is a static state-machine trace, not a behavioral experiment; no behavioral claim on the deterministic reduction. **Ruling (settled by arguments):** deterministic SBR **is** worth running — but only as (i) wire/machinery validation proving the SBR loop, action schema, budget decrement, and C_max pricing execute, and (ii) the v0.1 compatibility bridge (point-ECAC = the `A+P+R<C` tree at N=1/temp=0 when routes match the v0.1 skeleton). **No behavioral claim is ever licensed on Regime D.** Reporting-drift guard (hermes): the spec records the regime in `run_config` and makes the win predicate regime-conditional; it forbids citing Regime-D rigor on Regime-S runs and vice versa.

**C2b — Statistic.** Settled. mean-ECAC is primary **when sessions vary** (codex proposes, composer+gemini concur). Stochastic dominance is out as primary (codex rejects as R5-overread; hermes Block B requires its own loses-cell or it collapses). false-continuation rate is diagnostic only (all four). **Final: mean-ECAC primary in Regime S; diagnostic rates never the win predicate.**

**C2c — N derivation.** Codex: N≥10 precommitted. Composer: N≥10 is inert at temp=0; honest N comes from pilot variance of `effective_cost` and the desired CI on `mean_cold − mean_resumable`; precommit N from a dispersion probe. Gemini: suppression is intrinsically distributional, so the **dispersion probe is load-bearing** — K≥5 pilot draws same branch/fixture; `unique_realizations==1` → halt or downgrade to point-mode, emit `zero_dispersion_regime`, refuse behavioral win. **Ruling (settled by arguments):** N is **not** 10 by fiat. Regime S precommits: a temp in a precommitted range (e.g. [0.3,0.7]), seed if the backend exposes it (today `LocalEngine` does not — record `seed: unavailable` rather than silently omit), a pre-run dispersion probe (K≥5), and an N derived from pilot variance with `unique_realizations` documented in `run_config`. If `unique_realizations==1`, the runner emits `zero_dispersion_regime` and refuses to score a behavioral win. This kills the "API-jitter masquerade" and the §4c-1 adequacy masquerade at the verdict layer.

**C2d — Dynamic vs static C_max.** Codex: `C_max = max(sum_visible_surface_tokens, max_read_tokens) + max_steps*action_overhead`. Gemini: cut to static `c_max = max_read_tokens + max_steps*action_overhead_tokens`. Composer: "same thing if pinned," scorer is authority. **Ruling (settled by arguments):** static wins, and not just for simplicity. Under ECAC, C_max prices **only failure** — a successful exhaustive read pays its actual cost, not C_max. So C_max should be the **budget supremum** = the most a session *could* spend = `max_read_tokens + max_steps*action_overhead_tokens`. Codex's dynamic form equals the static form whenever `max_read_tokens ≥ sum_visible_surface_tokens`; when the catalog exceeds the cap (the bounded-retrieval case, which is the whole premise), the dynamic form prices failure at `sum_visible + overhead` — **more than any session could spend** — which is penalty inflation and reopens the carry-fee smell the chair ruled closed. **Final: `c_max = max_read_tokens + max_steps * action_overhead_tokens`, pinned at population precommit, with the precondition `max_read_tokens < sum of visible surface tokens` (the budget is genuinely binding — the engine faces a real read-vs-stop tradeoff).** Scorer-recomputed (`c_max_replay_ok`); mismatch → `confounded`. This is a real ruling from ECAC's semantics, not a coin flip.

**C2e — Programmatic vs authored self-falsification episodes.** Codex/composer: authored JSON fixtures `ballast-discriminator` + `neutral-frontier`, scored as cell_verdicts, fail-closed. Gemini: cut to runtime perturbation (strip stale summary / swap discriminator id), −60% fixture overhead. **Ruling (settled by arguments):** authored wins, with gemini's DRY intent captured. A scored `cell_verdict` requires a **population-precommitted fixture with its own oracle keys** — runtime perturbation breaks that precommitment, lets the fixture-honesty gate (`discriminator_symmetric`, `stale_only_asymmetry`, `catalog_flat`) be bypassed for the variants, and is exactly the reporting-drift vector hermes named (cite the authored baseline's rigor while the perturbed variants escape the gate). It also risks a buggy perturbation silently altering the catalog hash. The overhead cut is marginal (two small JSON files) and the risk is structural. **Final: authored fixtures, population-precommitted, with a shared base and declared overrides for the variant fields (`discriminator_surface_id`, `stale_summary` presence) — this captures gemini's DRY intent without breaking precommitment. Each variant runs the same fixture gate pre-execution.** Hermes's "loses-cell where the regime is uninformative" = these two scored cells; oracle keys explicit beyond title.

**C2f — `affordance_symmetry_ok` static vs runtime.** Codex/composer: per-fork-group runtime guard over realized sessions. Gemini: static, once in `check_prf_fixture.py`. **Ruling (settled by arguments):** both, split by what each can catch. Population-level symmetry (catalog_hash, action_space_hash, sort rule, budgets, discriminator visibility/legality, title grammar) is **static-gated in `check_prf_fixture.py` pre-execution** — cheap, catches fixture-authoring errors, gemini is right that this is the bulk. A **thin runtime guard** confirms the realized `affordance_presented` rows match the static declaration and that the `catalog_hash` recorded in `sbr_session` matches the gated population — this catches runtime drift (e.g., a foreground injection bug that accidentally hides a surface on one branch), which a pure static gate misses. Failure at either gate = `confounded`, not a behavioral loss.
