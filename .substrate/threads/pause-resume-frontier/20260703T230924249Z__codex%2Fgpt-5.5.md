I would supersede the Meridian shape with a **triangulation docket** family: 18-24 surfaces, all honest administrative records with parallel titles, where the answer is not in any one surface but in a three-leg conjunction. Example world: a facilities permit/case queue. The question is: "For case K-17, what disposition should be entered now: approve, defer, reject, or reopen?" The current disposition is determined by exactly three current catalog facts: the case's live status note, the exception register row, and the reviewer-clearance ledger. Each leg is a normal surface type, and there are several sibling surfaces of the same kind for nearby cases or adjacent rule sections. Titles are not theatrical and not hidden: `K-17 Status Note`, `Exception Register A`, `Reviewer Clearance Ledger`, `K-18 Status Note`, `Regional Deferral Bulletin`, `Inspection Addendum`, etc. The discriminating content lives inside the body rows, not the titles; the title tells you the surface class, not whether it is sufficient or stale. A capable cold engine should see multiple plausible first reads and no single read-stop path; an adequate answer requires reconciling at least three surfaces. My earlier rotate-order/blind-id/split-discriminator instincts survive only as guardrails: deterministic sort still branch-symmetric, ids still opaque enough not to encode answer lemmas, and the discriminator is split across required legs rather than hidden.

The frontier artifact should buy a **narrow obligation bundle**, not a stale answer. At freeze, the witness has read enough to derive: case K-17 is the live option; disposition depends on obligations O1 `verify current status note`, O2 `check exception register class E`, O3 `check reviewer clearance`, plus one reopen/discard rule if any of those keys changed. The artifact names the obligation ids and surface ids/classes, with no disposition value, no rank, no rationale. In resume, the resumable branch can cheaply read the three named legs, possibly a fourth if a reopen rule fires, then STOP with quality. Cold gets expensive honestly because it knows the question but not the witnessed frontier: among 18-24 catalog entries, 8-10 are plausibly status-bearing, and several are necessary-looking but non-dispositive. If a strong cold engine routinely reads exactly the same three legs and stops, that is a clean heir/cold-dominates null; the family failed to expose a pay-window. But if the frontier hypothesis is true anywhere, this is where it should pay: `A + three/four targeted reads < cold's exploration of six-plus plausible records`, with both branches able to reach quality.

For R2, I would keep §15's closed action space but change the action surface from fragile JSON to a constrained one-token command grammar: visible menu rows have stable handles `R01`...`R24`, and the only legal next actions are `R01`...`R24` or `STOP`. The harness maps a legal handle to `READ(surface_id)` and emits the same `route_decision`/`surface_read` rows as today; malformed completions are still refused, but the parser can accept case and whitespace variants without changing the ledger semantics. This is not free-text routing: it is a finite enum per session. It should reduce small-engine machinery noise without loosening the sealed action model.

Self-falsifiers: the **ballast analog** keeps the same catalog geometry and action grammar but moves the named obligation bundle to surfaces whose content is genuinely unnecessary for the oracle disposition; any PRF2-cost-win there self-refutes the measurement. The **neutral analog** keeps an artifact-sized foreground with the same token carry and parallel obligation shape, but its obligations do not select a prunable case frontier for the asked question, and there is no stale/live tension; with `a_i(resumable)>0`, any strict ECAC win is again a self-refutation. I would also add a pre-build calibration gate: with the foreground present, each target engine must pass quality on a scripted route that reads the three true legs, or the family is rejected before Regime-S. That makes "no pay-window exists" falsifiable in the right way: if the scripted route passes, dispersion exists, and cold still matches or beats resumable, we learned the artifact did not buy token management rather than merely poisoning the engine or hiding the answer.
