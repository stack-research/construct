### §9 Q2 — information-parity comparator (emphasis)

**Finding:** the literal spec text still carries the failure mode.

§6 L58 reads:

> *“cost is compared at the first prefix whose cumulative read set covers the oracle-marked answer-bearing surfaces — equal cumulative information, not merely equal quality.”*

If any implementer reads “oracle-marked” literally, the comparator steers itself: a human or fixture author can mark surfaces after T1, after routes, or after answers, and the comparator becomes a hidden authored key that selects whichever surface the hint already named or forces B+ through redundant pages to inflate its parity point. That is the exact hole my answer-axis-gating catch exposed in the first place — and the spec fold has **not yet excised the phrase** that names it as welcome.

**Reviewer convergence is real but shallow.** Codex and cursor both agree the comparator must derive answer-bearing surfaces mechanically, not author them. They do **not** agree on the operationalization:

1. **Placement:** codex anchors the match rule in `trigger_precommit`; cursor demands it be fixed in `population_precommit`, because trigger time is late enough to tune the certificate toward C's route hint. Cursor's claim is correct: `population_precommit` is branch-blind by construction (precedes live watch, per §2), so only there can the rule be free of route knowledge. Spec does not yet encode this.
2. **Derivation:** codex allows two anchors (T1-vs-T0 diff *or* canonical current-status field); cursor attacks the dual anchor as author-selectable and demands a single diff-gated derivation per `match_rule_id`. The spec family ("impersonal, official, version-controlled lifecycle transitions") is exactly the lifecycle-diff case cursor targets, but the spec text names neither path nor enum.
3. **Leg conditioning:** codex's form is silent on how silent/noise legs produce certificates; cursor adds that moved legs yield transition certificates, while silent/noise legs yield either a stable certificate or confounded — never forced through irrelevant churn.

I **attack the convergence before blessing it:**

- If the build encodes codex's form without cursor's three constraints, the hole survives: the match rule is still late-fixable, a dual anchor lets an implementer pick the easier derivation path after seeing routes, and the silent leg becomes yet another place where parity can be faked.
- Cursor's tighter form does close those three sub-holes, but Cursor did not receive a codex reply and the tightened form is not yet spec text. Converging intent is not converged protocol.

**Verdict:** reblock with fix. Build must encode the derived-certificate rule with cursor's three constraints — `match_rule_id` fixed at `population_precommit`, single diff-gated derivation per lifecycle packet, leg-conditional stable/confounded handling — before `information_parity_ok` becomes a sherd-passable gate. The spec line itself should be patched in the same machinery PR so later implementers cannot re-read the old phrase as license.
