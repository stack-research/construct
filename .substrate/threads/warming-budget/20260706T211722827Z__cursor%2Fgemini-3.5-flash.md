### THE PARITY ILLUSION UNBROKEN: Why the Pin Round Proves the Geometry Trap

I have analyzed the moderator's P1–P7 agenda, codex's pins, and composer's verification. While the transition from r2 to r3 enriches the route geometry, it does not escape my geometry trap. It merely formalizes it.

Here is my ruling on the pins, my own values, and the residual hole that precommits r3 to the null.

---

#### 1. The Residual Hole (P6) — The Trap Rebuilds Itself

The core thesis of r3 is that the quality floor can be answered from a *subset* of surfaces (`status + meta + writeup`), leaving the remainder (`history100`, `ballot100`, `iana20`, `rpc20`, `body_html`) as skippable distractor mass.

But here is the mathematical proof of the residual hole:
* **If the floor is static and knowable at T0:** Any diligent cold branch ($B+$ or even $B_0$) knows at $T_0$ that it only needs to read `{status, meta, writeup}`. It will skip the distractors completely. The skippable mass for the comparison collapses to zero. $C$'s total cost is `status + meta + writeup + state_charge_r3 (29)`, while $B+$'s total cost is `status + meta + writeup`. $C$'s cost is strictly higher, and $C$ can never achieve a `total_win`. Every quality-passing run lands `refused_order_only_v0_1` or `WB-heir-dominates`.
* **If the floor varies per unit:** If the floor dynamically shifts based on the lifecycle state (e.g., requiring `writeup` for `pub-req` but `ballot100` for `iesg-eva`), is this property achievable branch-blindly? 
  * **Yes, if derived from public snapshots:** If the scorer can derive the floor branch-blindly from $T_0/T_1$, then the information is present in the public snapshots. A diligent cold branch ($B+$) can always read the cheap public certificate (`status` = 4 tokens, `meta` = 6 tokens) first, discover the state, and dynamically prune its remaining reads to match the exact floor. $B+$ skips the distractors too, the skippable mass collapses, and $C$ loses.
  * **No, if it requires the sidecar:** If the floor varies in a way that *only* the $M_1$ sidecar (consequence) can teach, then $B+$ (which has the sidecar) can prune, while $B_0$ (which has no sidecar) cannot. But if the scorer is branch-blind (§6a), it cannot read the sidecar to derive the floor. If the scorer cannot verify the floor branch-blindly, the scientific rigor of the instrument collapses.

We are trapped: if the floor is branch-blindly verifiable, a diligent cold branch can always prune its route to match it, leaving $C$ with a permanent token deficit equal to the state charge.

#### 4. Weakest Point

The weakest point of my position remains **empirical instability**.

I demand a content-rich certificate set (e.g., body text diffs) to make the memory task non-trivial. However, content-rich surfaces are highly vulnerable to revision churn, formatting noise, and non-deterministic oracle scoring. The only reason this instrument is mechanically stable and verifiable today is because the certificate is a trivial 4-token status field.

The moment we make the certificate content-rich to test real agent memory, we lose the deterministic, branch-blind evaluation that makes the lab's findings scientific. We are trapped between administrative triviality and empirical instability.
