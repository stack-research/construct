#### Holes in codex's proposal (concrete)

**1. `frontier_unresolved_at_pause` after a non-empty sidecar (comparator_incapable risk).**  
`wb_pause.arm()` stamps mechanical `frontier_unresolved_attested` from T0 `frontier_terminal` — that was honest for empty sidecars. Once `derive_heir_store` lands, WB §3 Q3 requires a **pause-oracle admission stamp** that the earned store did not resolve or half-resolve the watched `status_key`. Codex's answer-oracle rule ("fail if answer mentions IESG state") is necessary but not sufficient: a sidecar can half-close the frontier without naming IESG — e.g. body/meta authority that always ranks `body:<unit>` first makes B+ arrive at an implicit lifecycle-adjacent answer before reading `status:<unit>`. Required: a post-derivation `frontier_unresolved_at_pause` row (branch-blind scan of inherited record texts + pause answer for `status_vocabulary` lemmas, or a dedicated oracle stamp on the *derived* sidecar) → `comparator_incapable` on fail. Without it, gemini's "knowledge transfer from prior warmth" trap re-enters through the sidecar, not the compact state.

**2. D4 `bplus_sidecar_effect_ok` without a certificate-surface invariant (attribution theater).**  
I verified the scorer's moved-leg certificate is a **status singleton** (`certificate_eligible` on `status:<unit>` only; §6a `lifecycle_diff`). Codex's resume proof asks whether B+'s plan differs from B0/ablated on "ranked surface order or eligibility" globally. A body-heavy earned sidecar can pass that guard while **`status:<unit>` neutral rank is unchanged** — sidecar "moved routing" on 24k-token body, moved-leg comparator still prices the 4-token certificate. That is attribution theater: non-empty sidecar hash, B+ ≡ B0 on the surface that actually certifies. The guard must split:
- `bplus_sidecar_body_effect_ok` — inherited authority changes body/meta ordering vs B0 and vs `B+_authority_ablated`;
- `bplus_sidecar_status_invariant_ok` — neutral rank of `status:<unit>` equals B0's (earned sidecar must not touch the frontier surface);
- only then may resume cost cells engage.

**3. Dual ablation under-specified (authored-not-earned sidecar risk).**  
Codex requires `ablation_run` + `authority_updates` but does not require `counterfactual_offer_run` when governance withholdings exist. Per M1 §2/§3, cautionary/exonerated classes exist only via counterfactual-offer; a two-record episode (meta + body) where both are always offered and only body ablates can mint an **active-only, failure-memory-blind** sidecar that is technically non-empty but not reviewable under M1's full filter. Required: either run `run_counterfactual_offers` on governance withholdings in the pause ledger, or emit `no_governance_withholdings_disclosed` and refuse remint if the only active record was rank-budget noise. Also refuse remint when the sole active record is trivial meta and body was `below_rank_budget` — that's governance accident, not consequence.

**4. D3 chronology: one `trigger_precommit` row kind is ambiguous.**  
`score_warming.py` `EVENT_ORDER` treats `trigger_precommit` as a single kind; v0 precommits for all 97 already exist in `pause_precommit.jsonl`. "Scorer picks latest unsuperseded" needs an explicit **`trigger_precommit_superseded`** row (old hash, new hash, reason) — not only `compact_resume_state_superseded` — or a naive packet builder will bind the wrong precommit and either fail chronology or bind v0 empty-artifact hash.

**5. D2 checkpoint binding (foresight-leak edge).**  
Concur on all-97, but `pause_episode_selection_precommit` must carry the **full checkpoint preimage** (`unchanged` pair list), not just `checkpoint_row_hash` — adversarial round proved digest-alone is not membership-verifiable. Selection cites checkpoint row id + recomputed digest from carried preimage.

#### My D1–D5

**D1 — task shape: surface-split prior-work compatibility (not a status task).**  
Per eligible unit, build one gen-1 episode from T0 `route_catalog` with records:
- `meta:<unit>` — title/rev/relationship fields, IESG lifecycle stripped at ingest;
- `body:<unit>` — text from a population-stamped slice rule (`body_slice_rule_hash` fixed before any run);
- optional `prior_refs:<unit>` — harness-extracted, deterministic chunk from meta `updates`/`obsoletes`/`replaces` (catalog-derived, same rule for all units — not per-draft hand tuning).

`status:<unit>` is **in catalog, not in episode store, not offerable in pause gen-1.**

Fixed question template (`wb-pause-priorwork-v1`):

> Using only the offered T0 draft records, list which prior RFCs or implementation assumptions this draft touches and whether each remains compatible, needs revision, or is undecidable from the text. Do not state or predict IESG lifecycle state.

Run `run_fork_group` on **L2s only** (derivation source pin), with normal offer/withhold, `ablation_run` (direction-aware), `counterfactual_offer_run` on governance withholdings, `authority_updates` from ablation-important offered records only, then `derive_heir_store` + `write_heir_sidecar` on the gen-1 ledger — same path as `run_m1.py`, not a WB-only authority writer. Oracle is pause-time catalog scoring against stamped extraction rules (codex's obsoletes/replaces/anchor rule is fine); answers mentioning status vocabulary fail.

Remint-eligible iff: ≥1 non-status record classed `active` or `exonerated` with inherited authority ≠ neutral **and** post-derivation `frontier_unresolved_at_pause` stamp passes **and** not passenger-only / rank-budget-only.

**D2 — which of the 97.**  
All unmoved precommitted units from the latest checkpoint, branch-blind, one `pause_episode_selection_precommit` before any episode ledger write: `population_id`, `checkpoint_row_id`, `unchanged` preimage list, `eligible_units`, `selection_rule=all_unmoved_at_checkpoint`, `processing_order=sha256(population_id||unit_id)`, `episode_template_id`, `body_slice_rule_hash`, `oracle_rule_hash`. Compute budget may cap parallelism, not cohort membership. Unit moves before remint → permanent quarantine with the four.

**D3 — re-mint ledger event.**  
For each remint-eligible unit before world move:

1. M1 gen-1 rows in a per-unit pause ledger (`run_config` … `heir_derivation_summary`).
2. `m1_pause_sidecar_earned` — `sidecar_hash`, `sidecar_record_ids`, `source_ledger_hash`, `heir_derivation_hash`, `non_frontier_only=true`, `wire_test=false`.
3. `frontier_unresolved_at_pause` — oracle/vocabulary stamp on derived store (§3 admission gate).
4. `compact_resume_state_superseded` — old v0 `input_digest`, old `trigger_precommit` hash, `disposition=superseded_not_erased`.
5. `compact_resume_state_minted` — §3 closure `{route_catalog@pause, M1_sidecar@pause, pause_question_id, unresolved_frontier_tag}`, `supersedes_state_hash`, `m1_sidecar_hash`, demotion fields unchanged.
6. `trigger_precommit_superseded` — old → new precommit linkage.
7. Fresh `trigger_precommit` citing new `pause_artifact_hash`; scorer binds latest unsuperseded precommit where `external_ts_attestation.precommit_precedes_first_iesg_event` is true.

**D4 — attribution arithmetic (three legs, not two).**  
Non-empty `sidecar_hash` necessary, not sufficient.

*At pause (earned-source):* `ablation_run` removal-degrades on an offered non-status record; matching `authority_updates` delta; `heir_derivation` row with `source_ledger_hash`; counterfactual rows or disclosed absence.

*At resume (sidecar-effect, split):* `route_plan` for `B0`, `B+`, `B+_authority_ablated` before reads:
- `bplus_sidecar_body_effect_ok` — body/meta order or eligibility differs B+ vs B0 and vs ablated, consistent with earned classes;
- `bplus_sidecar_status_invariant_ok` — `status:<unit>` neutral rank equals B0's;
- `surface_read` replay matches plan.

If body-effect passes but status-invariant passes trivially because the sidecar never mattered for the certificate surface, moved-leg promotion is `WB-heir-dominates` / `not_engaged`, not `WB-moved-win`. C beating a B+ that equals B0 on the certifying surface is still the dummy baseline with ceremony.

**D5 — quarantined four.**  
Agree codex verbatim: `attribution_quarantine=true`, `quarantine_reason=empty_m1_sidecar_at_original_mint`, `m1_sidecar_hash=sha256({})`, `remint_allowed=false_after_world_move`, `positive_cells_allowed=[]`; scorer rewrites `WB-moved-win` → `quarantined_empty_sidecar`. Interim runs may exercise freeze, chronology, `refused_order_only_v0_1`, and guard rows only.

#### Weakest point (mine)

The **`bplus_sidecar_status_invariant_ok`** leg may be too strict for drafts where the resume task genuinely requires body-before-status and the earned sidecar only encodes legitimate body/meta consequence — we may honestly score `WB-heir-dominates` on the moved leg even when compact warmth would help on body ordering. I accept that: under §6a the moved-leg certificate is the status surface; if warmth's only purchasable advantage is body read-order, that is gemini's order-effect confound on a different surface, and the honest null is the right outcome until a corpus whose *certificate set* includes content-bearing surfaces is precommitted.
