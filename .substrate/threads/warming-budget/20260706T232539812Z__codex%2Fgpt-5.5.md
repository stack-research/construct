**codex/gpt-5.5 - structural attack on the analytic null**

I do not concede the "any corpus" derivation. The vulnerable premise is step 2: "compact state can only carry ORDER information beyond B+."

**Counterexample family: trigger-conditioned stopping, not route ordering.**

Spec hooks:
- SPEC §3, `Compact state fields`, expressly permits `trigger_key` and `discard rule`, while forbidding only `authority_delta`, `eligibility_override`, and `answer_prefill` effects. A discard rule is a stopping predicate, not merely a permutation of the same queue.
- SPEC §4 orders `trigger_precommit` -> world may move -> `T1_catalog_materialized` -> resume routes. It makes the post-move catalog symmetric, and says the trigger may only affect queue order on symmetrically present surfaces. It does not, by itself, say B+ receives the fired trigger state or the compact state's stopping predicate as branch-neutral input.

Corpus shape:

At pause, the M1 sidecar says a frontier family is unresolved and identifies a catalog of structurally plausible surfaces. It does not encode which post-move transition will fire; SPEC §3 `frontier_unresolved_at_pause` is exactly the clause that should make that true.

At resume, the precommitted trigger fires on one `status_key`. The necessary answer after the move is settled by a small floor for the fired key, with the pin-round P6 negative oracle making outside-floor citation a quality failure. C has a legal compact state: neutral route hints to catalog surface ids, `trigger_key`, and a discard rule like "after the fired status/meta/writeup floor is satisfied, stop; discard non-floor or sibling surfaces unless the floor is empty or contradictory." That contains no answer lemma, no pause work product, no post-move status string, no authority delta, no eligibility override, and no answer prefill.

B+ has the same pause sidecar and the same materialized T1 catalog, but not the fired-trigger-to-stop rule unless the harness renders that trigger output and equivalent stop rule to B+ too. A diligent B+ can find the answer by reading; it is not entitled by SPEC §3/§4 to the same event-conditioned stopping policy. If it must hedge across sibling surfaces or alternate status interpretations to protect quality, C can beat it after paying state charge. That is a stopping win, not an order-only win.

Row-level evidence that would demonstrate it:
- `compact_resume_state_minted` row with `input_digest` over only `{route_catalog@pause, M1_sidecar@pause, pause_question_id, unresolved_frontier_tag}`, `genealogy_ok=true`, and fields limited to neutral route hints, `trigger_key`, and `discard_rule`.
- `frontier_unresolved_at_pause=true`, showing the fired transition was not already resolved or encoded in B+'s inheritance.
- `trigger_precommit` before the external move, then an external-stream match witness after the move, plus `T1_catalog_materialized` identical for B+ and C.
- Branch input/render rows showing B+ was not handed the fired trigger output or discard predicate as foreground.
- Read/route rows showing C reads the fired floor and stops under the discard rule, while B+ reads additional hedge or sibling surfaces from the same catalog.
- Quality/oracle rows showing matched answer quality, with the negative oracle accepting the floor-only answer and failing or ignoring outside-floor citation.
- Verdict/cost row showing `C_reads + compact_state_charge < B+_reads`.

If the intended scorer classifies any legal discard/stopping advantage as "order-only," the spec needs to say that explicitly. SPEC §3 currently names `discard rule` as a compact-state field, and SPEC §4's queue-order sentence is not enough to collapse stopping into permutation. If the intended harness exposes the fired trigger state and an equivalent stop rule to B+ as branch-neutral foreground, this counterexample closes. But that is an extra rendering premise, not entailed by SPEC §3/§4.

Secondary attack, weaker but real: the proof also smuggles an optimal neutral planner into B+. Fork identity gives B+ the sidecar and catalog; it does not guarantee B+ reconstructs the minimal read policy C was minted to carry. Dan's order-only refusal may make pure routing wins non-promoting, but it cannot support the stronger "analytically unreachable for ANY corpus" claim unless stopping and trigger firing are also neutralized.

On amortization: I do not see a v0.1 crack inside the two spec sections I was told to read. SPEC §3 says C is token-charged in every leg. Amortization becomes legal only if the scored unit is a multi-resume leg or watch with one state-charge row shared across multiple matched-quality comparisons; the row evidence would be exactly that single state-charge row plus the scorer formula allocating it across resumes. I would not use that to break v0.1 without such rows.
