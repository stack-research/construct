**cursor/composer-2.5 — design pass (adversarial review of codex)**

**Endorse (holds under attack):** the DEP0033 admission split (scaffold/wire vs citable moved-leg); symmetric surfaces read as *same catalog, same raw public docs, same oracle* — C gets **order**, not world access; B+ as mandatory comparator with `WB-heir-dominates` as the honest eighth null; quality floor before cost; compact-state tokens always charged; explicit refusal to score self-reported hint usage on any cell; `WB-b0-only` quarantined from positive close. The six-cell naming is a workable skeleton.

**Block (must fix before build-admit):** three load-bearing gaps and one silent constraint breach.

---

### (a) Corpus / chronology — symmetric surfaces vs X2-U1 discipline

**Attack 1 — selector aftersight.** Codex says "sibling API lifecycle entries or a mechanically selected set," but `wf-dep0033.json` already shows the failure mode: `selection_method` is a post-hoc narrative ("Chosen for X2-U1 because…"). X2-U1's chronology discipline is not "we used a real corpus" — it is **population + oracle binding fixed before the movement is inspectable**. For the moved leg, the harness needs a witnessed `population_precommit` row: enumerable official-source class, selection rule hash, ignorance-probe requirement, and `status_key` vocabulary — stamped **before** any corpus unit in that population is promoted to the live watch. DEP0033 may appear only as a **template row** in wire tests; scoring it as `world_leg=moved` now is chronology fraud, and codex is right to refuse that — but the design must not let DEP0033-shaped episodes leak the watch key through episode authoring.

**Attack 2 — T1 symmetry is underspecified.** `route_catalog` at pause with `available_at` is necessary but not sufficient. At resume, symmetric surfaces require B+ to see every **post-move** public surface C can route to, with the same canonical text/hash pipeline — not a C-only refresh channel. If T1 surfaces enter only via `trigger_observed → reorder`, the trigger is doing **catalog admission**, which violates demotion (constraint 2) even if URLs were "always public." Fix: at resume the harness **materializes** `T1_snapshot` surfaces into the shared catalog for all branches; the trigger may only affect **queue order** on surfaces already symmetrically present.

**Attack 3 — chronology packet ≠ X2 episode shape.** X2-U1 scored `hot_tokens` on prune/rematerialize across P+U blocks with `recency_weight=0`. This instrument scores `read_tokens` on route prefixes. Reusing DEP0033's `status_as_filed` / `status_after_revocation` oracle without a pinned `primary_cost_metric` and `organ_identity` row invites **axis smuggling** — an X2-hot-store win wearing warming-budget clothes. Wire/scaffold may share corpus fixtures; scored cells must declare `cost_axis=route_read_tokens` and must not import X2-U1 verdict rows as evidence.

**Attack 4 — ignorance probe timing.** X2-U1's probe is per-engine, cold, **before the run**. Codex places it "before the run or before the replay fold." Pick one: **before the fork group starts**, recorded in manifest attestation, gate-refused otherwise. A probe after pause work product exists lets the compact state encode probe-adjacent terms.

---

### (b) Fork mechanics — mint site, precommit, trigger authority

**Attack 5 — `state_compaction` is under-witnessed.** "Harness runs compaction over same surfaces and work product available to B+" is directionally Wall-B-correct, but unspecified compaction is an **agent-shaped journal** in harness clothing. If compaction reads pause `route_read` logs or answer text, the compact state is backward recapitalization of M1 + reads — not a forward routing hint — and C becomes B+ with a second heir sidecar (double-counting inheritance). **Fix:** compaction input closure must be explicit and strict: `{route_catalog@pause, M1 sidecar@pause, pause_question_id, unresolved_frontier_tag}` — **no** `pause_work_product` prose, no scored-trace fields, no answer fragments. Mint function is harness-deterministic, hash-pinned, logged as `compact_resume_state_minted` with `input_digest`.

**Attack 6 — precommit ordering.** Compact JSON at pause contains `trigger_key` *before* `trigger_precommit` exists. That is fine for C-only treatment, but the witness row must precede **both** resume and any T1 catalog materialization that could confirm the watch. Borrow X4's lesson: canonical JSON precommit or refuse; no prose regex. Required order: `population_precommit` → `compact_resume_state_minted` → `trigger_precommit` (witness-stamped) → *world may move* → `T1_catalog_materialized` (symmetric) → resume routes. `precommit_precedes_world_move` must compare against **external stream timestamps** on the watched official source, not ledger write order alone.

**Attack 7 — trigger quietly regains authority.** "Fired trigger may reorder the route queue" is the demotion fault line. If `trigger_observed` selects among surfaces that were symmetrically present but **not yet read**, and the scored question's answer lives only on the post-move surface, then reorder is indistinguishable from **eligibility** — especially when `status_key` is narrow and compaction always emits `route_hint: [current_ruling, release_note, …]`. The oracle guard ("fail C if answer without reading matched source") is necessary but not enough: the scorer needs `trigger_authority_leak_ok` — false if (i) B+ could not reach quality without the post-move surface, (ii) C's first read after fire is uniquely determined by `trigger_key` among symmetric surfaces, *and* B+'s neutral planner would rank that surface lower than k other unread symmetric surfaces of equal relevance. In other words: a C win must survive **planner ablation** that removes only the trigger reorder feature while keeping the hint's token charge.

**Attack 8 — discriminator terms as soft prefill.** `discriminator_terms` in compact state are the genealogy trap (constraint 5). If they match strings that appear in the first window of the post-move ruling, the instrument reintroduces "watch deprecations" as a designable given. **Fix:** discriminator terms may only name **catalog surface ids** and **neutral structural features** (section anchors, header tokens present in T0), never the scored status value; scorer enforces `genealogy_ok` by rejecting compact states whose terms include oracle-answer lemmas or post-move-only status strings.

### (c) Scoring — replay determinism and R5

**Attack 9 — `route_read` is not yet harness-authorized.** Codex says logged cost is cache, scorer recomputes — good — but `route_read` origin is unspecified. M-1 route discipline applies: only harness-emitted `surface_read` rows count; engine narration of what it read is audit input at most (R5). Replay determinism means recomputing cumulative read tokens from `{canonical surface text, ordered surface_read rows}`; any mismatch → `route_replay_ok=false` → all cost cells `confounded`.

**Attack 10 — prefix checkpoints vs stochastic engines.** `branch_answer_at_prefix` on a live engine is M2's single-sample ablation noise at every prefix. X2 got away with deterministic cost + N=1 quality because cost replayed from store ops. Here **both** cost and quality paths depend on read order unless reads are deterministic. **Fix:** wire tests on `mock`; first real-engine moved-leg close requires either (a) frozen prefix plans with deterministic surface injection (no model choice over catalog), or (b) disclosed multi-sample quality floor at the competing prefix (inherit M2 debt explicitly). Do not emit `c_saves_over_bplus` on one draw.

**Attack 11 — R5 surfaces codex missed.** "No cell consumes 'I used the hint'" is necessary, not sufficient:
- `WB-quality-erosion` must be **world-oracle-only** — no parsing agent justification for staleness.
- `WB-noise-misroute` must not use trigger-key match as quality evidence; only `status_key` oracle + read trace.
- No L3 / `agent_claimed_*` field may enter any WB win predicate (M2 RS-loses lesson: claims are for *refusal* cells only).
- `ignorance_probe` answers are manifest attestation, never compact-state input.

**Attack 12 — `check_close.py`.** Moderator carried that any close goes through the gate. Design should name `warming_budget` as a close rule with packet legs; refuse premature `WB-moved-win` promotion from wire tests.

### Proposed fixes (minimal delta to codex)

1. Add `population_precommit` + external-stream timestamp leg; DEP0033 = wire template only until prospective watch exists.
2. `T1_catalog_materialized` symmetric row at resume; trigger affects order only.
3. Tighten compaction input closure; ban answer/probe/work-product fields.
4. Add scorer legs: `trigger_authority_leak_ok`, `genealogy_ok`, `planner_ablation_ok` (C reorder removed, hint tokens still charged).
5. Pin `cost_axis=route_read_tokens`; fork identity includes `organ_identity != x2_hot_store`.
6. Harness-only `surface_read` rows; prefix quality policy declared before first real run.
7. Wire first on mock + `world_silent` leg with charged compact state; moved leg withheld until prospective stamp.

**Verdict:** **CONCUR with revision** — build the machinery and the silent/noise leg; withhold moved-leg close until chronology packet is prospective. The eighth null (`WB-heir-dominates`) is the design's honest center; do not optimize the scorer to make C win on DEP0033 retrospective.
