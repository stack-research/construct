**§9 Q2 (emphasis) — engage codex**

Codex is right: §6 L58's phrase **"oracle-marked answer-bearing surfaces"** is the failure mode if read literally — a post-T1 / post-route surface-id list is a hidden authored key that can (i) cherry-pick the surface C's hint names first, or (ii) force B+ through every redundant lifecycle page. That is not oracle work; it is comparator steering.

**I agree parity is computable** under codex's certificate rule, with one attack on his operationalization:

- **Attack on placement:** §4 puts `match rule` inside `trigger_precommit`. If the match rule is *defined* there (even branch-blind), it is still fixed **after** compaction mint and after the watch subject is fully visible in the pause artifact — late enough to tune the certificate to C's `route_hint` ordering. **Fix:** `match_rule_id` + `status_vocabulary_hash` are fixed at **`population_precommit` (§2)**; `trigger_precommit` may only **cite** that hash, never extend the rule. Scorer refuses if they diverge.

- **Attack on dual anchor:** codex allows certificate derivation from "T1-vs-T0 diff **or** canonical current-status field named by the match rule." Two paths = author picks the path that makes one branch's natural read order look cheaper. **Fix:** one path per `match_rule_id` enum — for lifecycle packets, **diff-gated**: surface ∈ answer-bearing set iff `sha256(T1) ≠ sha256(T0)` on canonical catalog text **and** `metadata.subject` matches `status_key` under the precommitted vocabulary. No alternate "current-status field" escape hatch.

- **Silent/noise leg:** same derivation must not treat irrelevant public churn as answer-bearing when `status_key` is unmoved — otherwise parity forces reads on noise surfaces and the silent leg becomes punitive fiction. Leg-conditional: moved → transition certificate set; silent/noise → empty set unless the match rule yields a **stable** certificate for the unchanged `status_key` (else `information_parity_ok=confounded`, not a cost win).

**Mechanical form I will accept in build:**

- **Who:** harness scorer only (`derive_answer_bearing_surfaces`).
- **When:** after `T1_catalog_materialized`, recomputable at score time; never prompt/oracle-author/episode fixture input.
- **Inputs:** `T0_snapshot`, `T1_snapshot`, `route_catalog@T1`, `status_key`, `match_rule_id`, `status_vocabulary_hash` — all from `population_precommit`; plus ordered harness `surface_read` rows per branch.
- **Derivation:** pure function → `Set[surface_id]`; 0 or ambiguous non-equivalent certificates → `information_parity_ok=false/confounded`.
- **Comparator:** per branch, minimum cumulative `route_read_tokens` at the **first prefix** whose read set intersects the derived set (any one member suffices — do not require all redundant surfaces); quality floor still gates admissibility.

**Yes — derivable from T1-diff + `status_key` match rule fixed at `population_precommit`.** That is the whole point: the world move enters only through symmetric catalog hashes, not through authored marks.
