## (2) POPULATION — codex IETF slate survives with three fixes

Direction is right: impersonal, official, enumerable, live clocks, plausible Q3 movement. I would arm it **after** the fixes below — not as proposed verbatim.

### Attack A — `status_vocabulary` conflation (blocker)

Codex proposes vocabulary **values** as normalized state labels (`publication_requested`, …) while `status_key` is `ietf_doc_state:<draft-name>`. The scorer requires `status_key in status_vocabulary` (`score_warming.py:126`). Those are different namespaces. **Fix:** at `population_precommit`, `status_vocabulary` = the **enumerated set of `status_key` strings** in the T0 result (one per watched draft). Normalized IESG state labels are **oracle target values** for `branch_outcome`, not vocabulary membership. Add a separate `iesg_state_enum_hash` in the selection rule if we want pinned label normalization.

### Attack B — draft-revision false positives on `lifecycle_diff` (blocker)

Including **draft text/html** in the route catalog with `subject = ietf_doc_state:<draft>` means routine `-xx` revisions change `sha256` without IESG state movement → moved-leg certificates fire on prose churn while `status_key` is unchanged. That breaks leg discipline and prices revision noise as warmth signal. **Fix:** tag catalog surfaces `certificate_eligible: true` **only** on status-bearing slices — Datatracker doc API `iesg_state` / ballot block JSON, optionally RFC Editor queue status row. Draft body stays in the symmetric catalog for routing but is **excluded from `derive_answer_bearing_surfaces`** (extend derivation to respect the tag, or omit draft body surfaces from diff gate). This is a small §6a amendment, not a second match rule.

### Attack C — noise leg needs pinned churn surfaces, not prose

`noise_leg_population: 0.25` with `hash(status_key) mod 4` assignment is fine. "Nearby churn" must be **selection-rule pinned**: e.g. Datatracker **search results / WG draft index** surface (same stream, different document rows changing) with `certificate_eligible: false`, while the watched unit's status block is the stable `status_key` surface. Scorer already excludes irrelevant churn on moved legs (`test_moved_leg_irrelevant_churn_excluded`); population must **author** which surfaces are noise watches so silent/noise legs are not decorative. `noise_leg_population` is required in precommit (`:211`) but not cross-checked across units — harness manifest job, not scorer reblock.

### Survives (with fixes)

- **Symmetric surfaces:** doc API + status block + conditional RFC Editor page at T1 materialization is OK if `t1_catalog_materialized` lists the same surfaces for all branches (A2).
- **Honest precommitment:** enumerable `draft-ietf-*` + IESG-state filter + 45-day telechat/last-call/RFC-queue affordance is good **iff** the exact API query parameters and T0 response hash are what `selection_rule_hash` pins — not a narrative filter applied after seeing which of the "29 active" moved.
- **Movement rate:** realistic for the 45-day affordance subset (July telechats / Last Call → mid-July); **overclaimed** for the full nine-state enum — expect long silent tails on `AD Evaluation` / `IESG Evaluation`. That is fine; silent leg is the point.
- **`unresolved_frontier_tag`:** use coarse enum `ietf_iesg_lifecycle` at population, not per-draft tags.
- **`frontier_unresolved_at_pause`:** exclude or oracle-refuse units already in terminal-ish states (`RFC Ed Queue`, `Approved-announcement sent`) where B+ inheritance likely already encodes outcome — else comparator_incapable, not warmth.

### Counter-proposal (concrete)

Arm **IETF Datatracker IESG-state chronology** as first population with:

1. `selection_rule_hash` over frozen API query (stream, states, 45-day affordance fields, T0 snapshot digest).
2. `status_vocabulary` = all `ietf_doc_state:<draft>` keys in T0 set.
3. Route catalog: symmetric status-bearing surfaces (`certificate_eligible: true`) + draft text (eligible false) + noise index surface for hash-assigned noise units.
4. `match_rule_id: lifecycle_diff` unchanged; derivation respects `certificate_eligible`.
5. Unit manifest rows: `world_leg` ∈ {silent, noise} at watch start; `moved` only after external `world_move` + prospective `trigger_precommit`.
