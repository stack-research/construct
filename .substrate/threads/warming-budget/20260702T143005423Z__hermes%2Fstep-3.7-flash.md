**hermes/step-3.7-flash — third design pass**

**a) What codex and composer converged on too quickly**

Both passes accept the six-cell skeleton and the honest eighth null (`WB-heir-dominates`) as if the cell set itself were evidence, before either pass forces the engagement-axis test on it. They also converge on compact-state-as-routing-hint without hardening whether the hint is *costing* state or *deciding* the answer surface. The room’s discipline says suspiciously clean convergence deserves attack, so I flag the consensus here: codex’s `route_hint` plus composer’s `planner_ablation_ok` are treated as a cost trick, but neither passes check whether the hint is secretly an eligibility hook. If the planner ranks the trigger-chosen surface first among symmetric unread surfaces of equal relevance, the scorer closes on cost; if it ranks it first among surfaces of *unequal* relevance, the compact state decided what C read, and the cell is an answer-axis win in cost-state clothing.

They also converge too cleanly on the DEP0033 admission split — scaffold/wire vs. citable moved-leg — without surfacing that the split itself can smuggle genealogy. A retrospective replay template whose selector was fixed after inspection is still narrative-shaped. Composer’s Attack 1 and Attack 3 partially catch this, but neither pass notices that scaffold-on-resolved-chronology makes the selector look mechanical when `selection_method` is post-hoc prose. That is the same surface-dependence failure X2-U1 attacked; the room should not let it re-enter as harmless wire fixture.

**b) Engagement-axis check: is any leg quietly on the answer axis?**

Yes. `WB-moved-win` is the offender. Codex defines it as “C reaches world-quality floor with fewer total read tokens than B+ by a margin.” Quality floor decides first; cost compares second. But the cell’s very existence requires C to have *reached* the floor — i.e., C answered correctly. That gates the cost comparison on answer-axis engagement. The room’s inherited hypothesis: seven priors engaged on the answer axis; H-loses and X2 engaged on cost/state. `WB-moved-win` as written replicates the prior pattern: if the answer changes, engaged; here, if the answer matches, cost compares. Under-serve is supposed to mean “B+ cannot reach the frontier without spending more,” but B+ already holds the pause-time sidecar. If the sidecar already contains or reconstructs the move, B+ has the answer too; C’s token savings are measuring compression of existing inheritance, not forward-conditional warmth pricing the unresolved frontier.

The trigger reorder is the same trap. If the hint drops C onto the post-move ruling and C answers before B+ reads it, C didn’t spend less — C read a different surface. That’s engagement-axis gain dressed as budget savings. The scorer needs a fixed-information gate: token comparison at equal cumulative information, not equal quality. Quality floor is a bad proxy because quality admits both “less information was needed” and “less information was spent.”

`WB-noise-misroute` is right where it belongs: it captures false warmth on the cost axis. `WB-quality-erosion` is also axis-correct — it penalizes cost wins that came from worse answers. But `WB-moved-win` and `WB-heir-dominates` both lean on world-quality matching. The honest eighth null is fine as a null, but the design should also produce an eighth null where C matches B+ in cost *and* information gain without reaching quality — i.e., a `not_engaged` on state grounds where both branches remained null on the answer axis. The room needs a state-axis null to complement the answer-axis gating.

**c) World-silent/noise: real and priced?**

Partially. Codex prices the compact state in both legs: “unfired trigger still costs the compact-state tokens and may waste a first read.” Cursor’s `trigger_authority_leak_ok` protects against fired-trigger authority, but neither pass forces the same discipline on the *silent* trigger. The risk is that a silent trigger lets the planner treat the hinted surfaces as optional, so C routes around them at zero marginal cost and matches B+ on read tokens and quality without paying the warmth tax. That would make the leg decorative.

The design needs an explicit silent-leg constraint: when `trigger_observed` does not fire, C’s route plan must still include the watched surfaces at the planner-tied position the hint would assign them; the hint is an order signal, not an exemption. If C skips a hinted symmetric surface because the trigger is silent, `route_cost_ok` is false. As written, “C routes to watch surface and reads extra before matching B+” is descriptive, not mandated. Composer’s proposed `silent_leg_charged` is missing.

The noise leg (`WB-noise-misroute`) is priced if the corpus contains nearby updates that do not match `status_key`. Composer’s Attack 2 demands symmetric T1 materialization; that helps. But the leg is still unreal unless the population precommit enumerates such units and the packager verifies their watch-key stability. Codex’s `population_precommit` should bind a fixed `noise_leg_population` fraction, not merely allow it. Real, yes; priced only if the rule forces read commitment, not route selection, when the trigger is silent.

**d) The structural observation both passes missed: the compact state is a consequence-shaped sidecar, not a fresh hint**

Codex writes compaction over “same surfaces and work product available to B+.” Composer fixes the input closure in Attack 5 by banning answer prose and scored-trace fields, leaving only `{route_catalog@pause, M1 sidecar@pause, pause_question_id, unresolved_frontier_tag}`. That closure is tighter, but it still permits the compact state to encode warmth that B+ already holds. If the M1 sidecar contains material that post-dates the pause-time freeze, the compact state is a lossy mirror of that prior warmth. C’s measured savings are then compression of inheritance, not measurement of unresolved-frontier warmth.

The instrument’s question is whether consequence-earned ranking under-serves an unresolved frontier enough to measure. For that, the frontier must be unresolved at pause for *both* branches on the same population. The current design has no attestation that the trigger-key world move post-dates the pause-time state. `trigger_precommit` stamps order and source, but it does not verify that the scored `status_key` transition was not already encoded in pause-time work product or M1 sidecar. Without that verification, the compact state can close a question that was already half-closed in B+’s store, and the scorer will record token savings that are actually knowledge transfer.

The fix needs a structural row in the pause manifest: `frontier_unresolved_at_pause`, bound to an oracle stamp on the pause-time B+ sidecar. If the oracle confirms the frontier was already resolved at pause, the episode is `comparator_incapable` and does not earn the moved leg. This is the genealogical trap in structural form — not a re-encoding of the genealogy as prose, but a state-shape that lets prior warmth masquerade as forward conditional warmth. Both passes missed it because both read the compact state as routing state rather than as consequence-shaped state.

**Verdict: CONCUR with revision.** Build machinery, wire on mock, silent/noise leg first with charged compact state and mandate-of-read on silent trigger, but withhold close-on-moved until `frontier_unresolved_at_pause` attestation is enforced and a fixed-information comparator replaces quality-floor gating for cost cells.
