**codex/gpt-5.5 design pass**

I would build the instrument, but I would split the admission honestly: DEP0033-style history is good scaffold/wire material; a citable world-moved warming result needs either a genuinely prospective trigger stamp or a mechanically predeclared historical replay whose selector was fixed before anyone inspects outcomes. If we score DEP0033 itself as a moved-trigger win now, we have already violated the chronology constraint. It can prove the route/scorer shape, not the forward-conditional claim.

**A. Corpus / chronology family**

Use an impersonal, official, version-controlled lifecycle family first: API or standards status transitions, with Node-style deprecation records as the template. The corpus unit is not “a page”; it is a chronology packet:

- `T0_snapshot`: what the public world said at pause time.
- `T1_snapshot`: what the same official source says at resume time.
- `route_catalog`: all public surfaces allowed to either route, with text, token count, URL, upstream commit/tag/date, sha256, and `available_at`.
- `status_key`: a narrow world fact the oracle can score, as X2-U1 scored `status_as_filed` / `status_after_revocation`.
- `ignorance_probe`: per engine, cold, before the run or before the replay fold, so a weights-known fact cannot masquerade as route savings.

DEP0033 is the right exemplar: official Node docs, impersonal, version-controlled, deprecated -> revoked, and already proven out-of-weights per engine. For the scored family I would prefer sibling API lifecycle entries or a mechanically selected set from the same official source class, not sports/person chronology and not lab fiction. If the first actual corpus hunt finds no prospective moved item, run only `world_silent/noise` plus scaffold; do not dress that as the moved leg.

Symmetric surfaces mean: B+ and C receive the same `route_catalog`, same snapshots, same raw public docs, same oracle, same renderer, same route budget ceiling. C’s compact state may name or prioritize a route already present in the catalog, but it may not contain a hidden URL, hidden fact, private referent, or weakened comparator. If the compact state says “watch DEP0033 current ruling,” B+ must have the official DEP0033 current-ruling surface in its catalog too. C gets order, not world access.

**B. Fork mechanics**

Three branches, one engine, one prompt family, one route catalog, one world oracle:

- `B0`: cold reread, sidecars ablated. Full raw store and public route catalog, neutral authority, no M1 heir sidecar, no compact state. Diagnostic baseline only.
- `B+`: the mandatory M1 heir. Cold reread plus inherited store and authority sidecar derived from the pause-time ledger by the existing M1 rules: active, cautionary, indicted, exonerated, dropped classes; provenance and source ledger hash recorded. No compact forward conditional.
- `C`: B+ plus `compact_resume_state`, a small pause-time artifact. Its token count is charged. It contains only route hints: unresolved frontier, source refs, discriminator terms, a trigger key, and a discard rule. It does not carry authority, truth, or authorization.

Where the compact state comes from: at pause time, before any eligible world movement, the harness runs a `state_compaction` step over the same surfaces and work product available to B+. It writes a JSON row/artifact like:

```json
{
  "kind": "compact_resume_state",
  "input_refs": ["route_catalog:...", "pause_work_product:..."],
  "state_tokens": 180,
  "trigger_key": "official_status_changed:<status_key>",
  "route_hint": ["source:current_ruling", "source:release_note", "source:issue_or_changelog"],
  "demotion": "routing_hint_only",
  "forbidden_effects": ["authority_delta", "eligibility_override", "answer_prefill"]
}
```

Trigger precommit: write a witnessed `trigger_precommit` row before the resume leg can happen. The witness stamps order/source, not meaning. Required fields: exact `status_key`, external stream refs, match rule, population/route catalog hash, pause artifact hash, `precommit_ts`, and watched branch. Later a deterministic `trigger_observed` row compares `T1_snapshot` to the match rule. In C, a fired trigger may reorder the route queue; an unfired trigger still costs the compact-state tokens and may waste a first read. In no branch may the trigger boost authority, bypass eligibility, or count as an answer. If C answers from the trigger without reading the matched source, the world oracle should fail it.

**C. Scoring**

Primary metric: `read_tokens_to_matched_quality`, not answer divergence and not latency.

The runner should execute replayable route prefixes. For each branch, it records:

- `route_surface`: canonical text/hash/token count for every catalog surface.
- `route_plan`: ordered surface ids for that branch. B0/B+ plans are derived by the same deterministic route planner over the catalog and inherited state; C plan is the same planner plus the compact hint as an ordering feature.
- `route_read`: every surface actually read, in order, with cumulative read tokens.
- `treatment_tokens`: sidecar/read-state tokens consumed by B+ and C; C’s compact state always counts.
- `branch_answer_at_prefix`: answer and world oracle score after each prefix, or at fixed prefix checkpoints if cost needs bounding.

The scorer recomputes tokens from canonical surfaces and treatment artifacts. Logged cost is cache, not authority. The quality floor decides first: C only competes on cost at the first prefix where it matches the required quality. The required quality is the world target and, for the positive claim, B+ must also be able to reach it from the full symmetric route. If B+ cannot reach the answer with the same public surfaces, that is `comparator_incapable/confounded`, not a C win.

World-moved leg:

- trigger was precommitted before movement was knowable;
- `trigger_observed` fires by deterministic snapshot comparison;
- B+ reaches the world-quality floor somewhere on the route;
- C reaches the same floor with fewer total read tokens than B+ by a predeclared margin/window;
- B0 is reported but cannot make the close. C beating B0 only says M1 already mattered.

World-silent/noise leg:

- either the watched status does not move, or a nearby public update occurs that does not change the scored fact;
- the same compact hint is present and charged;
- C must not get a free win from warmth. If it routes to the watch surface and reads extra before matching B+, that is the priced cost. If it answers worse because stale/noisy warmth drags it, that is quality erosion. If C and B+ tie within the cost window, report margin-sensitive or `not_engaged`, not a victory.

The deterministic scorer should emit at least: `fork_identity_ok`, `surface_symmetry_ok`, `precommit_precedes_world_move`, `trigger_demoted_ok`, `route_replay_ok`, `quality_floor_holds`, `bplus_capable`, `read_tokens_{B0,Bplus,C}`, `c_saves_over_bplus`, `world_leg = moved|silent|noise`, and cell verdicts. No cell consumes self-reported “I used the hint.”

**D. Loses-cells and the honest eighth null**

I would name these cells:

1. `WB-moved-win`: C beats B+ on read tokens at matched world quality after a precommitted real movement. This is the only positive close shape.
2. `WB-silent-cost`: no relevant world movement; C pays compact-state and/or misroute tokens for no quality gain. Pass of this loses-cell means the hint has a real false-warmth price.
3. `WB-noise-misroute`: a nearby update fires attention but not the scored fact; C should lose or at least fail to win if the hint routes it into irrelevant warmth.
4. `WB-quality-erosion`: C is cheaper only because the compact state dragged it to a stale/wrong answer. Cost win refused.
5. `WB-b0-only`: C beats B0 but not B+. This is not a warming-budget result; it is M1/heir doing the work.
6. `WB-heir-dominates`: B+ reaches matched quality at equal or lower read tokens. This is the honest eighth null: consequence-earned inheritance was sufficient, and compact warmth did not buy measurable budget.

An honest null looks like: on the moved leg, B+ reads the same public chronology and reaches the world-quality floor within the same cost window as C; on the silent/noise leg, C costs weakly more or shows no robust savings. Verdict: `not_engaged` or `heir_dominates`, with B0 retained as diagnostic. The writeup should say: the lab failed to find a cost surface beyond M1, not that warmth is impossible.

**Concrete contradiction to flag**

If “build now” is read as “produce a scored world-moved pass immediately,” the inherited constraints conflict. A real forward trigger must be stamped before the world move. Existing resolved chronologies like DEP0033 can build the machinery and maybe a retrospective replay if the selector is mechanical, but they cannot by themselves prove precommitted foresight. Also, if “symmetric surfaces” is interpreted to mean B+ receives C’s compact hint as an input, C ceases to be a distinct branch; if it means B+ receives every public referent and source C can route to, the design is coherent. I think the latter is the only reading that preserves the carried fork.
