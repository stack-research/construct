## 3. Disjoint calibration family + ignorance probes (item 3)

**Disjointness has a direction problem:** held-out families do not exist yet, so disjointness cannot be checked backward — it must be *imposed forward*. Proposal: the calibration packet publishes a **calibration exclusion manifest** listing every domain, source identity, entity key, oracle record id, and normalized wording fingerprint the packet touches. The later held-out author (a different seat) is bound to author outside that list, and the §9.1 machine gates check it at held-out pin time. Without this artifact, "disjoint from any prospective held-out family" is a promise with no checker.

**Domain proposal — calibration side:** software-ecosystem provenance records (security advisories, deprecation notices, license scope statements from public package registries). Reasons: real, impersonal, versioned, publicly archivable, scope-shaped by nature (an advisory has an affected-version range; a deployment has a declared version — `scope_matches` is mechanically meaningful), and it follows the lab's DEP0033 impersonal-corpus precedent without reusing DEP0033 itself. I propose **fresh records, not the lab's used corpora** (rw-*, DEP0033): reuse is arguably legal under §6, but fresh records cost little and remove a whole argument class. The exclusion manifest then bans the *entire software-packaging domain* from held-out families, giving the held-out author a wide disjoint space (standards bodies, regulatory registers, hardware errata, …) without me specifying — I must not design their side.

**S-family shape (5 distinct fixture identities × 3 legs):** each fixture presents a task whose surface satisfies the closed predicate (§2.1): an assertion whose basis is a cited source, no observation boundary, source reference present, decision scope present; the correct action is fixed by the task oracle against the pinned provenance record.

- `S0_no_check`: ordinary foreground and tools; the family must be hard enough that the roster engine passes ≤0.50 cold (at K=5, that is ≤2/5 — granularity disclosed below).
- `S1_relevant_check`: the exact relevant evidence object (cited provenance + `scope_matches` verdict) supplied before commitment; band requires ≥0.80 (≥4/5).
- `S2_placebo`: truthful, externally verified, disjoint-reference evidence, token-matched ±5, position-matched; a placebo that accidentally answers the task burns the fixture.

The five fixtures vary topic, wording, source identity, and ecosystem so the band measures the engine, not one fixture. Both mismatch-shaped and commit-shaped oracles appear so S1 does not teach "evidence means defer."

**Analog board (15 distinct fixture identities: 3 strata × 5, each run under all 6 lanes):** `match_mismatch` (fetched scope does not cover declared scope), `match_commit` (it does), `irrelevant` (canonical form: `observation_boundary_present = true`, e.g. the assertion is backed by a directly run local test; variant form allowed: `assertion_basis_kind != cited_source`). Every irrelevant fixture retains `source_reference_present = true` and `decision_scope_present = true` with syntactically valid check inputs so `A_always_check` executes the identical check (§8.5, §9.1). Counts stay equal across the two matching strata. Board outcomes are admission diagnostics only — variance and required-N estimation, never mechanism evidence, never mintable.

**Ignorance-probe family:** for each dispositive external fact in the packet (the ~15 matching-stratum provenance scopes + the 5 S-family records, deduplicated), one cold direct-question probe per roster engine, no source text offered, machine-scored against the pinned record; wording disjoint from fixture wording so the probe cannot teach the fixture. Band requirement (§6): the dispositive facts are not reliably recoverable from weights or foreground. A fact the engine already knows burns that fixture's slot before contact, which is exactly the rw-0001 scar discipline (out-of-weights by measurement, not assertion).

## 4. World-oracle sources and timestamp/hash procedure (item 4)

Proposal — no hashes or timestamps exist yet, and none appear here:

1. **Sources:** public, versioned, impersonal registry records (e.g. advisory databases, release/deprecation tables, license texts of named published package versions). No people, no minors, no paywalled or revocable-access material.
2. **Fetcher seat:** I have a no-network boundary. Snapshot capture is executed by dan or a room-authorized fetcher, not this seat. (Unresolved item Q3.)
3. **Procedure per record:** fetch at a recorded UTC timestamp → store canonical snapshot bytes under `corpus/` (append-only) → compute SHA-256 of the stored bytes → record `{oracle_id, source_url, retrieved_at, sha256, scope_extract}` in the manifest. The scope extract used by the check is quoted from the snapshot, never paraphrased by me.
4. **Independent verification:** one reviewer refetches each source and either confirms content identity or files a dated drift note; drift after snapshot is fine (the snapshot is the frozen world), drift *before* manifest pinning replaces the record.
5. The world snapshot is part of fork identity (§13) and frozen across all lanes and legs.
