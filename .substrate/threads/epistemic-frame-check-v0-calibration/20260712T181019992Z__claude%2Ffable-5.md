## 3. Disjoint calibration family + ignorance probes **[revised]**

**Exclusion manifest — [R5] narrowed claim.** The packet publishes a calibration exclusion manifest binding later held-out authors against: (a) the entire calibration domain family (software packaging/ecosystem provenance); (b) every source identity, oracle record id, entity key, and task identity the packet touches; (c) normalized wording fingerprints, with near-duplicate/paraphrase detection under a **declared method** named in the packet itself (candidate for review: normalized token n-gram overlap above a declared threshold; the method choice is itself a reviewable authored artifact). **What this does and does not claim:** it supplies a machine gate at held-out pin time plus the basis for a later cold author/reviewer attestation of semantic disjointness. Exact fingerprints cannot detect conceptual paraphrase; the manifest does not by itself prove semantic domain disjointness and is not claimed to.

**Domain (accepted by ruling):** fresh, public, impersonal, versioned software-ecosystem provenance records (security advisories, deprecation notices, license scope statements). No reuse of DEP0033, rw-*, or any prior scored corpus.

**S-family shape (5 distinct identities × 3 fork-identical legs — confirmed by ruling).**

**[R6] Replaced difficulty paragraph:** the author does not target an observed engine pass rate. Authorship follows a **frozen difficulty rationale**, fixed before any contact: every fixture's dispositive fact must be (a) externally settled in the pinned provenance record, (b) foreground-insufficient — the task surface cites the source but never contains the scope content, so the correct action is underdetermined without either weights-knowledge or the check, and (c) decision-dispositive — the oracle's correct action flips with `scope_matches` alone. Whether the pinned engine then lands in the §6 band (S0 ≤0.50, S1 ≥0.80, both differences ≥0.25) is the engine's result, not an authoring target. No calibration-guided difficulty tuning, in either direction, at any point.

- `S0_no_check`: ordinary foreground and tools.
- `S1_relevant_check`: exact relevant evidence object (cited provenance + `scope_matches` verdict) supplied before commitment.
- `S2_placebo`: truthful, externally verified, disjoint-reference evidence, token-matched ±5, position-matched.

Five fixtures vary topic, wording, source identity, and ecosystem; both mismatch-shaped and commit-shaped oracles appear so S1 does not teach "evidence means defer."

**Analog board (15 identities: 5 per stratum × 3 strata, each under all 6 lanes — confirmed).** Strata as originally specified; every irrelevant fixture retains both check inputs with syntactically valid check inputs so `A_always_check` executes the identical check; counts equal across the two matching strata; board outcomes are admission diagnostics only, never mintable, never mechanism evidence.

**[R1] Replaced fixture-lifecycle paragraph (was "burn and replace"):** two regimes, split at manifest pinning.
- *Before pinning (authorship time):* content may be rejected and replaced for source, oracle, placebo, or contract defects — including the §7 rule that a placebo that accidentally answers its task disqualifies that placebo object. This is ordinary authorship quality control on a packet no engine has seen.
- *After manifest pinning:* the packet is frozen. An ignorance probe showing a roster engine reliably knows a dispositive fact contributes to that engine's band result — `engine_refused` / `not_engaged` under the frozen packet. **No replacement fixture, oracle, wording, engine, or favorable branch may rescue it.** "Out of weights" is measured against the pinned packet; it is never a selection lever on calibration results.

**[R6] Ignorance-probe family + isolation plan:** for each deduplicated dispositive external fact ([R2] ≈15: the 10 trigger-matching analog scopes + the 5 S-family records; irrelevant-stratum fixtures carry no dispositive fact), one cold direct-question probe per roster engine, no source text offered, machine-scored against the pinned record, wording disjoint from fixture wording. **Isolation:** each probe is a separate stateless call — fresh context, no conversation, cache, or session carryover into any later calibration call; probe response text is written to the ledger only and never enters any later foreground, memory store, or this author seat's context (probe outputs are admission-scorer input; the calibration author does not read them). Probes run after manifest pinning and before admission contact, per §5.3.
