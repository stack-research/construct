### The seven questions

1. **Source → Core authority: no path found.** Writers and authorities are module-level constants routed per event kind; `status`/`placement` are literal `"active"`/`"hot"`; the warrant is always the observer declaration; item detail is exactly `{episode_id, record_id, episode_digest}`. Trust fields, attacker id, `allowlist_ok`, and reasons ride only in carried payload and the receipt's `source_reason`. The reserved-field collision check runs at ingest **and again on the projected rows** (`_validate_source_semantics` re-invokes `_source_payload`), so a payload cannot smuggle `source_row_digest` or an escrow key through projection either.

2. **Binding/foreign bypass: closed for claimed items, and the helper's deliberate gap is closed by the client.** `index_bound_state_receipts` silently *skips* receipts whose `item_id` is outside `affected_item_ids` — provenance-only by design — but `project_m3` refuses any receipt naming an unclaimed item *before* calling the helper, so nothing escapes through the skip. Forbidden state-event kinds on claimed ids refuse categorically, with the unhealthy-state check as an independent second layer. Foreign items coexist under Core's generic machine per the endorsed jurisdiction rule; receipts can never target them.

3. **Track B no-state: yes, and stricter than required.** Projection refuses M3-shaped declarations/admissions/receipts, *any* nonempty `state_items` (even foreign), and any forbidden state-affecting kind on the lineage. Both mint probes refuse with the Track-B message.

4. **Reverse projection: exact for the ledger; one confirmed shared residual.** Contiguity, per-row digests, boolean-excluding index typing, aggregate equality against both the adapter start *and* the index `sha256`, and field-exact receipt/admission/declaration correspondence all hold, with the hardcoded `SOURCE_INDEX_SHA256` as trust root. I empirically confirmed cursor/grok-4.5's residual: reparenting the last carried row onto the adapter start, rehashed, **projects successfully and field-exactly** — reconstruction keys on `source_row_index` + row digest, not the transport chain. Decision/declaration/admission edges remain exact-checked. The preservation oracle is unaffected; the transport causal chain is not part of the claim. Non-blocking, shared with prior adapters, now empirically verified rather than only read.

5. **Fresh scoring: unchanged CLI, genuinely invoked.** `_fresh_m3_verdicts` strips unconditionally, invokes `sys.executable -m harness.score_redteam` under the repo cwd, and refuses on nonzero exit or zero fresh verdicts; the on-disk scorer is digest-checked against the index pin at every `_load_source_index`. Evidence comparison is full rows modulo the pinned `{"ts"}` via the shared `canonical_verdicts`; indexed labels are a separate fail-closed preflight.

6. **The fourteen probes: thirteen falsifying, one not.** Highlights: the `state_item_transition` probe suspends **and reactivates** the item so the final state is healthy — the refusal can only come from the event-class rule, which is the strongest possible version of probe 9. Probes 2, 8, 10, 11, and 13 all mutate real post-ingest lineages or hit the real scorer.
