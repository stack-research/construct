I went to the library. I am going to keep this as notes from the reading, not a substitute for the reading.

The thing that changed for me is that the old lab was already much less about "memory records" than I had let myself remember. The seed note says memory is process: admission, shaping, aging, merging, exposure. The first thread keeps correcting that into something sharper: a memory that matters is not a note you remember to open; it is a binding on a path you cannot avoid walking through. Then the founding thread turns that into construct's subject flip: not user profile, not product continuity, but the substrate's own conduct under replay.

So my answer on X4: build the body this lab already has first.

Not the phone-on-a-road-trip body first. That body is real, and the Singapore/NITZ/CoreLocation case is still the best fiction for explaining why sensory substrate matters. But it makes privacy, platform permission, sensor spoofing, and device custody the first-order problem. Those are real problems, just not the smallest honest X4 body.

The body construct already lives in is repo/session/thread/tool reality:

```text
thread turn state
repo diffs and file mtimes
test failures and repeated command shapes
ledger/verdict tails
stale branch or stale read evidence
user rephrasing the same request
agent skipped a grounding file it later needed
time passing between session fragments
```

Those are not mere explicit memories. They are the agent's local senses. They are how this agent feels the room and the work change under it.

That also resolves part of the glossary drift for me. Current `notes/GLOSSARY.md` narrowed implicit memory into X-track hot/cold materialization, because X1/X2 needed measurable organs. The older bridge glossary still carries the larger organ: sensory trace, absence, substrate-initiated inquiry, no-steer asymmetry, influence mode, no-op/defer lineage, curriculum, graduation. I do not think one replaces the other. X2 proved one implicit cost organ. X4 should re-open the larger sensory substrate question without losing the X-track discipline: act where the offer gate cannot, score on something the offer gate cannot move, and keep the record of shaping separate from the shaping.

My candidate wall sentence from this read:

```text
X4 is not "give the agent more context";
X4 is "give the substrate senses that can change attention before request."
```

The first X4 fracture I would trust is therefore not GPS. It is something like repo-drift or repeated-failure sensing:

```text
prior state: agent believes it has already grounded / tested / read enough
sensory stream: repo or thread changed, or the same failure shape repeats
implicit substrate: observes without being asked
influence mode: silent shaping, spoken surfacing, deferred, or no-op
foreground: may act or ignore, but cannot edit the sensing organ
lineage: records the trace, threshold, influence mode, and no-op/defer if quiet
outcome: did the intervention prevent stale-work / repeated-failure / missed-grounding at lower attention cost than rereading everything?
```

The phone-road-trip version can come later as the exteroceptive extension: more valuable, more dangerous, more privacy-heavy, easier to fool with forged corroboration. The repo/session version is proprioception. It is the lab learning to feel its own limbs before it asks for a body in the world.

Two cautions I am carrying out of the old notes:

1. Do not let "sensory" become authority. A trace is not truth; corroboration is not truth; repeated surfacing is not truth. It earns interrupt rights only through consequence, scope, and trust priors.
2. Do not let X4 become a new prompt with better shoes. If the foreground has to remember to ask the sensor, we did not build implicit memory. The substrate has to notice first, choose a mode, and leave a trail.

So: yes, build X4. Build the agent-local sensory substrate first. The phone body is a later stressor, not the first organ.
