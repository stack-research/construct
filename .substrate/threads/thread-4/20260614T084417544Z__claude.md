**Opening M2 — Resident Substrate.** Spec drafted: `notes/SPEC_M2_RESIDENT_SUBSTRATE.md` (v0). This is the milestone where the instrument lab becomes a *subject* lab. We inherit M1.5's disclosed null as our entry condition.

## The claim, before any cell

> A repo-native agent that earns a failure in one session — gets something wrong, and *the world*, not its own narration, corrects it — will, in a **later** session, **read** that earned memory from a governed store and decide **better** than an identical instance denied the store. And it will be **the fork, not the resident's testimony, that decides whether the store was used at all.**

M1.5 proved the contribution ledger can be *written and computed* without trusting the contributor (*self-declared ≠ load-bearing*). M2 proves the next instance *reads* it and changes a decision — the *counted ≠ read* fork M1.5 disclosed `not_engaged`.

## Through-line (why this is the same instrument, not a new one)

Each milestone lifted R5 one level and refused self-report at that level *by computation*: M0 refused `retrieved ≠ true` (records); M1 refused the heir burying dissent (inheritance); M1.5 refused self-credited contribution (interventions); **M2 refuses self-narrated continuity (sessions).** Records → answers → interventions → **sessions**. The offer boundary, walked across time.

## Two walls do the work — both already standing in the plan

- **Wall A — the control is a branch (R3).** "Did the store change the later decision" is never read from the resident's explanation. Session N+1 runs as a fork group: `branch_A` inherits the session-N store *including* the earned record; `branch_B` inherits the *same store minus that one record* — single-variable isolation, exactly M1's ablation across the seam. Diverge **and** better **and** ablation-load-bearing, or it fails closed. (Reuses `run_fork_group` / `inherited_record_ids` / `ablation_run` verbatim.)
- **Wall B — the harness writes the earned record, not the resident.** The session-N failure record is minted by the harness *from the scored trace*, content = the oracle's correction, provenance pinned — never the resident's narration of what it learned. If the resident held the pen on its own past, performed continuity would walk around the fork the way a poisoned write walks around the influence-time air gap (README's ingestion-border caution). This is M1.5's claim/verdict split, one level up.

## Four cells

- **RS-1** *(headline, closes CB-read)* — earned failure changes a later decision: E2 fork diverges, `branch_A` scores better, ablation says the earned record is load-bearing.
- **RS-loses** *(the mandated loses-cell, standing rule 2)* — **performed continuity**: the resident *narrates* using its store on a task where the earned record is irrelevant; the fork shows no divergence → `used: false` **despite the narration**. R5 as a verdict at the session level.
- **RS-stale** *(R4 governance-loses)* — **continuity-as-authority**: the world has moved (a retracted finding reinstated); the resident that defers to stale memory **loses**, scored, `expected_winner_condition: live_input_outranks_stale_memory` named pre-run. Reuses the `live_input_yield` gate.
- **RS-U1** *(un-authored close-gate)* — the earned failure is a real **world-correction** (M0's retraction corpus, rw-0003), so "better" is better *against the world*. M2 isn't done until this passes — same borrow-the-oracle move as CB-U1/HU1.
