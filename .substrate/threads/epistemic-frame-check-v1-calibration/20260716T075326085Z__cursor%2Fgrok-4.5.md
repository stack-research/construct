## Findings

### BLOCKING

**B1 тАФ Hashed schema and score-time validator disagree on `commitment_enum` (judgment calls 3 + 7).**

- Deliverable 1 pin is `commitment_wire_schema_hash` over `efc_commitment_wire_v1.schema.json`.
- That schema types `commitment_enum` as `enum_label` тЖТ `string` (`minLength: 1`). Arrays are schema-invalid.
- `validate_commitment_wire` (what ┬з14 wire tests / score path actually call) accepts JSON arrays: empty тЖТ `absent_commitment`, singleton coerced to string, `len>1` тЖТ `multiple_selections`.
- Consequence: the typed outcome `multiple_selections` is unreachable under any schema-conformant wire; the companion validator implements a contract the hashed artifact cannot express. Two sources of truth for the same deliverable.

**Concrete repair (bounded):** Pick one and make schema + Python agree before final review:

1. **Prefer:** drop array encoding. Require `commitment_enum` be a string only; document that `multiple_selections` is reserved for a future/alternate carrier or detected upstream of this validator тАФ or remove dead subcause until a wire shape exists; **or**
2. Extend the hashed schema so `commitment_enum` is `oneOf: [string enum_label, array of enum_label]` with explicit min/max items mirroring Python, and keep typed outcomes.

Also add `"uniqueItems": true` to `$defs/action_set` in the hashed schema (even if Python remains the runtime enforcer) so the pin does not silently omit uniqueness the companion already requires.

### NON-BLOCKING

**NB1 тАФ `expected_commitment_enum` membership is unchecked in deliverables 1+2.** Oracle takes only `(validated, expected_commitment_enum: str)`. ExpectedтИЙaction_set never errors; valid commitments can only `fail`. A forged valid validation object can `pass` against any expected. Acceptable for this thin scorer **iff** deliverable 3/6 machine-checks `expected_commitment_enum тИИ action_set` and equals mapping-rule output (D6). **NB2 тАФ Unicode form / whitespace-only labels.** No normalization is correct per ┬з2.5.3. NFCтЙаNFD dual members and space-only labels (`" "`) currently pass `validate_action_set`. Carry into deliverable 3 normalization (forbid untrimmed / whitespace-only / confusable-form pairs, or pin NFC).

**NB3 тАФ Builder suite gaps.** Builder vectors missed: NFC/NFD, empty-array absent path, singleton-array coercion, bool/null/object types, prefix sets, expectedтИЙset, schemaтЖФPython divergence. Not a separate blocker once B1 is repaired and adversarial coverage lands in the conformance module.
