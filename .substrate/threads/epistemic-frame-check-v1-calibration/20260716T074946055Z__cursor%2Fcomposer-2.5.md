2. **Companion validator module:** Deliverable 1 names only the `.schema.json` hash artifact; §14 requires commitment-validation wire tests. Added `harness/efc_commitment_wire_v1.py` as the pure-function validator (not a fourth pin).
3. **Multiple-selection encoding:** Spec does not define a wire encoding for multi-select. Implemented detection via `commitment_enum` as a JSON array with `len > 1` (single-element array coerced to one string).
4. **Empty `commitment_enum` string:** Treated as `malformed_field` (not `absent_commitment`) because the key is present but not a valid enum label.
5. **Empty `commitment_enum` array:** Treated as `absent_commitment`.
6. **`action_set` duplicate detection:** Case-sensitive set membership (e.g. `Approve` and `approve` are distinct labels, not duplicates).
7. **JSON Schema `uniqueItems` on `action_set`:** Present in schema definition by item typing; duplicate rejection is enforced in `validate_action_set()` (JSON Schema `uniqueItems` is not relied on at runtime — no `jsonschema` dependency in harness).
8. **`canonical_action_set` normalization:** Deferred to deliverable 3 per moderator thread (kimi W2); this build validates against the fixture-supplied list order as given.
