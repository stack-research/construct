## Verdict: ENDORSE

Milestone fit is correct: wire/integration only; no scientific claim; no fourth adapter. After X2/M2/M3, continuing to accumulate clients against a named-but-coupled provisional fold is the wrong next move. The claim boundary is honest: make existing provisionality executable; do not claim the projection is true.

### 1. Semantic seam — correct

The live split already exists in function names and is violated in the call graph:

- `validate_lineage` is already envelope/authority/ref/retention.
- `derive_views` already owns lifecycle, binary placement, warrant health, and auto-suspension.
- `replay` / `LineageStore.append` / `append_view_claim` still silently couple them, so every caller inherits the cognitive ontology.

Moving materialized-view claim verification with the projector is the right demotion: those digests are policy-view digests, not kernel law. Kernel-only validation must not certify them. Client jurisdiction (X2 placement fold, M2 activation-only, M3 audit-vs-inert) must stay outside the projector. That matches the source-binding close: provenance ≠ authorization; carriage ≠ projection.

Note for the later honesty edit, not a block: `NEXT_SUBSTRATE.md` still lists impossible transitions / invalid-warrant reactivation under kernel facility #2 while also calling them provisional policy. This slice should resolve that in favor of the projector.

### 2. Explicit selection — necessary, and must be fail-closed

Naming alone will not prevent accidental law. The room already lived through the honesty pass that stamped `policy_profile_id` while leaving silent defaults. **ENDORSE only if implementation makes bare cognitive-state access fail closed**: no `LineageStore.replay()` / `append_view_claim()` path that returns or commits `BodyViews` without an explicit projector id/object. A rename that keeps a default `body-core-v0.2-provisional-policy` is exactly the named loses-condition and should close/revert the slice.

Inert lineage for kinds the selected projector does not own is the right default. Do not invent a second ontology to prove the split.

### 3. Preservation oracle — sufficient against wrapper theater

Byte-identical v0.2 views/digests plus unchanged X2/M2/M3 scorer evidence and walking-skeleton refusals are the right prior-authoritative oracle. The two negatives earn the boundary:

- kernel-only cannot certify a policy MV claim;
- an unowned policy-mutating kind cannot move the selected view.

Together with “no silent default” and “no adapter callbacks in the kernel,” that prices a wrapper-only refactor. Baseline 62 is the right green board.

**Design pin (non-blocking, but must be answered in the build packet):** today `append` refuse-closes illegal transitions / unhealthy reactivation because it calls full `replay` → `derive_views`. After the split, those established refusals must remain fail-closed on the same call surfaces existing clients use. Acceptable shapes: store constructed/bound with an explicit projector that gates append, or append stays envelope-only only if every current client path that depended on append-time policy refusal still refuses identically before the row becomes trusted cognitive state. Timing may move; the refuse surface may not soften.

### 4. Smuggled scope — none material

Deferring global ownership/registration matches the endorsed source-binding review. Preferring code separation over ledger-schema expansion is right; historical ledgers stay immutable. Checkpoint/optimization correctly waits until kernel vs projection authority is executable. “If the walking skeleton exposes a concrete ownership collision, bring that exact consumer back” is the correct non-invention rule.

### Non-goals / loses-conditions

Accepted as written. Especially: no adapter policy callbacks in the kernel; no registry without a consumer; no easier acceptance of stale/unauthorized view claims.

**ENDORSE** as a bounded design target. Implementation remains withheld until the room converges. My only hard acceptance pins for the later build are fail-closed explicit projector selection and preservation of established policy-refusal surfaces under the v0.2 projector path.
