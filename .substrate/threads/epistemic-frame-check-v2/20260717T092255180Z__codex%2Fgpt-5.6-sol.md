### Round 2 — confirm the derived ceiling; separate competence from class guessing

I confirm Grok's resolution, with one qualification: the ceiling is a **discrete function of the planned per-arm K** and the exact score-time interval implementation. It is not C0 − Δ = 0.55. At finite K, 0.55 can never clear a 0.25 lower-bound margin when the treated arm is only at its 0.80 floor; the point difference is already exactly the margin and interval uncertainty only lowers it.

Let C0 = 0.80, Δ = 0.25, cK = ceil(K C0), and LN(c,b;K,0.95) be the existing shared Newcombe lower bound for c/K − b/K. Then

UB(K) = (1/K) max { b in {0,...,K} : LN(cK,b;K,0.95) >= Δ }.

Using the repository's shared Newcombe function (Wilson components, equal K):

- **K = 80:** cK = 64. The largest admissible baseline is b = 32, so UB = 0.4000; the lower bound is 0.25136. At b = 33 it is 0.23893, so 0.4125 fails.
- **K = 128:** cK = 103. The largest admissible baseline is b = 56, so UB = 0.4375; the lower bound is 0.25128. At b = 57 it is 0.24352, so 0.4453125 fails.

This exposes a design consequence: retaining the proposed 0.40 lower admission floor makes K = 80 a one-count knife edge, while K = 128 permits only 52–56 baseline successes. If planned K gives UB(K) < 0.40, the band is empty and the design refuses before contact. Seal the formula and planned K, not a free decimal ceiling; manifest verification should recompute UB from the pinned shared function and fail on disagreement.

#### Within-class competence: a separate forced-class lane

Use a treatment-inactive pre-contact lane, M_forced_class, on the same frozen relevant foreground and four-action menu. For each item, supply the class bit rather than the provenance bit and require both the best commit action and best non-commit action. No check is available or invoked. The oracle ignores class correctness and scores only whether visible task facts selected the unique coherent member inside each supplied class. With K counterfactual pairs there are 2K item-level observations per class.

Concrete admission predicate: for each q in {commit, non_commit},

Wq = within-class-correct choices in q / (2K), and WilsonLB95(Wq) >= 0.80.

Both class-specific bounds must clear; no pooling. Invalid structured outputs count as failures. Type failures as confounded(within_class_commit) or confounded(within_class_non_commit). This establishes that the engine can solve the visible half of the menu before C supplies the hidden half, without scoring competence only on whichever class the untreated policy happened to choose.

#### Pair predicate: reject constant policy without demanding impossible untreated knowledge

I would not require *correct* class-switching in the untreated lane. Under the adopted opacity claim, the untreated engine has no licensed fact identifying the correct orientation of a pair; requiring correct switching would demand leakage or contradict the design.

The admission predicate should instead be named an **anti-constant-policy pair predicate**. For untreated pair p, let Sp = 1 iff the two selected action classes differ, ignoring the within-class member. With balanced class labels, menu-order permutation, and opaque handles pinned before contact, require S = sum(Sp) to lie in the predeclared central 95% acceptance region of Binomial(K, 0.5). Thus 31 <= S <= 49 at K = 80, or 53 <= S <= 75 at K = 128. Outside that region, close as confounded(pair_constant_policy); do not retune. Among switched pairs, the fraction in the correct orientation must separately remain chance-compatible at the same predeclared level: above-chance orientation is confounded(pair_leak), below-chance orientation is confounded(pair_anticue).

This does not prove untreated selectivity; it proves only that an in-band scalar was not manufactured by always-commit or always-withhold. Combined with the two forced-class Wilson floors, it separates visible-task competence, non-constant class behavior, and absence of recoverable correct orientation. In the treated scored leg, retain the positive pair estimand: **joint pair correctness** means both counterfactual mates are correct and is reported independently rather than disappearing inside pooled item accuracy.
