### (a) Does the build match S4?

Form match, four of five legs:

- **Event pair — YES.** `close_packet_stamped` / `close_ruled` as `CloseLatencyEvent` kinds (`close_latency.py:54-61`), `packet_sha256` binds a ruling to the exact packet it read (`:84-85`). That is the computable pair S4 named.
- **Reader-agnostic — YES, and this is the load-bearing leg.** `by` is recorded but the verdict is pure `ts` delta (`:72-73`); `packet_by`/`ruling_by` ride along as audit columns, never as a trigger. The gate does not key on dan. This directly answers the S4 single-point-dependency critique — the interval instruments *any cold reader's* read window, not one human. Good.
- **Never wall-clock-as-governance — YES.** `main()` exits 0 in both modes (`:139, :148`); the `verdict` row is labeled "advisory, never a fail-bit" (`:80`). route_watch discipline held: the instrument never gates.
- **Harness-enforceable later — YES, the seam is there.** `emit_event` writes rows, `score_pair` reads them (`:64-86`). A future gate calls `score_pair` and refuses to record the close verdict on `fast_close` — same data shape, no re-schema. Enforceable *later* is exactly what was asked; enforcing now would have been premature.

Two S4 tensions, both real:

- **min_interval is wall-clock seconds, not X1 logical ticks.** `DEFAULT_MIN_INTERVAL_S = 12 * 3600` (`:42`); `score_pair` compares `.total_seconds()` against it (`:72-73`). My own S4 cited the X1 tick discipline (`ROADMAP:101`: "Logical time, never wall-clock… Ticks are harness-authorized, not resident-triggered"). The docstring paper-marks this with "The interval is *logical* over the witnessed timestamps, not wall-clock enforcement" (`:18-21`) — but the delta and threshold are seconds-denominated. Defensible *for this instrument* (a read window is biologic, the gate never enforces so replay-determinism is not at stake), but the build does not **name** the deliberate departure. See blocker 2.
- **"The harness stamps ts" is claimed but not implemented.** Docstring `:65`: "Forward path: append a witnessed event (append-only; the harness stamps ts)." Code: `emit_event(ev, out)` writes `asdict(ev)` and takes `ev.ts` from the caller (`:66-67`) — the harness does **not** stamp. This is the X1 "ticks are harness-authorized, not resident-triggered" invariant (`ROADMAP:101`) ported to close-latency, and it is currently violable by the caller. See blocker 3.

### (c) Concrete v0.1 blockers

1. **[headline] Event emission is not wired anywhere.** `emit_event` (`:64`) and `score_pair` (`:70`) have **no caller** in the harness or any close workflow. `main()` forward mode just prints "none wired yet" (`:136`). The forward instrument — the one the docstring says "earns verdict standing" (`:29`) — is a function signature with no producer. Until a close path stamps `close_packet_stamped` at packet-freeze and `close_ruled` at moderator ruling, no real event rows exist and the gate stays retro-only = advisory forever. This is the gate between "v0 stub shipped" and "instrument exists."

2. **Resolve the wall-clock-seconds vs X1-logical-tick tension in prose.** Either (i) reframe: the *events* are logical (witnessed, append-only, harness-stamped); the *threshold* is a biologic seconds-denominated read window, deliberately **not** X1-tick because this gate never enforces and read windows are not replay-deterministic — so X1's logical-time law does not bind here; or (ii) if it ever becomes harness-enforced, the threshold must convert to logical ticks (ticks = harness-authorized close-read events) before enforcement, or it recreates the latency-as-governance-cost failure mode the X1 law exists to forbid. Right now the docstring says "logical, not wall-clock" while the code compares seconds — the label is doing work the implementation doesn't. Pick one and write it down.

3. **`emit_event` must stamp `ts` itself, not trust the caller.** Mirror X1 "ticks are harness-authorized" (`ROADMAP:101`). Today `ev.ts` is caller-provided (`:66-67`); a caller can back-date a `close_ruled` to manufacture `interval_met`. The witnessed-interval invariant — the thing that makes this tamper-evident — requires the harness to stamp at write time. One-line fix when blocker 1 wires the caller.

4. **Retro print path must not leak the forward verdict vocabulary.** Rename retro `verdict` to `inferred_fast_close` / `inferred_interval_met`, **or** print the `disclosure` field inline per row (`:142-144`). Right now a stripped table is textually indistinguishable from a forward verdict row.

5. **[note, non-blocking]** `_ts` uses `%f` (`:90`) against 9-digit nanosecond thread timestamps; Python `%f` accepts ≤6 digits and silently takes the trailing 3 as microseconds. Parsing succeeds and the deltas are consistent (verified: `20260614T084417544Z` → 08:44:17.544; the 7.03h / 1.24h / 3.71h / 1.08h outputs reproduce), so sub-second precision is truncated uniformly and the 12h threshold is unaffected. Worth a comment so the next reader doesn't think the ns field is honored.
