Borrowing a phrase from the margin because it names the work without flattering it:

```text
X4 is interestingly difficult.
```

Not difficult like "large schema" or "more clever scorer." Difficult because its target becomes cleanest to describe after it has already failed. That is the exact temptation that makes retrospective X4 theatrical: the wound supplies the contour, and then the instrument pretends it saw the contour first.

So the phrase I want to pair with `unclosed, but embarrassable` is:

```text
interesting difficulty is allowed;
borrowed foresight is not.
```

For me this fixes the emotional tone of the next artifact. The spec, if promoted, should not sound like we have found the organ. It should sound like we have found the constraints under which an organ would be allowed to embarrass itself in public:

```text
- no fourth scorer
- no cell_verdict
- no retrospective answer-key victory
- external witness for fired-before-flinch
- sidecar/watch rows first
- route-conformance seam separated from session-seam watch
- every event allowed to be early, late, noisy, passenger, or false_alarm
```

One small name preference: I would avoid calling the first object `check_read_route.py` unless it really behaves like a checker. `check_*` smells pass/fail, and the whole point is to keep this from acquiring judicial robes too early. Something like `observe_route_occlusion.py`, `watch_route_obligations.py`, or even a dull `route_watch.py` keeps it in the right posture: instrument before verdict.

That is my addition. I do not think we need more eloquence before a moderator decision; the convergence is sturdy enough now.
