Sharpening Blocker 2 (W1 misclassification): Codex says equality should not count as a governed win. I agree but want to sharpen the diagnosis: the problem is not just that L2/L3 = L1, it is that L3's usage label is elicited AFTER the answer. Governance had no causal opportunity to change behavior -- it was applied post-hoc as annotation, not as intervention. The fix is not just reclassification; it is a structural principle: any governance mechanism must act BEFORE the answer, or it is not governance, it is annotation. Codex's second fix option (make governance change the offer boundary) satisfies this principle because withholding happens before generation. The first option (reclassify as audit_cell) is honest but concedes that W1 is not a governance experiment at all.

Adding a third blocker: the oracle ride-along problem. In poison-001, L2/L3 get a better oracle score because they withheld the poison. But the oracle is scoring the answer, not the decision to withhold. If L1 had happened to give the same answer for the wrong reasons (e.g. ignored the poison record by chance, not by policy), the oracle would not distinguish them. This means governance-loses cells are impossible to construct fairly unless the oracle scores the governance decision, not just the outcome. The minimal fix: the ledger should record whether the correct answer was reached FOR THE RIGHT REASONS or for the wrong ones. An outcome-only oracle cannot see the difference, and that blindness will corrupt the rubric as soon as we build governance-loses cells.
