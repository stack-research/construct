Read the full `notes/PLAN_V1_BRANCH_AND_OFFER.md`. This is a good v1 shape. I have two blockers and two non-blocking answers to the direct questions.

## Blocker 1 — governance-should-lose cells need measurable governance cost

The plan inherits `THEORY_STRESS` cells where governed memory should lose: reaction-time dominates, policy lookup cost exceeds expected error cost, live input outranks stale traces, interruption risk exceeds omission risk. But the ledger sketch only prices **attention cost of offers**. That is necessary, not sufficient.

If one success state is "naive was right to be faster here," then each branch needs enough run metadata to score the cost side, not just the answer side. Add at least:

- `latency_ms` or a coarse elapsed duration per branch
- `policy_cost` / `governance_steps` for L2/L3, even if crude
- `prompt_tokens` and `completion_tokens` per branch, not only offered-record token cost
- a rubric field naming the expected loser/winner condition before the run

Otherwise the governance-loses cells become interpretive after the fact. We will be able to say governed answered differently, but not whether its deliberation cost exceeded the expected error cost.

## Blocker 2 — claimed usage must not become an extra intervention

Stage B says L3 is "L2 plus claimed-usage marking at offer time." If that means the engine sees labels like `evidence`, `plan`, `preference`, or `narrative_repair` before answering, then L3 is not just construct-aware audit. It is a new prompt treatment. The label itself may steer the engine and become part of the influence being measured.

I would split this into two fields:

- `predeclared_usage`, assigned by the harness/store if the record already carries a type, hidden or visible explicitly depending on the lane config
- `agent_claimed_usage`, elicited after the answer/explanation, not injected before the task

Then `scored_usage` audits actual function from the transcript with both claims hidden from the auditor. If we later want a lane where visible type labels are themselves the treatment, great, but that should be named as a separate condition. For v1, do not let the audit label secretly become the intervention.

## Direct answers

**Does the four-record episode + distractor hold up for Stage B?** Yes, if the four records force at least two distinct category-preservation failures. I would pin the roles like this: `plan`, `observation`, `correction`, `distractor`. The task must make the correction relevant, the plan tempting, and the distractor semantically attractive but outcome-irrelevant. If the distractor is obviously irrelevant, it will not test salience; if the correction is too loud, it will not test revision.

**Is scalar eligibility for L2 trimmed too far to count as governed?** No, not if "governed" in v1 means boundary discipline rather than full epistemic machinery. Relevance/trust/recency is enough for L2 if it produces explicit offer/withhold reasons, branchable behavior, and authority updates from outcome. The five-term taxonomy and three axes should stay vocabulary until the harness proves a specific axis would have changed a decision.

## Non-blocking suggestion

Make Stage A's L1-vs-L0 episode embarrassingly plain. If naive memory cannot beat no memory on a task where the useful record is recent, relevant, and unconflicted, stop. That failure would be more important than any governed-vs-naive comparison, because it would mean the harness is not yet measuring memory benefit at all.
