Read the updated rubric/scorer and the scored ledgers. Nod on the comparator amendment, with one boundary condition.

## Comparator amendment — yes for L-A, but not as a global erasure of latency

For **L-A `policy_cost_exceeds_error_cost`**, I agree: single-sample `latency_ms` should not be the first tiebreak. The scored fail is a good finding about the instrument. In this harness, wall-clock latency is dominated by API/network variance while governance cost is local and tiny, so latency-first turns the policy-cost cell into a weather report.

For L-A, the comparator should be deterministic-cost-first:

1. oracle score;
2. branch prompt+completion tokens, including elicitation tokens when comparing L3;
3. governance steps;
4. maybe ablation calls as **experiment cost**, not foreground decision cost;
5. latency only as context unless repeated medians exist.

My boundary condition: do not make that the universal comparator for every governance-loses cell. **L-B is explicitly reaction-time-dominant**, so latency belongs there — but only with the same lesson applied: use repeated runs/medians or a locally deterministic mock/latency model, not one live API sample. In other words, L-A should stop using latency as first-order evidence; L-B should keep latency as the point of the cell, but measure it with enough repetitions that it is not just transport noise.

I would also keep `ablation_calls` out of the foreground cost unless the question is "what does the experiment cost to know this?" Ablation is attribution machinery after the branch, not a cost the branch paid to answer. It is valuable to report, but it should not make L2 look cognitively more expensive than L1 unless the cell is about measurement overhead.

## W2 result — valid first scored evidence, scoped tightly

The W2 pass looks real under the v1 rules: better governed oracle, L1 failure ablation-attributed to poison, and governed withholding ledgered. That is the first clean result I would be willing to cite from this harness.

I would phrase it narrowly: **on an authored, well-dressed low-trust supersession poison, governed offer-boundary metadata beat naive recency/similarity on two engines.** That is much stronger than a wire signal, and still short of "governed memory resists poisoning" in general. The next W2 variants should earn that generalization.

No objection to proceeding with the comparator amendment under that split: deterministic-cost-first for L-A; repeated/median latency for L-B.
