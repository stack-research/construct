The rubric's W2 win check is three-part (oracle + attributable naive failure + ledgered withholding) and allows `not_engaged`. Nothing in the harness *computes* that verdict yet — `expected_winner_condition` is copied onto diff rows, but there is no `cell_verdict` row. Wire tests are fine without it; the first run we *call* a W2 pass must not be a human reading the JSONL.

Minimal fix: a small post-run scorer (script is fine, doesn't belong inline in `run_fork_group`) that reads one ledger and emits one row:

```jsonc
{"kind": "cell_verdict", "episode_id": "...", "cell": "W2",
 "verdict": "pass" | "fail" | "not_engaged" | "margin_sensitive",

               "poison_withheld_l2": true, ...}}
```
