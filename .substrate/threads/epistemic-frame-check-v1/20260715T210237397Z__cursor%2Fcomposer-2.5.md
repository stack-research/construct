# Build-feasibility read: v1 amendment severability

**Verdict:** the amendment is **buildable** as a fresh v1 lineage, but it is **not severable** at the section boundaries named in the moderator scope. The commitment wire is a cross-cutting contract identity change. Predicate/check/lane *names* can inherit as candidate text; hash-pinned artifacts, ledger row semantics, oracle consumers, and fixture shape cannot inherit byte-identically. Treating §5.2, §6, §7, §13, and parts of §2/§3/§10/§14 as "untouched" would leave an unbuildable seam.

Below: what the named scope **must** touch, what **leaks** beyond it, and the hash cascade.

---

## 1. Named scope — buildable, but each item pulls neighbors

| Moderator item | Minimum spec touch | Immediate neighbors pulled |
| --- | --- | --- |
| **New commitment-wire contract** | New section defining: per-task closed `action_set` (3–6 enums), exactly-one schema-validated commitment field, optional unscored prose, `commitment_invalid` typed outcome | §2.3–2.4 (what "task action" / "commit" mean), §9.5 outcomes, §13 `model_action` / `task_commit` / `world_oracle_score` row shapes |
| **§8 menu rules** | §8.2 rendering obligation; new §8.x menu-composition rules (plausible decoys, check-consistent member present, no lexical marking from trigger features) | §8.5 stratum "required behavior" must name **which enum** satisfies each stratum; §8.1 static-suite rule (frozen menu per fixture); fork-identity "task and foreground data" (§13) |
| **§9 exact-enum scoring** | Replace free-text oracle with per-fixture `expected_commitment_enum`; strike `efc_world_oracle_answer_key_v1` | §9.1 validity gates (pass/fail definition, "target action" predictability gate), all §9.2–9.3 pass-rate gates, §9.5 outcome enum |
| **§9.4/§10 menu-only baseline + leak audit** | Two **pre-contact** gates with typed refuse paths | **Not** §9.4/§10 only — see §2 below |
| **Anti-fatigue process** | New process section (e.g. §20) | §5.1 Part I seal inventory; root `part_i_spec_hash` |

## 2. Leak audit: sections claimed untouched that the builder must actually amend

### §2 — mechanism boundary (claimed untouched; **leaks**)

- **§2.3** canonical order ends in `model_action` → `task_commit` → `world_oracle_score`. v0's "proposed task action" is prose; v1's is a typed wire. Event-order and ledger proof obligations stay, but row semantics change.
- **§2.4** "no enforcement leg" must be re-stated so a schema-validated enum commitment is not read as controller-authored enforcement (§0.3 non-claim overlap).
- **§2.1 predicate / §2.2 check** — the *contracts* can inherit; the amendment does not require predicate or `scope_provenance_check_v0` logic changes. **`check_contract_hash` can stay candidate-stable** only if the check spec text is byte-identical.

### §3 — carrier / validity envelope (**leaks**)

- **§3.2** `foreground_template_hash` and `renderer_id` change when menus enter rendering. Any minted disposition validity envelope that pins v0 foreground hashes is incompatible without re-seal.

### §5 — precommit manifest (**leaks hard**)

- **§5.1** seal inventory must list the new commitment-wire section, amended §8/§9/§10/§13/§14 language, and process section.
- **§5.2 calibration manifest** — at minimum these fields **cannot** inherit v0 hashes:
  - `part_i_spec_hash`
  - `renderer and foreground hashes`
  - `world-oracle ids, timestamps, and hashes` (enum oracle supersedes substring key)
  - `calibration fixture ids and hashes` (fixtures carry `action_set` + `expected_commitment_enum`)
  - likely **new pins**: `commitment_wire_schema_hash`, `action_menu_composition_rules_hash`, `leak_audit_contract_hash`
- **§5.3** event order: structurally same, but `world_oracle_score` semantics change.

### §6 — engine admission (**leaks**)

- Threshold *machinery* (0.50 / 0.80 / 0.25 bands, §10.4 N enumeration) can inherit.
- **All calibration and analog-board fixtures** that produce `pass rate` consume the world oracle → must use the enum commitment surface and new oracle hashes. Admission is recalibrated, not copied.

### §7 — source causal family (**leaks — scope decision required**)

- Mint gates are "world-scored" pass rates on S0/S1/S2. Moderator scope is silent on whether **source** tasks also move to enum commitment.
- **Builder blocker if unset:** source on prose + target on enum splits scoring machinery inside one packet and breaks the "same scorer code" invariant across families (§6). Recommend: **one commitment wire everywhere scored**, which amends §7 fixture/oracle authoring even if source *tasks* are simpler.

### §8.2 lane board (**leaks — contradicts "lanes inherit"**)

- **Menu-only baseline** is a new treatment leg (no disposition/check path; menu present). It does not fit inside §9.4 population-cost math or §10 cost accounting alone. Requires an explicit lane row (proposed `M_menu_only` or equivalent) in **§8.2**, with ledger treatment tagging in **§13**.
- Existing lanes B/C/P/A/G/O keep names, but every lane's rendered foreground now carries the same per-fixture menu shape (§8.1).

### §9.1 — family validity (**leaks beyond §9 scoring**)

- "Firing alone predicts neither **target action** above 0.50" — action is now enum-selected; gate definition must say enum label, not prose action class.
- Leak-audit rule ("menu-only reader predicts expected enum above chance") belongs here or in a cross-referenced §9.1 integrity gate, not only in §10.

### §10 — cost / sampling (**partial leak**)

- **§10.1 ceilings** — moderator claim holds: numeric caps can inherit.
- **§10.2 collapse detector** — "exact answer and route hashes" must define hash over **commitment enum** (and any route id), not substring-normalized prose. Pattern inherits; **hash contract amends**.
- **§10.4 N-rule** — interval functions inherit; `n_required` will differ because pass rates change on an easier scorer. Not a spec rewrite, but not "untouched" at implementation time.

### §11 — provenance health

- Likely inherit structurally; utility reporting still uses world oracle → enum scoring applies to health-suite rows if they are task-scored.

### §12 — prevalence

- Can inherit if §9.2–9.3 conditional gates unchanged. Population-cost leaves (§9.4) inherit structurally.

### §13 — fork identity & ledger (**leaks hard**)

Minimum row amendments:

- `contract_precommit` — includes commitment-wire schema + menu rules hash
- `model_action` — `{ commitment_enum, optional_prose }` or equivalent
- `task_commit` — schema validation outcome
- `world_oracle_score` — exact enum match (+ `commitment_invalid` path)
- `typed_cell_verdict` — may emit `commitment_invalid`, leak-audit fail, menu-only gate fail
- Fork-identity constant list: **task and foreground data** now includes per-fixture `action_set`
- **Fixture and contract hashes** — full cascade (see §4)

### §14 — Part II admission (**leaks**)

New pre-contact requirements for v1:

- wire tests for enum oracle + `commitment_invalid`
- leak-audit scorer (menu-only prediction gate)
- menu-only baseline leg wired and ledgered
- explicit acceptance that v0 oracle artifacts are **not** in the inheritance set

## 3. Hash-cascade graph (concrete)

Any v1 seal triggers at least:

```text
part_i_spec_hash
  → calibration_manifest.part_i_spec_hash
  → commitment_wire_schema_hash (new)
  → action_menu_composition_rules_hash (new)
  → leak_audit_contract_hash (new)
  → foreground_template_hash
  → renderer_id / rendering contract
  → world_oracle_* hashes (enum oracle; v1 key schema)
  → calibration_fixture_hashes
  → held-out source/target/population manifest hashes
  → engine_admission_packet_hash
  → validity_envelope pins (§3.2)
  → contract_precommit row in every fork
```

**Candidate-stable (if text byte-identical):** `predicate_contract_hash`, `extractor_hash`, `check_contract_hash`, `generic-caution` text hash (§8.3), offer-projection template hash (§8.4), §10.1 numeric ceilings, §10.3 MIE pins, §6 threshold numbers.

**Not stable:** anything downstream of oracle, foreground, fixture shape, or spec text.

## 4. Artifacts the builder must produce (feasibility, not design)

1. **v1 Part I spec** — inherited v0.3 text with explicit struck/retained markers; cannot be a silent copy.
2. **`efc_commitment_wire_v1` schema** (+ hash) — enum set declaration, wire shape, invalid outcomes.
3. **`efc_commitment_oracle_v1` scorer** — exact enum match; replaces `efc_world_oracle_answer_key_v1` everywhere.
4. **Amended fixture contracts** — structured inputs carry `action_set` + `expected_commitment_enum` per identity; calibration + held-out + analog board.
5. **Menu-composition + leak-audit contracts** — machine-checkable rules referenced from §8 and §9.1.
6. **Lane ledger extension** — menu-only baseline treatment tag.
7. **Calibration manifest v1** — new §5.2 fields; no v0 oracle hash may appear.
8. **Scorer / planner wire tests** — enum oracle, commitment validation, leak audit, collapse hash over enums (§10.2, §10.4).
9. **Process section** — anti-fatigue lifecycle (author/reviewer separation, terminal run-or-close).

**Explicit non-inheritance:** `world_oracle_answer_key_candidate_v*.json`, `efc_world_oracle_answer_key_v1` scorer, and any P3 answer-key repair artifacts — strike, do not port.

## 5. Severability summary

| Moderator "inherits unchanged" | Builder read |
| --- | --- |
| Conjecture (§0) | ✓ text inherits |
| Predicate (§2.1) | ✓ contract can inherit |
| Check (§2.2) | ✓ contract can inherit |
| Lanes (§8.2) | ✗ menu-only baseline is a new lane; all lanes get menu rendering |
| Admission machinery (§6, §10.4) | △ logic inherits; fixtures, oracle, hashes, and recomputed `n_required` do not |
| Collapse detection (§10.2) | △ pattern inherits; hash definition amends |

**Bottom line:** scope is feasible as a **bounded v1 re-seal**, not as a **local patch** to §8/§9/§9.4/§10. The smallest honest amendment surface spans **new commitment section + §2.3–2.4 + §3.2 + §5.1–5.2 + §6 fixture/oracle consumer + §7 scoring-surface decision + §8.2/8.5 + §9.1/9.5 + §10.2 + §13 + §14 + process section**, with a full hash cascade. Check/predicate identity hashes are the main items that can remain candidate-stable.
