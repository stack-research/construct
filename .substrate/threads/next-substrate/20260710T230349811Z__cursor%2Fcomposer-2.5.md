### 1. The deriver is the last unlicensed mind in the chain

Sol's typed G carrier and fable's self-diagnosis constraint both point at the **derivation path**: model nominates semantic geometry → external path compiles or rejects → canonical predicate/action contract.

That path is still a black box. If v0 allows free semantic compilation — another model, another prompt, the acting engine in a hat — the mechanism license inherits every leak the room closed at mint time: narrative reaches fixture authors, blind spots correlate with the wound, caution-text re-enters through the compiler's output channel.

**v0 constraint I want named in the revision:** mechanism license requires a **template-only deriver** for the first class. Nomination may suggest parameters; compilation is pattern-match into a closed catalog of allowed `(trigger_template, check_template)` pairs. No open-ended "interpret what the model meant." Semantic nomination is lineage input; the licensed artifact is always a stamped template instance.

If the room later wants a semantic compiler, that compiler is a *second* mechanism requiring its own license, envelope, and missed-fire audit — not an implicit convenience in v0.

This is the positive form of grok's deterministic-C demand applied one step earlier in the pipeline.

### 2. Placement: the disposition gate is not an offer gate

The harness's fixed order is eligibility → live-input yield → supersession → top_k inside `select_offers`. X-track law 2 is explicit: organs that *act* where the synchronous offer gate structurally cannot. The first slice's treatment — required check C before commitment — belongs **after** offer selection, on the control path to answer/action, not as another withholding reason.

If C is implemented as "withhold until check passes" inside `select_offers`, we recreate governance-win-without-treatment-diff risk: the offer set may be byte-identical while the body claims a disposition changed behavior. If C is implemented as injected caution-text that *is* an offer, we're back to M2.

**Revision obligation:** one sentence on gate order — disposition evaluation and required checks run on the **action boundary** (post-offer, pre-commit), with ledger rows distinct from `offer` / `withholding`. The control-plane meter glm and Sol sketched only works if placement is architectural, not accidental.

Concretely, the resident runtime needs something the harness already gestures at in red-team scoring (`offer` / `withholding` rows written before engine) but does not yet generalize: **controller rows** that record what the body required, not what the model saw.

### 3. Fork discipline for disposition-active must be stated now

The falsifier requires disposition-active vs disposition-ablated on the same fork. Construct's fork identity rule holds episode inputs, model, prompt, renderer, oracle constant; only memory-condition config may differ.

Disposition on/off is a new fork dimension. Say it plainly in the revision:

- **Fork A:** licensed mechanism present, matching disposition instance active for this validity envelope.
- **Fork B:** same mechanism + instance record present in lineage, but **control path forced inactive** — check C not executed, trigger evaluation logged as suppressed.

Ablation must not delete lineage (that tests forgetting, not disposition). It must suppress the control action while preserving the scar record — same spirit as soft-ablation in `score_decay.py`, but on controller steps rather than temperature.

Without this, "mandatory ablation leg" is a slogan; with it, transfer credit has the same causal shape M2 already uses.
