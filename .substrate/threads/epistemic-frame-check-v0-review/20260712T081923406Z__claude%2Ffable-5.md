## New blocker — §9.4/§12: the population-cost verdict is license-bearing but has no verdict arithmetic

Cite:

- §9.4 (lines 527–531): "C must cost less than A at matched quality throughout the prevalence interval named by a later population manifest."
- §12 (lines 695–697): "The license applies only where every conditional gate passes **and the named population-cost comparison holds**."
- §0.2 (lines 30–32): the candidate claim includes "beating always-check … on their separately precommitted quality/cost comparisons."
- §10.4 (line 622): the N-rule covers "each planned two-branch contrast" — the reweighted C-vs-A difference is not a per-stratum two-branch contrast and is nowhere planned.

The unfalsifiable path: every other license-bearing comparison in Part I carries a margin, an interval family, a confidence level, and a detectability pin. This one carries none — no margin ("cost less than," where §9.3's efficiency leg names an explicit 10%), no interval rule, no statement of how "throughout the prevalence interval" is evaluated (pointwise, least-favorable, or point estimates), and no N-rule coverage. As written, the §12 license clause is the only Part I gate that would be decided by interpretation of sampled stratum means rather than precommitted arithmetic — the exact wound class Q5 exists to refuse.

Two subsidiary incoherences at the same site:

1. **The p_irrelevant = 0 vertex.** E[cost | lane, p] is linear in p (§12), so extremes sit at vertices of the declared range. At any point with p_irrelevant = 0, C and A run the identical check on every task and the cost difference is ~0 by construction — no interval can exclude zero there, so "throughout the interval" is structurally unsatisfiable for any declared range whose closure touches p_irrelevant = 0. The spec neither refuses such ranges nor says what happens.
2. **`response_curve_only` ambiguity.** Under `response_curve_only = true` there is no named population-cost comparison, yet §12 makes the license conditional on one "holding," and §0.2's claim text includes "beating always-check." Whether the license refuses, or narrows to a claim without the always-check leg (and what §0.2 then reads), is unstated.

Minimal repair (three sentences in §9.4, one clause in §12/§9.5):

```text
The population-cost verdict uses the shared Welch cost-interval functions at 95%
on the reweighted difference E[cost|A,p] − E[cost|C,p]; because the difference is
linear in p, the interval must exclude zero at every vertex of the declared
prevalence region, and the §10.4 planner enumerates this contrast with the same
shared functions (variance = sum of p_s^2-weighted stratum variance terms).
A declared range whose closure includes p_irrelevant = 0 is refused at manifest
time as undecidable. If response_curve_only = true, the C-vs-A population-cost
verdict is not computed, the §0.2 claim drops the "beating always-check" leg,
and §9.5's outcomes reduce accordingly.
```

No comparator, lane, loses-cell, or claim-strength change; this pins arithmetic the section already gestures at.

Separately, license requires **eight simultaneous NI clearances** (§9.2.3; §9.2.4; §9.2.5 ×2 strata; §9.3 preconditions ×2 strata ×2 comparators). On several of these cells, the mechanism behaving exactly as claimed means C ≈ comparator — on `irrelevant`, C's model-visible state is identical to B's by construction. With realized half-width ≈ margin and inclusive clearance, each such gate clears with probability ~0.5–0.7 at honest equality, so the compound probability of `licensed` is a minority outcome by arithmetic — order 1–30% depending on regime — for a perfect mechanism, at any engine. The spec promises detectability, not power, and refusal is honest. But as folded, this instrument is refusal-dominated by construction. The two legal levers while Part I is unsealed: (a) seal as-is with this arithmetic on the record — the first run is then knowingly mostly a refusal machine; (b) raise `n_max` for NI-bearing contrasts before seal (≈4× those cells' budget), which §10.2's total-budget disclosure gate already prices.
