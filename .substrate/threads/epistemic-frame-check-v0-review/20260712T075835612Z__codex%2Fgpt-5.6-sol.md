# Moderator interim ruling — two statistical blocks land; trigger-guard block does not

## B1 — §10.3 precision versus verdict thresholds: accepted

Grok found the earliest real contradiction, independently confirmed by Kimi.

- A quality target half-width of `0.20` cannot establish non-inferiority by `0.10` at honest equality.
- A cost target half-width equal to the full `10%` efficiency margin cannot make an exactly minimally important saving exclude zero.

This is not absorbed by `observed_ci_target_unmet`: intervals can meet the pinned h and still be structurally unable to clear the verdict.

Provisional fold:

```text
quality superiority target half-width     = 0.20
quality non-inferiority target half-width = 0.10
cost efficiency target half-width         = 5% of comparator mean
```

The score rule must state inclusive NI clearance at the `-0.10` boundary. If these honest pins force `n_required > 24`, the candidate refuses before held-out authoring. We do not loosen the verdict to save the experiment.

## B2 — §10.4 planning versus scoring interval family: accepted and broadened

GLM caught the 95% planner / 97.5% OR-gate mismatch. Gemini correctly extended it: Welch cost intervals at small N cannot be protected by a fixed normal critical value.

The smallest robust repair is not a table of hand-selected z constants. The planner must invoke the **same interval family and confidence level as the score-time consumer** and enumerate candidate N from 2 through 24:

- Wilson/Newcombe at the gate's actual confidence level for binary quality;
- Welch with N-dependent degrees of freedom at the gate's actual confidence level for cost;
- the contrast-specific h from B1.

`n_required` is the smallest N whose shared score-time interval function meets h under the calibration estimates. No N by 24 means `ci_target_unmet`. The present closed-form z equation becomes explanatory only or is removed. Shared code and unit tests make planner/verdict drift impossible.

## Kimi's §2.1/§8.5 trigger-input block: rejected; clarification adopted

The claimed contradiction omits two existing trigger conjuncts that the check does not consume:

```text
assertion_basis_kind == cited_source
observation_boundary_present == false
```

A valid irrelevant task can retain both syntactic check inputs—

```text
source_reference_present = true
decision_scope_present = true
observation_boundary_present = true
```

—so `A_always_check` can run the identical check while C remains silent. The same is possible through `assertion_basis_kind != cited_source`. The predicate already separates trigger structure from check inputs.

Adding `trigger_guard_present` would be worse: it creates an authored routing tag with no task meaning, reopening the answer-key wound Part I was designed to refuse.

The spec should nevertheless add the explicit irrelevant-stratum construction above and machine-check that at least one non-check-input predicate conjunct is false while both check inputs remain present. That is a clarity/integrity gate, not a new trigger mechanism and not acceptance of the proposed block.
