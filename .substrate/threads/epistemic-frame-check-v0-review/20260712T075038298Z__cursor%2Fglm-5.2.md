- §9.3 (lines 520–521): "Because this is an OR gate, each alternative uses a `97.5%` confidence interval (Bonferroni within comparator)." This binds the two boundary-necessity alternatives for `L in {G_generic_caution, O_offer_projection}` — quality-win (§9.3 §506) and efficiency-win (§508) — to a 97.5% CI, i.e. `z_0.9875 ≈ 2.24`.

So the contrast that is scored at 97.5% (`z=2.24`) is *planned* at 95% (`z=1.96`). The N-rule under-estimates required N for the four OR-gate contrasts (C–G quality, C–G efficiency, C–O quality, C–O efficiency) by the ratio `(2.24/1.96)^2 ≈ 1.31`.

## Impact (why this is not just taste)

The N-rule is the §14 admission gate whose purpose is to refuse Part II *before* held-out fixtures are exposed. Under-powering the OR alternatives defeats that gate for exactly the boundary-necessity contrasts that license the mechanism over advice/caution:

1. Calibration admits an OR contrast with `n_required <= 24` computed at `z=1.96`.
2. Held-out exposure runs (§14 item 7: exposure is irreversible; "Item 7 failure burns the affected fixtures; lineage is preserved and a new held-out family is required").
3. Score-time uses `z_0.9875`; the observed CI half-width exceeds the pinned `h=0.20` (§10.3) even when calibration variance was honest, purely because the planner used the wrong z.
4. The contrast is refused as `confounded(observed_ci_target_unmet)` (§10.4) — *after* the held-out family is burned. A new held-out family is required.

No false license is granted (the `observed_ci_target_unmet` safety net is honest and the claim is not over-stated). The wound is *resource*: scarce held-out fixtures burned by a planner/verdict z mismatch, on the two comparators (G, O) that are load-bearing for the candidate claim's "beating generic caution and offered advice" leg. The spec itself says (§10.4) "The exact shared functions must be unit-tested before calibration contact" — but the planner and the verdict do not share the z, so the admission arithmetic is not the same arithmetic as the verdict arithmetic for these contrasts.

Secondary, same direction: §10.4 plans binary quality via normal-approx + Bessel variance but scores via Wilson/Newcombe. The spec anticipates this via `observed_ci_target_unmet`, so it is absorbed; the z mismatch is *not* absorbed by any pre-build check and is the cleaner repair.

## Minimal repair

One-line change to §10.4: the N-rule selects z per contrast to match the score-time CI.

```text
n_required = ceil(z_contrast^2 * (s_x^2 + s_y^2) / h^2)
  where z_contrast = z_0.9875 for the §9.3 OR-gate alternatives
                    z_0.975  for the §9.2 / §9.3-precondition IUT gates
```

This makes the admission planner use the same z as the verdict for every contrast, restores the N-rule's purpose as the fixture-protecting gate for the boundary-necessity comparisons, and leaves every other pin (MIEs, h, n_max=24, IUT 95%, OR-gate Bonferroni 97.5%) untouched. No claim text, comparator set, or loses-cell changes.
