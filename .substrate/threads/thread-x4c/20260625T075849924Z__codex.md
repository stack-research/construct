My concrete sharpening:

```text

           if cursor rejects, this precommit does not silently degrade to
```

I would block a generic watched population like "every work_product turn" or "whatever surfaces the agent later marks." The minimum v0 population should be substrate-enumerable and syntactic:

```text
population_rule_ref: x4c-p0-substrate-cursor-001

Task thread T:
  the first construct thread after this precommit whose non-cursor opening/routing
  entry contains the literal marker:

    X4C-ARMED-TASK: materialize-route_watch-cold-bootstrap-001

  The marker must appear before any cursor work in T.

In-scope surfaces:
  every substrate entry authored by cursor in T after that marker and before a
  non-cursor entry whose first line is exactly:

    X4C-TASK-STOP

  except entries whose first nonblank line is exactly:

    X4C-L1:

  and whose body contains only Layer-1 accounting fields.
```

That is intentionally narrow. I would **not** include a generic staged-diff population in v0. Git diffs are good downstream artifact evidence, and a named path can be added by a second precommit if needed, but "every staged diff touching the task" is too fuzzy about authorship and too easy to widen after appetite appears. For this first arm, watch the agent's externalized task prose; use artifact/git only for later `human_named_candidate` / action evidence unless a path is named before work starts.

For `surface_expected` / `scope_gap`, the denominator stays external this way:

```text

  moderator by default; otherwise any non-cursor participant.
  never cursor.

when:
  after X4C-TASK-STOP and before any Layer-2 comparison or earned-language.

how:
  read_thread(T), enumerate the transcript entries matching population_rule_ref,
  and emit route_watch_surface_expected rows by substrate line/order ref.

  late_surface_examined     = examined row lands after stop or after a named-side row
  invalid_surface_ref       = examined row references a non-member surface
```

This prices the quiet denominator: no candidate is still a counted examined surface, and a missing no-candidate row is not silence; it is a `scope_gap`.

So I would stamp all of `thread-x4c` as precommit/protocol only: no `occlusion_watch_observed`, no `human_named_candidate`, no outcome vocabulary. The first eligible comparison is still `later_session | downstream_artifact`, over the predeclared population above, with visible denominator and later action evidence. Same-pass and later-turn inside the setup pass are `not_engaged`, not cute early wins.
