## 1. Manufactured `earned` is possible from trusted row fields + substring matching

`compute_outcomes` treats externality as `watched_agent_is_author is False` and candidate identity as `(observed candidate_key).lower() in candidate_ref` (`harness/occlusion_watch.py:298-325`). It does not verify that `named_ts` is substrate/git-stamped, that `author_kind` is non-watched, that the evidence is an artifact rather than a string/list field, or that the candidate identity is exact. A handwritten row can therefore satisfy the guards by assertion.

I ran this synthetic probe against the actual function:

```text
observed = occlusion_watch_observed(candidate_key="seam_distance",
  surface_basis="work_product", seam_distance="later_session",
  observed_ts="2026-07-01T00:00:00Z")

named = human_named_candidate(candidate_ref="not actually the same wound: seam_distance appears in this sentence",
  watched_agent_is_author=False, named_ts="2026-07-02T00:00:00Z",
  evidence=["asserted-patch"], did_it_change_attention_or_action=True)

compute_outcomes([observed, named]) -> verdict earned, catches 1
```

That is a fake catch past the advertised guards. The problem is not that Layer 2 has a happy-path `earned` fixture; the problem is that the guard inputs are themselves unaudited JSON fields. This violates the §2 important sentence if anyone cites Layer 2 as more than a toy advisory. **Block any claim that `earned` is structurally unmanufacturable until named-side source validation and exact candidate identity are enforced.**

## 2. `SESSION_GAP=4h` can over-claim later_session

The docstring says the proxy can only under-claim, never invent a catch (`harness/occlusion_watch.py:31-34`). The implementation segments the combined witnessed timeline whenever adjacent timestamps differ by `> SESSION_GAP` (`harness/occlusion_watch.py:112-129`, `211-222`). A same human/agent work session with a 4h01 lunch/meeting/build gap becomes `later_session`.

Probe:

```text
precommit 20260626T120000000Z
surface   20260626T160100000Z
same actual session, 4h01 apart
seam_distance(..., timeline) -> later_session
```

So the proxy does not fail only toward under-claim. It can create earned eligibility from wall-clock silence. That is a hard seam blocker: with the fake-earned row above, the gap proxy supplies the missing `later_session` guard.

## 3. Precommit parsing is gameable and non-canonical

The parser scrapes the whole precommit body with two regexes (`KEY_RE` / `PATH_RE`, `harness/occlusion_watch.py:61-62`, loaded at `148-149`). It is not parsing the declared block.

Probe result:

```text
candidate_key: witness invariant -> §2      # unquoted intended key, dropped
“seam_distance” -> §11                       # curly-quoted intended key, dropped
Do not include this example: "smuggled_key" -> nope

parsed_keys -> ('smuggled_key',)
```

That means ordinary prose can smuggle a key, and reasonable precommit phrasing can silently drop one. This is exactly the borrowed-foresight path §11 was trying to close. **Block until the precommit has a canonical fenced/JSON/YAML shape or the parser refuses ambiguous prose.**

## 4. The scoreboard lets the dogfood spray disappear

This is the biggest honesty problem in the current artifact.

Layer 1 over the live repo finds the watch firing on its own construction:

```text
surfaces examined: 6 (S2 diffs: 5)
observed key-hits: 11
```

Layer 2 reads only `runs/x4/occlusion_watch.jsonl`, which currently has one row — the dan flinch — and no persisted `route_watch_surface_examined` / `occlusion_watch_observed` rows. So it prints:

```text
catches 0, flinches 1, fire_rate 0.0, noisy False
```

That is honest only for the persisted ledger. It is not honest as a statement about the declared S1∪S2 population or the dogfood build. The current code computes noisy as `len(observed) / len(examined)` over ledger rows (`harness/occlusion_watch.py:292-294`, `360-365`), while SPEC §11 defines the denominator as `examined-with-candidates / declared population` (`notes/SPEC_X4_OCCLUSION_WATCH.md:294-296`). Two issues follow:

1. generated-but-unwritten observations vanish from the outcome denominator;
2. if they were written, the numerator is key-hit rows, not examined surfaces, so a multi-key surface can push fire_rate above 1.0.

The dogfood is not theater if it is explicitly calibration. But calibration spray still has to be counted as calibration spray. Right now `0 catches / 1 flinch / fire_rate 0.0` lets the organ off easy.

- not “earned is structurally unmanufacturable”;
