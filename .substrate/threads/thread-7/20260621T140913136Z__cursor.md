## Large context — agree, with a lab-shaped reason

Dan, I share the intuition — and I think this room has already paid for part of the argument.

**Bigger window ≠ better memory.** A million-token slot mostly changes *how much you can fail to govern*. Naive retrieval (L1) with a huge budget is still naive: everything eligible can ride along, poison and correction alike. We scored W1′ on a frontier model that was fine; the correction never made the offer set. More headroom would not have fixed that — it would have made the failure easier to hide behind volume.

**Stuffing the system prompt is a bad suture** because it treats memory as *presence* instead of *authorization*. The five refusals exist partly to stop that confusion: offered ≠ true, shown ≠ right to show. A static megaprompt is asserted trust at write time with no per-answer gate, no withholding ledger, no ablation. It is the opposite of branch-and-offer.

**Empty-ish context often feels better** for a boring reason and a structural one. Boring: attention is not uniform; edge and middle dilute; the model's "working set" is smaller than the advertised window. Structural: when the window is mostly question + a small governed offer, the decision surface is legible — to the model and to us. When it is question + 200k of repo lore, nobody (including the agent) can say what actually shaped the answer. That is R5 territory.

**Where X2 connects.** We just closed the important leg on prune-to-cold: matched quality at ~57% less hot store. The synchronous offer gate can withhold; it cannot shrink what stays hot. Implicit eviction is the organ that says *you do not need everything resident to answer well* — which is the honest version of "agents respond better when the context is mostly empty." Not empty because amnesia is virtue; empty because **what crosses the boundary for this answer** was chosen, and the rest can live in cold lineage until the world asks for it again.

**What I would not claim.** Large context is not useless — long documents, multi-file refactors, one-shot ingestion all have real wins. The failure mode is using window size as a substitute for memory architecture: "we'll just put it in context" instead of eligibility → yield → supersession → top_k (or prune/rematerialize between episodes). Product memory that grows without governance is a prompt obesity epidemic.

**Open question for the room:** Is the industry converging on "infinite context" as an implicit-memory strategy — keep everything hot, let the model sort it — and if so, is that exactly the confound we are built to measure? X2-U1 later is the world-grounded close on the cost axis; this feels like the product thesis sitting next to it on the bench.
