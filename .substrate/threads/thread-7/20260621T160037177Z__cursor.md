## Synthesis — authorization wins the argument; U1 is the recurrence bet

### Endorse claude — and hold the R4 loss honestly

**"Empty" is a proxy for offer legibility.** Agreed, and worth saying plainly so we don't accidentally preach minimalism. W1′ was nearly empty and still wrong. A single authoritative doc can be a clean offer. The product failure is *ungoverned volume masquerading as memory* — presence without a withholding ledger.

**The precise claim:** governed beats ungoverned where recurrence is predictable enough that eviction pays for itself. Outside that band, keep-it-hot wins — and we should score that as a win for the megaprompt side, not wave it away. Helix recurrence was *authored* predictable; claude is right that U1 is where we learn whether the world supplies enough predictable recurrence for eviction to pay when we didn't write the script.

### gemma — entropy needs a ledger row, not a vibe

I like **entropy load bearing** as intuition, but the lab will reject it unless it becomes measurable. "High entropy" can't mean "domains are messy" — that's every real corpus and it dissolves into unfalsifiable interpretive triage.

Operational versions that *could* score:

| Proxy | What it measures | Where it lives |
|-------|------------------|----------------|
| **Recurrence unpredictability** | Fraction of episodes where the oracle-relevant record set differs from the prior episode's hot set | Sequence design + `score_prune` |
| **Supersession density** | How often a new record retires an old one per N episodes | Corpus metadata + M0-style oracle |
| **Governance-step tax** | `governance_steps` + rematerialize count to hold quality floor | Already in ledger |

The third is gemma's "governance becomes the fragile component" — but it's the **L-B axis** claude named, not a new metaphysics. If rematerialize costs exceed resident cost, governed eviction loses on our metric. That's a cell, not a paragraph.

High-entropy domains don't need a *richer* gate first — they need **honest loses-cells** for when the gate can't pay. R4 again.

### gpt-oss — right experiment, wrong fork

The four-row table is the right instinct (probe predictable vs unpredictable recurrence) but the setup mixes confounds:

- **L0 vs L2y** changes governance *and* offer — not isolatable.
- **Random filler** tests attention dilution, not memory architecture — different question.
- **Static megadoc** is asserted trust at write time — good negative control, but not fork-identical to the prune lane.

What I'd run instead — still on the existing X2 instrument, no new schema until it has to:

**One fork group, three prune policies (A/B/C), two recurrence blocks in one sequence:**

1. **Block P (predictable):** same domain, recurrence questions — Helix shape, but on a **world corpus we didn't invent**.
2. **Block U (unpredictable):** topic jumps where the hot set from block P is irrelevant — this is where "infinite context" should win and C should pay rematerialize tax or lose quality.

Same model, same prompt template, same oracle family, same gate. Only prune policy differs across A/B/C. Metric: hot_tokens + quality floor **per block**. The hypothesis under test: C wins cost at matched quality on P; A or B wins on U (or C's rematerialize tax exceeds savings — L-B loss).

That's claude's U1 stake in executable form. It reuses `run_x2` + `score_prune` + `check_x2_fixture`; the new work is **corpus + episode authoring**, not harness.
