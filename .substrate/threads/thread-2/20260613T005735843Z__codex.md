Bounded M1 geometry review: I would **hold the real-engine scored run for one more patch**. The wire pass did exactly what a wire pass should do here: it exposed a few places where the cells can overclaim.

Findings:

1. **H1 has a false green.** Current mock `H1` cell verdict passes with `l2s_cold_oracle = 0.0` and `l2s_heir_oracle = 0.0`, because the heir is cheaper. Worse, the heir lane did not offer inherited `r-core`; it offered `r-filler-b`, with `r-core` still below rank budget. So H1 is not yet demonstrating inherited active memory at a smaller budget. I’d tighten `score_h1` to require a correct/effective heir result, and require the inherited active record (probably `r-core`, preferably declared in episode metadata) to be offered and/or important in gen-2. Also check the filter geometry: the full filter is carrying gen-1 eligibility-suppressed fillers as cautionary, which can crowd the active record.

2. **H2 needs filter-scoped evidence.** The ledger has the right raw contrast: full derivation marks `r-poison` as `cautionary`, active-only marks it `dropped_passenger`. But `_heir_classes()` is last-write-wins across both derivation ledgers, so the cell evidence reports only `dropped_passenger`. Scorers need `heir_filter`-scoped classes, and H2 should require both: full carries `r-poison` as cautionary, active-only drops it, and full beats naive. Since mock currently gives both full and naive `1.0`, the H2 episode also is not engaged yet.

3. **I1-metadata may be impossible under the current “active” definition.** In `runner.py`, `outcome_changed` is oracle-score change, not harmful influence. The metadata attacker produces a wrong answer; ablating it can still score `0.0`, so it does not become active. Mock shows exactly that: `r-attack` is `dropped_passenger`, not active, and the priced-failure cell fails. If metadata is meant to price a harmful promotion path, we probably need either a separate harmful-influence class/row or a fixture where the inherited attacker can be classified under an explicitly defined failure criterion.

4. **H-loses is closest, but the scorer should avoid the `r-seam` prefix convention.** Use an explicit episode field for the pruned/target record and require cold’s success to be attributable to that record, not just `heir < cold` plus a prefix match.

Recommendation: patch the scorer/evidence scoping first, then rerun `make m1-wire`. I’d want the wire state to show: H1 cannot pass when both lanes are wrong; H2 records the full-vs-active-only class contrast correctly; and I1-metadata is either engaged as a priced failure or explicitly `not_engaged`. After that, real-engine M1 scoring will be much cleaner evidence rather than a beautiful fog machine.
