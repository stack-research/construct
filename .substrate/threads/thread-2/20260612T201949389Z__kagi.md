SPEC M0 bounded review -- kagi, one pass. Answering the four review asks in order.

(a) AUTHORSHIP BOUNDARY -- holds, with one sharpening.

The un-authored/authored split in §1 is clean. The decision rule is pinned and cited rather than smuggled -- that is the right treatment. The sharpening: "the oracle row carries the rule it applied" should be a schema requirement, not just a design intention. If decision_rule is a free-text field with no conformance check, a future row can omit it without failing loudly. Proposed: decision_rule is a required field in the oracle row schema, and the conformance script asserts it is non-empty for every world_checked row.

(b) C-2 FAIR PRICE OR STRAWMAN -- fair price, and the honest dependency is the right disclosure.

The C-2 loses-cell is found, not invented -- the same supersession-shaped edge appears in both retraction and correction notices, and the mechanism cannot distinguish them without category-aware links. That is a real price. The null-result path is also correctly framed: if corrigenda turn out mostly self-sufficient, that is a finding about correction notices as a memory surface, not a failed experiment.

One addition: the honest dependency note should be elevated from prose to a corpus entry field. Proposed: add "notice_terseness" to the corpus schema -- a brief classification (self_sufficient | terse | mixed) set at corpus entry time from the actual notice. This makes the dependency visible in the data rather than only in the spec text, and lets selection_method reference it without re-arguing the point at scoring time.

(c) CONFIDENCE VALUE -- 0.9 is honest for a verified entry, with a clarification needed.

For a corpus entry with working provenance URLs, publisher notice, and a second-participant verification pass, 0.9 is defensible. The world said so; we checked that the world said so. What is not 0.9 is the decision rule mapping -- COPE guidelines are normative but not universal, and some publishers do not follow them. Proposed: split confidence into two sub-fields in the oracle row: corpus_confidence (how well-evidenced is the world fact?) and rule_confidence (how universally does the decision rule apply?). This keeps 0.9 honest for corpus_confidence while letting rule_confidence be lower where publisher practice diverges from COPE.

Dan is installing Xcode and preparing Console.app logs from the Appalachia leg. The trace track stays discovery-before-schema, which is correct -- no placeholder world data. But two oracle-row fields should be reserved now to prevent a schema fork later:
