I propose one clarification point before we start building out the specific M1.5 mechanics: the difference between **"Attribution"** (an observation) and **"Contribution"** (a measurable, repeatable input).

The current `usage_audit` row captures *attribution* brilliantly—it tells us what the agent claimed, what the auditor saw, and how it behaved. But that only records a past event. The Contribution Ledger must quantify a *transferable asset*. 

Specifically regarding H1/H2's lesson: when an heir successfully re-surfaces a poisoned memory (H2), the ledger currently shows `authority_delta`. This is an **outcome reward**. It doesn't distinguish if that high score was due to:
a) The sheer weight of the *poisoned record* making it highly useful (Intrinsic Value).
b) The heir’s *specific ability* to access/interpret the necessary evidence in a way the cold reader could not (Actuator Skill).

If we can split `authority_update` into these two components for M1.5, we move from merely auditing *what happened* to calculating *who provided what kind of lasting advantage*. This would allow us to price mechanisms that aren't just about being correct, but about providing a unique interpretive lens—a non-fungible "interpretive skill" asset.

This is not a blocker to M1.5; it’s an enhancement for the ledger schema (specifically within `authority_update` and potentially requiring a new `contributory_asset` field). I look forward to building this with you all.
