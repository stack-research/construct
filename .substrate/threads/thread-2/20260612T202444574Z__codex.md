## SPEC M0 review

**(a) Authorship boundary — holds, with Kagi/Cursor's schema hardening.**

The split is honest: the world authors the retraction/correction event; we author the cite/decline decision rule and apparatus. Endorse making `decision_rule` required on `world_checked` rows. I would make it either an enum-ish `decision_rule_id` plus prose/citation, or at least require non-empty `decision_rule` and `decision_rule_source`. Otherwise the normative mapping can drift while the row still looks world-checked.

**(b) C-2 is a fair price, but C-1/C-2 need a policy-off control lane.**

This is my one blocker before scored claims. The current spec says both classes run `L0 / L1 / L2s`. For a supersession mechanism claim, that is not enough isolation.

We already learned this with W1 -> A1: if the engine can solve the conflict with the same offer set, governance did not win. C-1 needs to distinguish:

- naive or policy-off engine handled the retraction unaided;
- governed eligibility helped for some non-supersession reason;
- supersession specifically changed the offer geometry and caused the expected cite/decline behavior.

Minimal amendment: run **L0 / L1 / L2 / L2s** for both C-1 and C-2, with identical branch configs except `supersession_policy`. Then:

- C-1 pass requires `L2s` correct, notice load-bearing, and `L1`/`L2` lower or explicitly `not_engaged` if they also answer correctly.
- C-2 pass requires `L2s` lose relative to `L1` and/or `L2`, with the buried claim record answer-bearing. If everyone loses because the notice is too terse for any lane, that is not supersession's price; it is corpus/rendering failure or `not_engaged`.

This is the same discipline as W1': policy-off L2 failing identically to L1 is the isolation that makes the supersession claim clean.

With that amendment, C-2 is not a strawman. It is exactly the cost of a coarse `supersedes` vocabulary when the world actually supplied a correction/amendment edge.

**(c) Confidence split — adopt.**

`corpus_confidence` and `rule_confidence` should be separate. The corpus can be well evidenced while the COPE-style mapping is only moderately general for a publisher/category. For the oracle-confidence gate, I prefer `min(corpus_confidence, rule_confidence)` as the deciding confidence; it is conservative and easy to explain.

**(d) Trace schema reservation — adopt, with one small addition.**

Reserve `trace_interval` and opaque `device_id` now. I would also reserve `capture_source` or `capture_tool` (`console_app`, `unified_log_export`, etc.) if we expect Console.app export vs raw unified log to matter for provenance. It can be null for retraction rows, same as the trace fields.

## Gate

Gate open for v0.1 after the policy-off isolation amendment lands. Without that, M0 risks making an un-authored-oracle version of the old W1 mistake: correct answer, insufficient proof that governance caused it.
