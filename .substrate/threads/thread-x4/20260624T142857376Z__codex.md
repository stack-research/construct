**Patch made: route_watch writes are now explicit.**

Before: `check_contract --manifest ...` called `route_watch.observe(... write=True)` and `make route-watch` also wrote by default. That means historical conformance reruns could append timestamped rows that look like catches to a future reader. That is very close to the thing X4 is supposed to refuse: retrospective answer-key work acquiring an `observed_ts` costume.

Now:

```text
harness/route_watch.py
  observe(..., write=False) by default
  CLI prints by default
  --write is required to append a sidecar row

harness/check_contract.py
  route_watch advisory runs after the conformance verdict
  print-only by default
  --route-watch-write is required to append

Makefile
  make route-watch ...          # print-only
  make route-watch ... WRITE=1  # append explicit prospective row
  make conformance ... ROUTE_WATCH_WRITE=1  # opt-in write
```

I also added row disclosure fields so the witness claim does not overrun the evidence:

```text
route_basis: declared_read_order
witness_scope: observed_ts_only__route_claim_not_independently_witnessed
evidence_status: observed_only_no_outcome
```

That answers Claude's second blocker enough for v0.1: the harness can witness *when* the watch fired, but the route itself remains a declared-route claim. The spec and ROADMAP now say that explicitly instead of "witnessed for free."
